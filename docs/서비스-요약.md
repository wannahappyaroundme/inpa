# 인파(Inpa) — 서비스 한눈 요약 (설계사 관점·비즈니스 언어)

> 이 문서는 "인파가 보험설계사에게 **어떤 가치**를 주고 **어떤 기능**을 제공하는가"를 개발 용어 없이 정리합니다.
> 사업 방향은 `docs/01~07`, 실제 구현 상태는 코드(`inpa_be`·`inpa_fe`) 기준. 현재 = Phase 1 진행 중(핵심 기능 대부분 동작, 일부는 법무 게이트로 잠금).

---

## 1. 한 줄로

> **인파는 위촉직 보험설계사가 [새 고객 발굴 → 보장 분석 → 갈아타기 제안 → 미팅 성사 → 고객 관리]를 한 흐름으로 끝내게 돕는 'AI 영업 파트너'다.**

분석 도구가 아니라 **영업 도구**입니다. 증권 한 장을 넣으면 "보장이 어디가 비었는지"가 보이고, 그걸 그대로 "고객에게 보낼 제안·문구·링크"로 이어집니다. 끝점이 분석이 아니라 **새 고객·새 계약**이라는 게 핵심입니다.

---

## 2. 설계사가 겪는 진짜 문제 (왜 인파인가)

| 문제 | 현실 | 인파의 답 |
|---|---|---|
| **만날 사람이 없다** | 매달 실적 압박은 오는데, 어려운 건 '설득'이 아니라 **'누구를 만나나'(발굴)** | 갈아타기 제안·고객 셀프진단 링크로 만날 명분을 만든다 |
| **갈아타기가 무섭다** | 발굴 1위 채널인 '기존 보험 갈아타기'가 부당승환(§97) 처벌 공포로 막혀 있음 | 해지손실·면책리셋 등 **불이익을 빠짐없이 짚는 비교안내 보조**로 빠뜨림을 줄인다(최종 비교안내·책임은 설계사) |
| **신입은 '틀'이 없다** | 보장을 어떻게 풀어 설명할지 막막함 | 담보 100+개 표준 틀 + 보험사별 담보명 자동 정리 |
| **분석이 내 화면에만 머문다** | 30분~1시간 만든 분석이 고객에게 전달이 안 됨 → 가치의 90%가 증발 | 분석을 곧장 **고객에게 보내는 링크/문구**로 직결 |
| **고객 관리가 손이 많이 간다** | 만기·생일·미납을 일일이 못 챙겨 방치 → 소개·추가계약 놓침 | 자동 알림 + A/S 위험 감지 + 미팅 예약을 한 곳에서 |

## 3. 인파가 주는 핵심 가치 3가지

```
① 분석 = 무기      증권을 넣으면 '갈아탈 이유'가 즉석에서 보인다  → 미팅을 잡는 총알
② 콘텐츠 = 미끼    무료로 가치를 먼저 보여준다(분석은 열고, 발송은 과금)
③ 관리 = 복리      쓸수록 고객 데이터·관계가 쌓여 소개·추가계약으로 돌아온다
```

## 4. 설계사의 하루 동선으로 보는 기능

설계사의 실제 영업 흐름 순서대로, 각 단계에서 인파가 무엇을 만들어 주는지.

### ① 발굴 — "만날 사람을 만든다"
- **고객 셀프진단 링크**: 설계사가 링크 하나를 뿌리면, 잠재고객이 직접 증권을 올리고 본인 동의만 누르면 자동으로 보장 분석이 됩니다. → **인바운드 문의가 알아서 들어오는 깔때기**. (분석 결과는 곧 설계사의 새 리드)
- **갈아타기 비교안내서**: 기존 보험 vs 제안 보험을 표로 자동 비교하고, 해지손실·면책 리셋 같은 **불리한 점까지 빠짐없이 짚어** 갈아타기 시 빠뜨림을 줄입니다. 점검 전에는 고객 발행을 잠급니다. **법적 비교안내(공식 확인서)·최종 책임은 설계사** (정식 발행은 §97 법무 확정 후 개방).

### ② 분석 — "보장을 한눈에 풀어 보여준다"
- **증권 자동 인식**: 증권 사진/PDF를 올리면 가입한 보험·담보·금액을 자동으로 읽어 정리합니다(손으로 입력 불필요).
- **보험사별 담보명 자동 정리**: 같은 담보라도 회사마다 이름이 다른 걸(예: 암진단급부금 = 암진단보험금) 하나의 표준 이름으로 통일.
- **보장 공백 히트맵(3색)**: 담보별로 "넉넉 / 부족 / 없음"을 색으로 보여줘, 신입도 보장을 풀어 설명할 틀을 갖게 합니다. (부족/넉넉 판정은 설계사가 직접 설정한 기준선이 있을 때만, 아니면 '보유 여부'만 중립 표기)
- **담보 100+ 한눈표**: 15개 이상 카테고리의 전체 담보를 트리로 보여줘 누락·중복을 한눈에.

### ③ 제안·접촉 — "고객에게 보낼 것을 만들어 준다"
- **고객 공유 링크**: 분석 결과를 고객이 볼 수 있는 링크로 만들어 전달. 고객이 언제 열어봤는지까지 확인.
- **AI 카톡 문구**(법무 게이트 연동): 목적별(니즈 환기·만기·생일 등) 메시지를 만들어 **클립보드 복사**로 제공. (자동 발송은 하지 않음 — 정직성 원칙)

### ④ 성사 — "미팅을 잡는다"
- **미팅 예약(캘린들리식)**: 설계사가 가능한 시간을 등록하면, 고객은 예약 링크에서 원하는 시간·방식(대면/전화/화상)을 직접 고릅니다.
- **구글 캘린더 자동 등록**: 미팅이 확정되면 구글 캘린더에 자동으로 일정이 생깁니다(고객 이름은 기본 마스킹).

### ⑤ 관리·유지 — "방치 고객을 자산으로"
- **대시보드(이번 달 목표)**: 월 목표 미팅·목표 보험료를 직접 설정하고 진행률·예상 월급(가입 보험료 × 배율)·이번 달 경과(D-day)를 봅니다.
- **알림 센터 + 리마인더 규칙**: 만기 임박·생일·상담 리마인드·미납·셀프진단 리드 등을 자동 알림. 며칠 전에 받을지 설계사가 설정.
- **환수 레이더(A/S)**: 보유 계약의 납입 상태를 보고 **환수(해지 시 손실) 위험이 있는 고객을 먼저 챙기게** 경보. (수기 입력 기반 추정 — 정확액은 보험사 권위)
- **고객 이력 타임라인**: 한 고객에 대한 분석·공유·동의·미팅 등의 흐름을 시간순으로.

## 5. 기능 전체 목록 (비즈니스 언어로)

| 영역 | 기능 | 설계사에게 주는 것 |
|---|---|---|
| **발굴** | 고객 셀프진단 링크 | 잠재고객이 스스로 진단 → 자동 리드 |
| **발굴** | 갈아타기 비교안내서 | 갈아타기 불이익 점검 보조 + 미점검 시 발행 잠금 (법무 게이트) |
| **분석** | 증권 자동 인식 | 손입력 없이 보장 현황 정리 |
| **분석** | 보험사별 담보명 자동 정리 | 회사 달라도 같은 담보로 통일 |
| **분석** | 보장 공백 히트맵(3색) | 보장을 풀어 설명하는 틀 |
| **분석** | 담보 100+ 한눈표 | 누락·중복을 한눈에 |
| **분석** | 내 기준선 설정 | "이 나이/성별엔 이만큼" 판정 기준을 설계사가 직접 |
| **제안** | 고객 공유 링크 + 열람 확인 | 분석을 고객에게 전달하고 반응 추적 |
| **제안** | AI 카톡 문구(복사) | 보낼 문구를 대신 작성 (법무 게이트) |
| **성사** | 미팅 예약 페이지 | 고객이 시간을 직접 고르게 |
| **성사** | 구글 캘린더 연동 | 확정 미팅 자동 일정화 |
| **관리** | 이번 달 목표 대시보드 | 목표·진행률·예상 월급·D-day |
| **관리** | 알림 + 리마인더 규칙 | 만기·생일·미납 자동 챙김 |
| **관리** | 환수 레이더 | 해지 위험 고객 우선 관리 |
| **관리** | 고객 이력 타임라인 | 고객별 전체 흐름 한 화면 |
| **커뮤니티** | 게시판·공지·FAQ·1:1 문의 | 설계사 간 정보 공유 + 운영팀 소통 |
| **판촉** | 판촉물 주문 | 샘플 보고 폼 입력 → 운영팀이 제작 |
| **계정** | 로그인(이메일/구글)·온보딩 | 가입·자격/소속 신고·매니저 연결 |
| **운영(내부)** | 관리자 콘솔 | 운영팀의 회원·주문·문의·통계 관리 |

## 6. 인파만의 차별점 (해자 3종)

경쟁군(B2C 보험앱·GA 내부 시스템·캔바·범용 AI)이 **절대 못 잇는** 칸.

1. **갈아타기 비교안내서(불이익 가드레일)** — 해지손실·면책리셋·예정이율 등 갈아타기 불이익을 빠짐없이 점검(미점검 시 발행 잠금). §97 책임 회피용 '합법 보증'이 아니라 빠뜨림 방지 보조 — 범용 AI·캔바는 못 줌.
2. **담보 100+ 표준 틀 + 보험사별 담보명 정리 사전** — 쓸수록 정확해지는 데이터 자산(후발주자 진입장벽).
3. **끊김 없는 한 흐름** — 발굴→분석→갈아타기→문구→링크→미팅→관리를 한 도구에서.

## 7. 인파가 "일부러 안 하는 것" (= 신뢰 포인트)

설계사와 고객의 신뢰를 위해 의도적으로 막아둔 선. 영업이지만 정직합니다.

- **"심의 완료/안전" 배지 안 붙임** → AI 결과물엔 항상 "AI 초안, 최종 책임은 설계사" 면책 표기.
- **원탭 자동 발송 안 함** → 카톡 자동발송 사칭 대신 '클립보드 복사'까지만 정직하게.
- **불법 DB 명단 영업 안 함** → 동의 기반 인바운드만.
- **민감정보(병력) 함부로 수집 안 함** → 국외이전 동의·법무 확정 전까지 수집 자체를 차단.

## 8. 요금 (비즈니스 모델)

- **Freemium**: 기능은 다 열되, 무료는 월 사용 횟수 제한 / 헤비 유저는 구독.
- 분석(증권 인식·히트맵·비교안내서 1건 생성)은 **무료**, 반복 발송·복수 생성·AI 메시지 등 '행동'은 유료(Plus+).
- **현재 베타**: 횟수 무제한, 결제·수치 한도는 정식 출시 전 적용 예정.

## 9. 현재 상태 (2026-06 기준)

- **동작 중**: 회원/로그인(이메일+구글), 증권 인식, 담보 정리, 히트맵, 고객/공유 링크, 셀프진단, 미팅 예약+구글 캘린더, 대시보드, 알림/리마인더, 환수 레이더, 게시판, 판촉물 주문, 관리자 콘솔.
- **법무 게이트로 잠금(정식 출시 전 개방)**: 갈아타기 비교안내서 **고객 발행**(§97 확정 필요), AI 메시지·병력 수집(국외이전 동의 확정 필요).
- 분석·진단 기능은 잠금 상태에서도 **중립 모드**(보유 여부만)로 동작하도록 설계됨.
