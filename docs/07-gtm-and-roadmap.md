# 인파(Inpa) — GTM(영업·마케팅) & 로드맵

> 본 문서는 **인파(Inpa)**의 시장 진입 전략(Go-To-Market)과 출시 로드맵을 다룬다.
> 투자자에게는 *어떻게 0→1000명을 만들고 단위경제를 닫는가*, 디자이너·개발자에게는 *무엇을 언제 만들고 무엇을 Day1에 계측하는가*를 한 문서로 제공한다.
> 정량값은 베타 90일 실측 전 가설이며 **(추정)** 라벨을 유지한다. 제품명은 모든 카피에서 **인파(Inpa)** 로 통일한다.

---

## 0. GTM 한 문장과 전제

> **인파의 GTM은 "분석을 미끼로 깔고, 발굴 행동에 과금하며, 공유링크로 바이럴을 자동 귀속하는" 구조다.**

세 줄 전제를 먼저 못 박는다. 이 셋이 어긋나면 아래 모든 전술이 무너진다.

1. **분석은 끝점이 아니라 영업 행동의 시작점(미끼)이다.** 따라서 가입자수·MAU는 보조지표로 강등하고 북극성은 *발굴 행동*을 센다.
2. **초기 100명은 디지털로 안 모인다.** 위촉직 설계사는 검색·광고로 안 움직이고 **지점이라는 물리적 군집**에 있다. 0→100은 손으로 만난다.
3. **귀속 계측은 사후 복원이 불가능하다.** `?ref=` 파라미터와 이벤트 스펙은 **첫 배포(Day1) 전**에 확정한다. 바이럴 *캠페인*은 늦춰도 *계측*은 못 늦춘다.

## 1. 북극성 지표 — GTM의 나침반

모든 채널·오퍼·퍼널은 단 하나의 숫자로 수렴한다.

> ### 북극성 = (미팅당 공유링크 발송수) × (열람률) × (소개 귀속 리드수)

**왜 곱셈 구조인가**

```
   발송 ───────×─────── 열람 ───────×─────── 소개귀속
 (많이 보내도)      (안 열리면 0)       (소개로 안 이어지면 0)
        └──────────── "발송→열람→소개" 영업 루프 전체를 한 숫자로 압축 ────────────┘
```

발송만 많고 안 열리면 0, 열려도 소개로 안 이어지면 0. 곱이기 때문에 어느 한 단계도 방치할 수 없고, 이 한 숫자가 곧 "영업 OS로서의 인파"의 정체성 증거다(분석 건수를 세면 계산기, 이 지표를 세면 영업 OS).

**계측 신뢰도 원칙 (충돌6 합의)**
- 카카오 원탭 자동발송이 불가하므로 "발송" 시점을 직접 추적할 수 없다.
- **신뢰 KPI = `share_view`(share_token 열람, 서버 측정 가능).** 수신자 행동이지만 위변조 불가하고 서버가 직접 잡는다.
- **클립보드 복사(`share_clipboard_copy`)는 '발송 의도' 프록시 보조지표로만** 사용. 복사→실발송 전환율 가정은 베타 검증 대상(openQuestion Q6).
- AI 메시지 복사는 별도 보조 카운터로 분리.

## 2. 이벤트 계측 스펙 — Day1 필수 (사후 복원 불가)

귀속 인프라는 첫 배포 전에 박는다. foliio의 `analytics/` 앱 + GA4를 재활용하고, 신규 `ref` 귀속 테이블만 추가한다. **계측은 Day1, 캠페인은 Phase2** (충돌4 합의) — 즉 인프라는 깔되 루프 활성화는 미룬다.

| 퍼널 단계 | 이벤트명 | 계측 위치(BE/FE) | 북극성 역할 | 비고 |
|---|---|---|---|---|
| 증권 업로드 | `ocr_upload` | detect API (BE) | — | aha 직전 |
| 분석 완성 | `analysis_complete` | calculate.py 후 (BE) | — | aha 순간 |
| 공유링크 생성 | `share_link_create` | generate_share_link (BE) | 발송 의도 | — |
| 클립보드 복사 | `share_clipboard_copy` | 복사 버튼 (FE) | **발송 프록시(보조)** | 신뢰 KPI 아님 |
| 링크 열람 | `share_view` | 공유뷰 진입 share_token (BE) | **열람률 분자(신뢰)** | 서버 측정 |
| 셀프진단 시작 | `selfdiag_start` | `?ref=` 랜딩 (FE→BE) | 소개 리드 | Phase2 활성화 |
| 소개 귀속 | `referral_attributed` | ref 매칭 (BE) | **바이럴 분자** | Phase2 활성화 |

> **개발자 노트**: `share_token`에 `?ref=<설계사코드>` 파라미터를 더하는 것이 귀속의 물리적 구현 전부다. foliio의 `share_token`은 이미 `share_expires_at` 만료를 지원하므로 코드 비용이 거의 0이다. 이벤트 6종 + ref 1종 스펙을 **첫 배포 전 동결**한다. 빠뜨리면 과거 데이터는 영영 못 만든다.

## 3. 오프라인 1:1 방문영업 — 0→100 (유일한 진짜 채널)

검색·광고로 안 모이는 위촉직 설계사를 **지점 군집**에서 손으로 만난다. 채널 우선순위는 "한 번에 군집을 잡을 수 있는 순서"다.

### 3.1 어디서 만나나 (채널 우선순위)

| 순위 | 채널 | 왜 여기인가 | 한 번에 닿는 인원 (추정) |
|---|---|---|---|
| 1 | **GA 지점 / 원수사 사업단** | 지점장은 **신입 생산성이 본인 KPI** → 게이트키퍼가 아니라 우군. "신입 OT 15분 데모" 제안. | 5~20명 군집 |
| 2 | **생/손보 자격시험학원·합격자 단톡** | 갓 합격한 신입 = 고객0·틀0·발굴 절박. 1순위 페르소나가 그대로 모임. | 단톡당 30~100명 |
| 3 | **GA 리크루팅 설명회 / 코드입과 교육장** | 신규 위촉 직후 = 도구 채택 골든타임. | 회당 10~40명 |
| 4 | **설계사 오픈채팅·네이버 카페** | 디지털이지만 오프라인 신뢰의 연장. *정보 제공자*로 먼저 진입. | 채팅방당 50~300명 |

> 핵심: **지점장을 적이 아니라 우군으로 본다.** 신입이 빨리 계약을 따면 지점장 KPI가 오르므로, 인파는 "신입을 빨리 생산적으로 만드는 도구"로 포지셔닝하면 지점장이 직접 OT 슬롯을 내준다.

### 3.2 15분 지점 데모 스크립트

**규칙: 샘플 데이터 금지. 반드시 그 설계사 본인 고객의 실제 증권으로 시연한다.** 본인 데이터가 화면에서 살아 움직이는 순간이 전환의 전부다.

```
┌──────────────────────────────────────────────────────────────┐
│  15분 데모 타임라인                                            │
├──────────┬───────────────────────────────────────────────────┤
│ 0:00 후크 │ "이번 달 갈아타기 제안 몇 건 하셨어요?              │
│          │  비교안내확인서 직접 손으로 쓰시죠? 그거 자동입니다." │
├──────────┼───────────────────────────────────────────────────┤
│ 2:00 실연 │ 본인 고객 증권 1장 업로드                          │
│          │   → 30초 만에 담보 히트맵 (🟢충분 🟡부족 🔴없음)    │
│          │   → 갈아타기 비교표 (좌 기존 / 우 제안 인디고)       │
│          │   → 카톡 문구 자동 생성                              │
├──────────┼───────────────────────────────────────────────────┤
│ 10:00 공포│ "§97 부당승환 걸리면 영업정지인데…"                 │
│   → 방패  │ "…인파는 비교안내서가 자동으로 남아 합법 방패가     │
│          │  됩니다. AI는 1차 보조, 최종책임은 설계사님."        │
├──────────┼───────────────────────────────────────────────────┤
│ 13:00 클로│ "지금 가입하면 평생 베타 멤버. 동료 1명 소개하면    │
│   징      │  Premium 한 달 무료." → QR 스캔 즉시 카카오 가입    │
└──────────┴───────────────────────────────────────────────────┘
```

**스크립트 설계 원리**
- **후크는 발굴 고통을 찌른다** (quota 압박, 손으로 쓰는 비교안내확인서).
- **실연은 5분 핵심 루프를 30초로 압축** 보여준다 (증권→히트맵→비교표→멘트).
- **공포→방패 전환이 클라이맥스.** §97 처벌 공포를 합법 방패 기능으로 뒤집는 메시지가 모든 채널의 후크다.
- **클로징은 즉시 가입 + 더블사이드 리퍼럴 씨앗**을 동시에 심는다.

> **컴플라이언스 가드 (정직성 레드라인)**: 데모·카피 어디에도 "심의 안전/완료" 배지를 쓰지 않는다. "AI 1차 보조, 최종책임 설계사" 면책만 고정한다. '파트너'라는 단어가 보험모집 위촉관계로 오해되지 않도록 "모집 보조 도구"를 병기한다.

### 3.3 오퍼 — 초기 100명 한정

- [ ] **평생 베타 멤버십** — `SUPER_BETA_EXPIRY=2099-12-31`를 그대로 활용. 정식 출시 시 유료기능 영구 할인. 코드 변경 0.
- [ ] **더블사이드 리퍼럴** — 추천 1명당 양쪽 Premium 1개월. `Notification` 패턴 재활용. **상한 연 3개월·셀프추천/유령계정 차단** (과금 거버넌스 합의, openQuestion Q10).
- [ ] **콜드스타트 부트스트랩** — 신입의 첫 고객 5명 분석을 온보딩 콜에서 같이 입력·분석해준다(대행). 신입의 **"첫 성공"을 손으로 만들어 주는 것**이 활성화율을 결정한다. 빈 화면에서 첫 5분에 이탈하는 콜드스타트 리스크를 직접 헤지.

```
콜드스타트 부트스트랩의 논리
  신입 = 고객0·데이터0 → 가입해도 빈 화면 → 5분 내 이탈
        └─ 해결: 온보딩 콜에서 본인 고객 증권 5장 같이 업로드 → 첫 히트맵 = 첫 aha
                 → "이게 내 고객이라니" 정서적 락인 → 활성 사용자 전환
```

## 4. 디지털 확산 — 100→1000 (증폭기, 후행)

오프라인으로 100명을 만든 뒤, 디지털을 *증폭기*로 켠다. 디지털을 먼저 켜면 콜드스타트 군집이 없어 공회전한다.

| 채널 | 자산 재활용 | 바이럴 메커니즘 | 컴플라이언스 |
|---|---|---|---|
| **콘텐츠 SEO/유튜브** | 정규화 사전을 SEO 자산화 | "[보험사명] 담보명 사전", "§97 합법 갈아타기" 검색 인입 = 자발적 인바운드 | 검색 인입이라 무작위 발굴 금지(⑤)와 무관, 안전 |
| **설계사 커뮤니티** | foliio `community/` 그대로 | 노하우 공유장 → 콘텐츠 락인 + 바이럴 | PII 필터(`content_filter.py`) 재사용 |
| **퍼스널 브랜딩 판촉물** | foliio `promotion/` 14템플릿 | 산출물에 **"Powered by 인파"** 워터마크 → 무료 노출 루프 | 워터마크는 **과금이 아니라 바이럴 장치**(전 티어 공통) |

> **정규화 사전의 이중 용도**: 해자(데이터 복리)이면서 동시에 SEO 자산이다. "삼성생명 암진단급부금 = 일반 암 진단비" 같은 매핑 페이지가 검색 인입을 만든다. 데이터 플라이휠이 그대로 콘텐츠 깔때기로 전환된다.

## 5. 추천·바이럴 루프 (share_token 재활용, 코드비용 ≈ 0)

foliio의 `share_token`에 `?ref=<설계사코드>` 한 파라미터를 더해 두 개의 루프를 완성한다. **계측은 Day1, 루프 활성화는 Phase2** (충돌4).

### 5.1 루프 A — 고객 분석 공유 (1차 바이럴)

```
설계사 ──분석링크 발송──▶ 고객 ──본인 보장공백 확인──▶ "내 지인도 받아볼래"
                                                          │
                                            ?ref= 박힌 셀프진단 링크
                                                          │
                                                          ▼
                                          지인이 셀프진단 입력 ──▶ 소개 리드 자동 귀속
                                          (share_view 자체가 무료 노출)
```

### 5.2 루프 B — 설계사 추천 (더블사이드)

```
설계사 A ──동료 B 초대(ref 코드)──▶ B 가입 ──▶ 양쪽 Premium 1개월 적립
                                              (Notification "추천 보상 적립")
   상한: 연 3개월 / 셀프추천·유령계정 차단
```

### 5.3 바이럴의 컴플라이언스 게이트 (충돌4 + openQuestion Q4 — 미해소)

> ⚠️ **루프 A는 법무 게이트에 종속된다.** "소개받은 지인"이 합법이려면 반드시 **지인 본인이 자발적으로 셀프진단을 입력**(인바운드)해야 한다. 설계사가 먼저 지인에게 연락하면 *무작위 발굴 금지(⑤)* 위반 소지. 그로스는 설계사 푸시를 원하지만, **셀프진단 인바운드만 귀속 인정**할지·설계사 푸시 허용 범위를 어디까지 둘지는 준법 자문 필요(Q4).
> - share_token 만료·회수·`robots noindex` 정책도 동반 확정.
> - 제3자(지인) 정보 수집 동의 1탭의 법적 충분성도 게이트 항목.

## 6. 인파 브랜드 — '당신 곁(In) 파트너' 당근식 활용

| 항목 | 내용 |
|---|---|
| **작명 자산** | 인슈어(Insurance) + 파트너(Partner). "당근=당신근처"식 **실존단어 + 약어** 작명 → 입에 붙고 검색 키워드 안 겹침. |
| **캐치프레이즈** | **"설계사 곁의 AI 영업 파트너, 인파."** |
| **톤** | 도구(tool)가 아니라 **동료(partner).** "혼자 발굴하지 마세요. 인파랑 같이." 신입의 외로움·막막함을 정서적으로 친다. |
| **가격 효과** | '동료' 포지션이 '소프트웨어 구독'보다 가격 저항을 낮춘다. |

> **법무 주의 (정직성 레드라인)**: '파트너'가 **보험모집 위탁(위촉) 관계**로 오해되면 무자격모집(④) 리스크. 모든 마케팅 카피에 **"모집 보조 도구, 최종책임 설계사"** 면책을 일관 표기한다. **상표 출원 가능성 사전 점검** 필요(인파/Inpa 선점 여부).

## 7. Freemium 전환 퍼널 — 한도선이 단일 최대 레버

원칙: **"미끼는 풀고, 행동은 막는다."** 분석·OCR·히트맵은 마찰 없이 무료, **발송·자동화·귀속·워치독**을 유료벽 뒤로.

### 7.1 티어별 무료/유료 라인 (충돌1 합의)

| 티어 | 가격(월, KRW) | 무료/포함 | 유료벽(다음 티어로) |
|---|---|---|---|
| **Basic** | 0 | OCR·히트맵·**비교안내서 1건 *생성*까지** | 비교안내서 *발송·복수 생성*, AI 메시지, 워치독, 귀속 ✕ |
| **Plus** | 19,900 | AI 메시지 엔진·니즈환기·무제한 진단·비교안내서 발송 | 워치독·리퍼럴 귀속·알림톡 ✕ |
| **Premium** | 89,900 | 워치독·리퍼럴 귀속·알림톡 | — |
| **Super** | 견적 | 팀/지점 단위 | — |

> **충돌1 핵심 합의 — "분석은 풀고 행동은 막는다"**
> - 마케터(미끼 최대화)·사업관리(전환율 방어)·기획(메시지 횟수 과금)의 3파전을 대표가 중재.
> - **비교안내서 1건 *생성*은 무료로 풀어 wedge를 보여주되**, *발송·복수 생성·AI 메시지·워치독·귀속*은 Plus+로 막아 반복 발굴은 과금.
> - **워터마크 강제과금 금지** — 가드레일은 전 티어 위생이어야 §97 방패가 신뢰를 얻는다.
> - 정확한 한도 숫자(월 3건/10건 등)는 **베타 90일 실측 후 토큰화**. 전부 **(추정)** 라벨 유지.

### 7.2 전환 트리거 설계

```
무료 사용 ──한도 소진 직전──▶ UpgradeGuideModal (402 CREDIT_EXHAUSTED 재활용)
   │                              └─ {reason:CREDIT_EXHAUSTED, limit, remaining}
   │
   ├─ 14일 풀언락 트라이얼 ──▶ aha를 한도 막힘 전에 먼저 경험시킴
   │   (신입이 첫 성공 직전에 끊기는 이탈 방지)
   │
   └─ 워치독 알림 도착 ──▶ "발송은 Premium" ──▶ 가장 강한 업셀 지점
       (만기 = 시간압박 = 지불의사 최고)
```

- **① 한도 소진 직전 모달**: `402 CREDIT_EXHAUSTED` → FE `UpgradeGuideModal`. 베타엔 `FREE_TIER_UNLIMITED=True`, 정식 출시 시 `False` flip.
- **② 14일 풀언락 트라이얼**: 신입이 한도에 막혀 aha 전에 이탈하는 것을 방지. 첫 성공 후 한도를 체감시켜 전환.
- **③ 워치독 업셀**: 만기 알림은 시간압박이라 지불의사가 최고. "알림은 무료, 발송은 Premium"이 가장 강한 업셀.

## 8. 채널 우선순위 — CAC 낮은 순

장기적으로는 CAC가 낮은 채널로 무게중심을 옮기되, **시점이 다르다**. 0→100 구간에는 CAC가 높아도 door-to-door가 유일하고, 100→1000 구간에서 인바운드·바이럴로 전환한다.

```
CAC 낮음 ◀─────────────────────────────────────────────▶ CAC 높음
  인바운드 SEO        지인소개 바이럴        지점 군집 door-to-door
  (Phase2~3,           (Phase2,              (Phase1, 0→100,
   self-serve)          ?ref= 더블사이드)      손이 가지만 유일)

  시점:   ③ 마지막 증폭    ② 두 번째 켬       ① 첫 채널 (필수)
```

| 채널 | CAC (추정) | 활성 시점 | 비고 |
|---|---|---|---|
| 인바운드 SEO | 1~2만 | Phase2~3 | 정규화 사전 SEO 자산 |
| 지인소개 바이럴 | 2~3만 | Phase2 | 더블사이드 리퍼럴 |
| 지점 군집 door-to-door | 3~5만 | Phase1 (0→100) | 손이 가지만 콜드스타트 유일 해법 |

## 9. 단계별 로드맵 · 마일스톤

빌드 의존성은 고정이다: **정규화 OCR이 모든 분석의 입력(선행) → 히트맵 → 비교안내서 → 메시지.** 컴플라이언스 게이트가 막히면 **중립 기능(OCR→히트맵 none→정규화)부터** 선출시한다.

```
Phase0 ──▶ Phase1 ──▶ Phase1.5 ──▶ Phase2 ──▶ Phase3
(2주)     (6주)       (4주)        (6주)      (지속)
게이트     발굴 wedge   리텐션·첫유료  성장루프    브랜딩·알림톡·팀
코드0줄    MVP 9개      워치독·과금    바이럴·추천  심의후·Super
```

### Phase 0 — 게이트 확정 (2주, 코드 0줄)

법무 게이트가 출시 가부를 쥔다. **코딩 전 PM·법무 선결.**

- [ ] **표준 보장 기준선 출처·면책 확정** (Q1) — 금감원/보험연구원/자체+면책 중 권위 결정. 확정 전까지 히트맵은 `enough/short` 보류, **`none`(0원 보유여부)만 회색 중립 표기**.
- [ ] **국외이전 동의서 개정안 + ConsentLog 구조** (Q2) — `consent_overseas_at` 별도 필드 + ConsentLog(누가·언제·범위·doc_version·ip). detect 호출 전 미동의 시 **412 게이트**.
- [ ] **§97 비교안내 정확요건 6항목 체크리스트** (Q3) — 미완료 시 발행 하드블록의 법적 근거.
- [ ] **정규화 사전 v0** — 상위 30담보 × 5대 보험사 시드.
- [ ] **ai_guardrail 면책 카피** — 안전배지 금지, 면책카피만.

> **fallback**: 게이트가 막히면 **OCR(M1) → 히트맵 none 중립(M3) → 정규화(M2)** 순으로 중립 기능부터 출시해 발을 들인다. 비교서·메시지는 §97·동의 확정 후 오픈.

## 12. 한 장 요약 (TL;DR)

- **북극성** = (미팅당 공유링크 발송수) × (열람률) × (소개 귀속 리드수). 분석 건수·MAU는 허영지표. 신뢰 KPI는 `share_view`(서버 측정), 클립보드 복사는 발송 프록시 보조.
- **0→100** = 오프라인 지점 door-to-door가 유일 채널. 지점장은 우군. 15분 데모(본인 증권 실연 + §97 공포→방패 전환) + 평생 베타 + 콜드스타트 부트스트랩.
- **100→1000** = 디지털 증폭(콘텐츠 SEO·커뮤니티·Powered by 인파 워터마크) + 바이럴(`?ref=` 더블사이드).
- **계측 Day1, 캠페인 Phase2** — `?ref=` + 이벤트 6종은 첫 배포 전 동결(사후 복원 불가).
- **Freemium 레버** = "분석은 풀고 행동은 막는다." 비교안내서 1건 생성 무료 / 발송·복수·메시지·워치독·귀속은 Plus+. 워터마크 강제과금 금지.
