## 1. 브레이크포인트 체계

### 1.1 인파 표준 브레이크포인트

인파는 **모바일 퍼스트**다. 설계사가 고객 앞에서 한 손으로 쓰는 것이 1순위 UX.

| 이름 | 픽셀 범위 | Tailwind 접두사 | 대표 레이아웃 |
|---|---|---|---|
| `xs` (기본) | 0 ~ 479px | (기본값, 접두사 없음) | 단일 컬럼, 하단탭 |
| `sm` | 480 ~ 767px | `sm:` | 단일 컬럼, 하단탭, 여백 확장 |
| `md` | 768 ~ 1023px | `md:` | 2컬럼 가능, 사이드네비 등장 |
| `lg` | 1024 ~ 1279px | `lg:` | 사이드네비 + 콘텐츠 멀티컬럼 |
| `xl` | 1280px~ | `xl:` | 넓은 데스크톱, max-w 제한 |

**Tailwind config 확장 (계약)**

```js
// tailwind.config.js — screens 오버라이드
screens: {
  xs:  '480px',   // sm과 통합(기본은 0 기준)
  sm:  '480px',
  md:  '768px',
  lg:  '1024px',
  xl:  '1280px',
}
```

> **주의**: CSS 변수 `--header-h`는 `design/tokens/inpa-tokens.css` SSOT. 데스크톱 `80px` / 모바일(≤768) `56px`. Tailwind에서 `pt-[var(--header-h)]`로 참조 — 하드코딩 금지.

### 1.2 컬럼 정책 (화면별 명문화)

dev/15 §5.4를 반응형 축으로 확장한 정본 표.

| 화면 | 모바일(xs~sm) | 태블릿(md) | 데스크톱(lg~) | 비고 |
|---|---|---|---|---|
| 홈(대시보드) | 1열 | 2열 가능 | 2열 (좌:액션큐, 우:캘린더) | 밀도화면 허용 |
| 고객 목록 | 1열 리스트 | 2열 그리드 선택가능 | 2열 그리드 | — |
| 고객 상세 | 1열(탭 4) | 1열(탭 4) | 2열(탭 내용 + 메타패널) 추정 | — |
| 히트맵 | 좌sticky+가로스크롤 | 좌sticky+가로스크롤 | 멀티컬럼 그리드 | dev/08 |
| 공유뷰A | **단일 480~720 중앙** | 동일 | 동일 (다중컬럼 금지) | dev/08 |
| 갈아타기 비교서 | 세로 누적 | 2열(좌기존·우제안) | 2열 | — |
| AI 메시지 | 1열 | 1열 | 1열 | 단순 폼 |
| 게시판 피드 | 1열 카드 | 1~2열 | 2열 | — |
| 판촉물 카탈로그 | 2열 그리드 | 3열 | 4열 | — |
| 캘린더 | 주간 스트립 | 월 그리드 | 월 그리드 + 우DayAgenda | dev/15 |

## 2. 모바일 내비게이션 — 하단탭 + FAB

### 2.1 하단 탭 5개 구조

```
┌─────────────────────────────────────────────┐
│           앱 콘텐츠 영역 (스크롤)              │
│                                             │
│                                             │
└──┬────────────────────────────────────────┬─┘
   │ [홈]  [고객]  [⊕발굴]  [분석]  [내정보] │  ← BottomTabBar
   │  fixed bottom-0 + safe-area-inset-bottom │
   └─────────────────────────────────────────┘
```

| 탭 | 아이콘 | 라우트 | 강조 |
|---|---|---|---|
| 홈 | 🏠 | `/home` | 기본 |
| 고객 | 👥 | `/customer` | 기본 |
| 발굴 | ⊕ | `/prospect` | **중앙 FAB 강조** (크기·색 차별화) |
| 분석 | 📊 | `/analysis` | 기본 |
| 내정보 | 🙋 | `/me` | 기본 |

**FAB(발굴 탭) 강조 처리 계약**

```
<BottomTabBar>
  tabs.map(tab =>
    tab.key === 'prospect'
      ? <ProspectFAB>   ← 원형 버튼, bg-accent (#F76707), 크기 56×56, 위로 8px 튀어오름
      : <TabItem>
  )
```

- FAB는 `position: relative; top: -8px`으로 탭바보다 8px 위로 돌출(디자인 확정 필요 — openGap).
- 탭바 높이: `56px + safe-area-inset-bottom`. iOS 홈바/갤럭시 제스처바 겹침 방지.
- 활성 탭: 아이콘 색 `--brand` + 라벨 `font-weight 700`. 비활성: `--ink-3`.

### 2.2 고객뷰 (/s/:token, /check/:token) — 탭·헤더 완전 숨김

```css
/* CSS 계약 */
.share-layout {
  --header-h: 0;              /* 글로벌 헤더 오버라이드 */
  padding-bottom: 0;          /* 탭바도 없음 */
}
```

- `BottomTabBar`, 글로벌 헤더 DOM에서 **렌더 자체 불가**(조건부 렌더 아닌 레이아웃 분기).
- Next.js `(share)` 라우트 그룹 → 별도 `layout.tsx`로 격리.

### 2.3 데스크톱 전환 — 사이드네비

`md` 이상(768px~)에서 하단탭 대신 사이드네비로 전환.

```
xs~sm (모바일)         md~ (태블릿/데스크톱)
─────────────          ──────────────────────────
하단탭 5개              ┌──────┬──────────────────┐
                       │사이드 │  콘텐츠 영역       │
                       │네비   │  (멀티컬럼 가능)   │
                       │──── │                  │
                       │홈    │                  │
                       │고객  │                  │
                       │⊕발굴 │                  │
                       │분석  │                  │
                       │내정보 │                  │
                       └──────┴──────────────────┘
```

- 사이드네비 너비: `(추정) 200px` (lg이상). md에서는 아이콘만 64px 아이콘레일.
- 콘텐츠 영역 `margin-left: 사이드네비 너비` + `padding-top: var(--header-h)`.

## 3. 터치 타깃 · 세이프에어리어

### 3.1 터치 타깃 최솟값

**모든 인터랙티브 요소 44×44px 이상** (iOS HIG · WCAG 2.5.5 기준).

| 요소 | 최소 시각 크기 | 터치영역 확보 방법 |
|---|---|---|
| 하단탭 아이콘 | 24×24px | 패딩으로 탭 영역 44×44 확보 (`p-3`) |
| 히트맵 셀 | 24×24 (모바일) / 32×32 (데스크톱) | `min-w-[44px] min-h-[44px]` — 시각 크기와 별개로 탭영역 44px |
| 필터칩 | h-8 (32px) 시각 | `py-2` → 탭영역 44px |
| CTA 버튼 (풀폭) | h-12 (48px) | 탭 영역 충분 |
| 스와이프 액션 핸들 | — | 스와이프 제스처(영역 제한 없음) |
| 바텀시트 핸들러 | — | 드래그 영역 40px 이상 |

```css
/* 히트맵 셀 터치영역 확보 계약 */
.heatmap-cell {
  width: 24px; height: 24px;          /* 시각 크기 */
  padding: 10px;                       /* 터치영역 → 44px */
}
@media (min-width: 768px) {
  .heatmap-cell { width: 32px; height: 32px; padding: 6px; }
}
```

### 3.2 세이프에어리어 인셋 패딩

iOS 홈 인디케이터 / 갤럭시 제스처바와 고정 요소 겹침 방지.

```css
/* 적용 대상: 하단 고정 요소 전부 */
.sticky-cta   { padding-bottom: env(safe-area-inset-bottom); }
.bottom-tabbar { padding-bottom: env(safe-area-inset-bottom); }
.bottom-sheet  { padding-bottom: max(16px, env(safe-area-inset-bottom)); }
```

Tailwind에서는 `pb-[env(safe-area-inset-bottom)]` 또는 `pb-safe` 커스텀 유틸리티 등록.

## 4. KPI 카드 반응형 (2열→3열→5열)

홈 대시보드의 KPI 카드 5종이 화면 너비에 따라 열 수를 바꾼다.

### 4.1 레이아웃 변화

```
xs (0~479px)         sm (480~767px)        md (768px~)
─────────────         ──────────────        ────────────────────
[고객][만기]           [고객][만기][할일]      [고객][만기][할일][신규][미열람]
[할일][신규]                                  (5열, 1행 가로 나열)
[미열람]
(2열 그리드)           (3열 그리드)            (5열 가로스크롤 or 1행)
```

Tailwind 클래스 계약:

```html
<div class="grid grid-cols-2 sm:grid-cols-3 md:grid-cols-5 gap-3">
  <KpiCard v="고객" />
  <KpiCard v="만기" />
  <KpiCard v="할일" />
  <KpiCard v="신규" />
  <KpiCard v="미열람" />
</div>
```

xs에서 5번째 카드(미열람)는 `col-span-2`로 2열 중앙 배치 또는 마지막 단독행.

### 4.2 KPI 카드 모바일 명세

- 카드 내부: 숫자 `text-2xl font-900 tabular-nums text-accent-blue` / 라벨 `text-sm text-ink-2`.
- `rounded-[var(--radius)]` (`12px`) + `border border-line` + `shadow-sm`.
- 세로 여백: `p-4` (모바일) / `p-5` (데스크톱).
- **판정어 금지**: 카드 안에 "충분/부족/위험" 텍스트 불가 — 사실 카운트(숫자)만.

## 6. 3축 분석 화면 모바일 레이아웃

고객 상세 화면(`/customer/:id`)의 4탭(분석/갈아타기/공백/이력) 각각.

### 6.1 고객 상세 — 탭 내비

```
┌─────────────────────────────┐
│ [←] 김영희  여 · 1985 · 월24만│  ← 고객 요약 헤더 + 동의칩
├──────────────────────────────┤
│ [분석] [갈아타기] [공백] [이력]│  ← 탭 4개, overflow-x-auto 가로스크롤
├──────────────────────────────┤
│       탭 콘텐츠 영역          │
│       (아래 각 §에서 명세)     │
└──────────────────────────────┘
```

탭 바는 `overflow-x-auto` — 탭 4개가 xs에서 좁으면 가로 스크롤. 탭 최소 너비 `min-w-[72px]`.

### 6.2 분석 탭 — 히트맵 모바일

> dev/08 §6.2 명세와 동일. 본 문서는 반응형 계약만 명문화.

```
xs~sm (모바일)
┌────────────────────────────────────────────┐
│ [간략 | 상세]                                │  ← ViewSegment
│ (전체)(없음)(부족)(충분)─── 가로스크롤 ──────│  ← FilterChips
│ ⓘ 기준선 확정 전 · 보유여부만               │
├────────┬───────────────────────────────────┤
│카테고리 │ 담보1 담보2 담보3 담보4 → (스크롤)  │
│(sticky)│ ┌──┐  ┌──┐  ┌──┐  ┌──┐           │
│진단비  │ │■ │  │┊ │  │▌ │  │■ │           │  셀 24×24
│후유장해 │ │┊ │  │■ │  │┊ │  │▌ │           │  탭영역 44px
│간병    │ │┊ │  │┊ │  │■ │  │┊ │           │
│  …    │  (아코디언 접기/펼치기 ▶)            │
└────────┴───────────────────────────────────┘
```

**모바일 히트맵 기술 계약**

| 항목 | 값 |
|---|---|
| 카테고리 열 | `position: sticky; left: 0; z-index: 10; background: var(--surface)` |
| 셀 영역 | `overflow-x: auto` |
| 셀 크기 | 24×24px (시각) / 44×44px (터치영역, padding으로 확보) |
| 셀 간격 | `gap-0.5` (2px) |
| 셀 radius | `rounded-sm` (4px) |
| 아코디언 | 카테고리별 기본 펼침. 15+ 카테고리 초기 렌더 경량화 |
| 필터칩 | `overflow-x: auto; white-space: nowrap; -webkit-overflow-scrolling: touch` |

**데스크톱 (md~)**: 좌 카테고리 sticky + 우 멀티컬럼 그리드(셀 32×32, gap-0.5). 필터칩 자동줄바꿈(`flex-wrap`).

### 6.3 갈아타기 비교표 모바일 — 세로 누적

```
xs~sm (모바일)              md~ (데스크톱)
──────────────────           ──────────────────────────
[기존 보험] (위)              [기존]      [제안(인디고)]
──────────────────           암진단  2천만   3천만  +1천만
암진단  2천만                 뇌진단  0원     2천만  +2천만
```

## 16. 관리자 페이지 반응형

관리자 페이지는 **설계사 앱과 별도 레이아웃**. 주로 데스크톱 사용이지만 모바일 접근도 지원.

| 화면 | 모바일 | 데스크톱 |
|---|---|---|
| 사용자 목록 | 1열 리스트, 검색바 상단 | 테이블 + 사이드 필터 |
| 통계/대시보드 | 1열 KPI 카드 스택 | 2~3열 그리드 |
| 설정 | 1열 섹션 스택 | 2열(좌:메뉴, 우:내용) |

## 17. 반응형 공통 컴포넌트 계약

### 17.1 공통 유틸 클래스 (재사용 필수)

```html
<!-- 페이지 콘텐츠 컨테이너 -->
<div class="mx-auto px-4 sm:px-6 max-w-[720px] md:max-w-none md:px-8">

<!-- 풀폭 하단 CTA 버튼 -->
<button class="w-full h-12 rounded-[var(--radius)] bg-[var(--brand)] text-white font-bold">

<!-- 탭 영역 44px 보장 (아이콘 버튼) -->
<button class="p-3 min-w-[44px] min-h-[44px] flex items-center justify-center">

<!-- 세이프에어리어 하단 패딩 -->
<div class="pb-[env(safe-area-inset-bottom)]">
```

### 17.2 모달 vs 바텀시트 분기

| 트리거 | 모바일 | 데스크톱 |
|---|---|---|
| 셀 탭 상세 (히트맵) | vaul 바텀시트 | 사이드 드로어 or 바텀시트 동일 |
| 빠른 입력 (+접촉/+메모) | vaul 바텀시트 | 인라인 폼 or 바텀시트 동일 |
| 한도 소진 (402) | 바텀시트 모달 | 중앙 모달 |
| 업로드 큐 (증권 OCR) | 전체 화면 진행 | 사이드 패널 or 모달 |
| 판촉물 주문 | vaul 바텀시트 | 중앙 모달 |

### 17.3 토스트 위치

```
모바일: top-[calc(var(--header-h)+8px)]  ← 헤더 바로 아래, 하단탭 위
데스크톱: top-[calc(var(--header-h)+16px)]  ← 동일
```

공유뷰(헤더 없음): `top-4` 고정.
