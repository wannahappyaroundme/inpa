## 2. 페이지 전체 구조 개요

랜딩 페이지는 7개 섹션으로 구성된다. 섹션 순서는 방문자의 이해 흐름(인지→기능→신뢰→행동)을 따른다.

```
섹션 1  히어로 + CTA
섹션 2  핵심기능 3축
섹션 3  차별점 2종
섹션 4  작동방식 3스텝
섹션 5  요금 (Freemium)
섹션 6  신뢰·정직성 고지
섹션 7  푸터
```

로그인 상태(토큰 존재)인 방문자가 `/`에 접근하면 `/home`으로 **클라이언트 사이드 리다이렉트**한다. 로그인 상태 확인은 `useEffect` 내 토큰 체크로 처리하고, SSG 빌드 자체에는 영향 없다.

## 3. 섹션별 스펙

### 섹션 1 — 히어로 + CTA

**목표**: 방문자가 5초 내에 "나를 위한 서비스인지" 판단할 수 있도록.

#### 카피

| 구분 | 문안 |
|---|---|
| **히어로 헤드라인** | 설계사님은 클로징만 준비하세요 |
| **서브카피** | 발굴부터 보장분석, 갈아타기 제안까지 — 인파가 다 합니다. 설계사님은 계약 마무리에만 집중하세요. |
| **보조 설명** | 증권 업로드 하나로 고객 보장 공백을 3초 안에 시각화. 합법적인 비교안내서까지 자동 생성. |

#### CTA 버튼

| 버튼 | 라우트 | 스타일 |
|---|---|---|
| 무료로 시작하기 | `/register` | Primary — `bg-[--brand]` 솔리드, 텍스트 흰색 |
| 로그인 | `/login` | Ghost — `border-[--brand]` 아웃라인, 텍스트 `--brand` |

두 버튼은 모바일에서 세로 누적(full-width), `md:` 이상에서 가로 병렬.

#### 배경 & 시각 처리

- 배경 틴트: `--accent-tint` (`#EAF2FE`) 그라디언트 또는 단색
- 우측(또는 하단 모바일)에 히트맵·비교표 UI 목업 이미지 또는 일러스트 배치 (자산 미확정 시 placeholder 박스)
- 로고는 글로벌 헤더에 위치 (별도 섹션 아님)

#### 글로벌 헤더 (sticky)

랜딩 전용 헤더가 아닌 공통 헤더 컴포넌트를 재사용한다.

| 항목 | 내용 |
|---|---|
| 좌측 | 인파 로고 (`design/logo/*.svg`) → `/` 링크 |
| 우측 | "무료로 시작하기" (Primary) + "로그인" (Ghost) |
| 배경 | `--surface` 흰색 + `--line` 하단 구분선 |
| sticky 높이 | `--header-h` (데스크톱 80px / 모바일 56px) |

모바일에서 우측 버튼 2개는 아이콘 없이 텍스트만, 충분한 탭 영역(min 44px) 확보.

### 섹션 2 — 핵심기능 3축

**목표**: 인파가 제공하는 3가지 핵심 도구를 한눈에 보여준다.

#### 카드 구성 (3개)

| # | 제목 | 아이콘 힌트 | 설명 |
|---|---|---|---|
| 1 | 보장 한눈표 (히트맵) | 격자/컬러 아이콘 | 고객의 현재 보장을 100개 이상 담보 항목으로 한 화면에 펼쳐 보세요. 충분·부족·없음을 색으로 즉시 확인합니다. |
| 2 | 히트맵 분석 | 차트/인사이트 아이콘 | 보장 공백을 자동으로 계산해 우선순위별로 정렬합니다. 설계사님이 설정한 기준선으로 판단합니다. |
| 3 | 갈아타기 비교안내서 | 교환/비교 아이콘 | 기존 보험과 새 제안을 나란히 비교하는 공식 비교안내서를 생성합니다. 부당승환 예방 항목이 자동으로 포함됩니다. |

> **컴플라이언스 주의**: 카드 설명에서 "충분/부족" 판정을 서비스가 단정하는 표현은 사용하지 않는다. "설계사님이 설정한 기준선으로 판단합니다"처럼 판정 권위가 설계사에게 있음을 명시한다 (`dev/09`, `dev/14`).

#### 레이아웃

- 모바일(xs~sm): 세로 1열 누적 카드
- `md:` 이상: 3열 그리드 (`grid-cols-3`)
- 카드 배경: `--surface-2` / 라디우스: `--radius` (12px) / 그림자: `shadow-sm`
- 카드 상단 아이콘 컬러: `--accent-blue`

### 섹션 3 — 차별점 2종

**목표**: 경쟁 서비스와 다른 점 두 가지를 설계사에게 설득.

#### 차별점 항목

**항목 A — 합법적 갈아타기 비교안내**

```
제목: 부당승환 걱정 없는 비교안내
설명: 보험업법 제97조가 요구하는 6가지 비교 항목(해지환급금 손실·면책기간·예정이율 등)을
      자동으로 체크하고 문서에 포함합니다.
      기준 미달 시 발행 자체를 차단해 설계사님을 법적 리스크에서 보호합니다.
강조 문구: §97 체크리스트 자동 완성 — 미달이면 발행 불가
```

**항목 B — 보험사별 담보명 정규화**

```
제목: 어떤 보험사 증권도 같은 틀로
설명: '암진단급부금', '일반암진단비', '암진단 특약' — 회사마다 다른 이름을
      100개 이상의 표준 담보로 자동 정규화합니다.
      OCR이 쌓일수록 매칭 정확도가 높아지는 학습 구조입니다.
강조 문구: 담보 100+ 표준화 · 보험사별 명칭 자동 매핑
```

#### 레이아웃

- 배경: `--surface` 흰색 또는 `--brand-ink` 짙은 네이비 반전 섹션(강조)
- 모바일: 세로 2개 누적
- `md:` 이상: 2열 (`grid-cols-2`), 좌우 등분
- 각 항목에 강조 뱃지(인라인 chip, `--accent-tint` 배경 + `--brand` 텍스트)로 강조 문구 표시

### 섹션 4 — 작동방식 3스텝

**목표**: "어떻게 쓰나요?"를 3단계로 보여준다. 진입 장벽을 낮추는 것이 목적.

#### 스텝 구성

| 스텝 | 아이콘 힌트 | 제목 | 설명 |
|---|---|---|---|
| 1 | 업로드 아이콘 | 증권 업로드 | 고객 보험 증권 PDF를 드래그하거나 카메라로 찍어 올리세요. 여러 장 한 번에 가능합니다. |
| 2 | 분석 아이콘 | AI 자동 분석 | 인파가 담보를 분류하고 보장 공백을 계산합니다. 보통 10초 이내에 완료됩니다. |
| 3 | 공유 아이콘 | 비교안내서 & 메시지 공유 | 고객에게 보낼 비교안내서와 카톡 메시지를 생성하고, 클립보드로 복사해 전달하세요. |

#### 정직성 주석

스텝 3 아래에 소형 텍스트(`text-sm`, `--ink-3`)로 다음을 표시한다.

> "자동 발송은 지원하지 않습니다. 메시지는 클립보드 복사 후 직접 전달해 주세요."

이 문구는 `dev/14` 정직성 레드라인(원탭 자동발송 없음)의 랜딩 페이지 적용이다.

#### 레이아웃

- 모바일: 세로 3개 누적. 스텝 번호(원형 배지, `--brand` 배경)를 카드 좌상단에 표시
- `md:` 이상: 3열 그리드, 스텝 사이 연결선(점선 또는 화살표 아이콘) 표시
- 배경: `--surface-2` (섹션 배경)

### 섹션 5 — 요금 (Freemium)

**목표**: 진입 장벽이 없음을 강조하고, 헤비유저에게 업그레이드를 유도한다.

#### 요금 카드 (2종)

| 항목 | 무료 플랜 (Free) | Plus 플랜 |
|---|---|---|
| **월 요금** | 0원 | 추후 공개 (베타 실측 후 확정) |
| **증권 OCR** | 월 N건 (추정, 베타 확정) | 더 많이 |
| **비교안내서** | 월 1건 체험 | 복수 |
| **AI 분석·메시지** | 제한 | 제한 완화 |
| **판촉물 주문** | 제한 | 제한 완화 |
| **보장 히트맵 조회** | 무제한 | 무제한 |
| **CTA** | 무료로 시작하기 → `/register` | 베타 신청 / 문의하기 |

> **베타 고지**: 두 카드 아래 소형 텍스트로 표시:
> "베타 기간 한도는 실측 후 공개됩니다. 지금 가입하면 베타 기간 전 기능을 무료로 이용합니다."

#### 레이아웃

- 모바일: 세로 2카드 누적
- `md:` 이상: 2열 병렬. Plus 카드는 `--brand` 상단 스트라이프 또는 "추천" 뱃지로 강조
- 카드 배경: `--surface`, 라디우스 `--radius-lg` (16px), 테두리 `--line`

### 섹션 6 — 신뢰·정직성 고지

**목표**: 인파가 "AI가 모든 걸 판단한다"는 오해를 바로잡고, 설계사가 책임 주체임을 분명히 한다.
이 섹션은 마케팅 텍스트가 아니라 **서비스 원칙 선언**이다.

#### 고지 내용 (확정 문안)

아래 두 항목을 카드 또는 인라인 블록으로 나란히 표시한다.

---

**항목 1 — AI 초안, 최종책임은 설계사**

```
인파가 생성한 비교안내서·메시지는 AI 초안입니다.
보장 판단과 최종 전달의 책임은 담당 설계사님에게 있습니다.
```

아이콘: 방패 또는 사람 아이콘 (색: `--success` 녹색)

---

**항목 2 — "심의완료" "안전" 배지 없음**

```
인파는 특정 보험 상품이 심의를 완료했다거나 안전하다는 배지를 표시하지 않습니다.
모든 보험 선택은 설계사님과 고객의 판단입니다.
```

아이콘: 금지/체크 아이콘 (색: `--ink-2` 중립 회색)

---

> **구현 레드라인** (`dev/14` 직결): 이 섹션의 문안은 임의로 수정하지 않는다. 마케팅 어조로 완화하거나 "인파가 보장을 분석합니다" 식으로 행위 주체를 AI로 단정하는 표현으로 대체 금지. 변경 필요 시 PM + 컴플라이언스 검토 후 `dev/14` 정본 갱신을 선행한다.

#### 레이아웃

- 배경: `--accent-tint` 틴트 또는 `--surface-2`
- 모바일: 세로 2 누적
- `md:` 이상: 2열 그리드
- 섹션 타이틀: "인파가 지키는 원칙" 또는 "정직한 AI 도구"

### 섹션 7 — 푸터

**목표**: 법무 링크 제공, 사업자 신뢰 확보. 모든 항목이 공개 접근 가능해야 한다.

#### 구성 항목

| 항목 | 처리 |
|---|---|
| 서비스 이름 + 로고 (작은 사이즈) | `design/logo/*.svg` |
| 이용약관 | `/legal/terms` (공개, dev/16) |
| 개인정보처리방침 | `/legal/privacy` (공개, dev/16) |
| 1:1 문의 | `/inquiry/new` — 비로그인 방문자는 링크 클릭 후 `/login`으로 리다이렉트 처리 |
| 사업자정보 | **[미확정 — placeholder]** 상호명 / 사업자등록번호 / 대표자 / 주소 / 이메일. 정식 출시 전 기입. |
| 카피라이트 | `© 2026 Inpa. All rights reserved.` |
| SNS 링크 | 미확정 — placeholder (추후 추가) |

> **placeholder 처리 원칙**: 미확정 항목은 "준비 중" 텍스트 또는 회색 처리(`--muted`)로 표시한다. 빈 칸으로 두지 않는다. 정식 출시 전 실제 정보로 교체 필수.

#### 레이아웃

- 배경: `--brand-ink` (딥네이비) / 텍스트: `--surface`(흰색) + `--ink-3`(회색)
- 모바일: 세로 누적 (로고 → 링크 목록 → 사업자정보 → 카피라이트)
- `md:` 이상: 좌(로고+설명) + 우(링크 4개) 2열, 하단에 사업자정보 + 카피라이트 풀폭

## 4. 디자인 토큰 사용 계약

`design/tokens/inpa-tokens.css`의 변수를 랜딩 페이지에서 사용하는 방식을 못박는다. Tailwind `tailwind.config`에 아래 매핑이 완료된 것을 전제로 한다.

| 용도 | 토큰 변수 | Tailwind 클래스 예시 |
|---|---|---|
| 1차 CTA 버튼 배경 | `--brand` | `bg-[--brand]` |
| Ghost 버튼 테두리·텍스트 | `--brand` | `border-[--brand] text-[--brand]` |
| 히어로 배경 틴트 | `--accent-tint` | `bg-[--accent-tint]` |
| 섹션 배경(교대) | `--surface` / `--surface-2` | `bg-white` / `bg-[--surface-2]` |
| 카드 라디우스 | `--radius` / `--radius-lg` | `rounded-[--radius]` |
| 강조 숫자·아이콘 | `--accent-blue` | `text-[--accent-blue]` |
| 푸터 배경 | `--brand-ink` | `bg-[--brand-ink]` |
| 경계선 | `--line` | `border-[--line]` |
| 보조 텍스트 | `--ink-2`, `--ink-3` | `text-[--ink-2]` |
| 신뢰 아이콘 (항목1) | `--success` | `text-[--success]` |

> **블루 3종 역할 교차 금지**: `--brand`(CTA), `--accent-blue`(강조/숫자), `--proposal`(데이터 전용)을 한 요소에 혼용하지 않는다. 토큰 SSOT 경고 그대로 적용.

## 5. 반응형 계약 (dev/18 연동)

| 섹션 | 모바일 (xs~sm) | 태블릿 (md) | 데스크톱 (lg~) |
|---|---|---|---|
| 1 히어로 | 카피 위 / 이미지 아래 / 버튼 full-width 세로 | 좌(카피+버튼) / 우(이미지) 2열 | 좌우 2열, max-w 제한 |
| 2 기능 3축 | 카드 1열 누적 | 3열 그리드 | 3열 (카드 폭 제한) |
| 3 차별점 | 2개 세로 누적 | 2열 그리드 | 2열 (컨텐츠 max-w 제한) |
| 4 3스텝 | 1열 누적, 스텝 번호 배지 | 3열 그리드 | 3열 + 연결선 |
| 5 요금 | 2카드 세로 누적 | 2열 병렬 | 2열 (Plus 강조) |
| 6 신뢰·고지 | 2항목 세로 누적 | 2열 그리드 | 2열 |
| 7 푸터 | 세로 단일 컬럼 | 좌(로고)/우(링크) 2열 | 2열 + 하단 풀폭 |

- 전체 섹션 가로 패딩: 모바일 `px-4` (16px) / `sm:px-6` / `md:px-8` / `lg:px-16`
- 섹션 세로 간격: `py-16` (모바일) / `md:py-24`
- 전체 최대 폭: `max-w-6xl mx-auto`
- 터치 타깃 최솟값: 44px × 44px (`dev/18` 기준 준수)
