# 인파(Inpa) 빌드 계획 — Phase · 스프린트 · 게이트

> **문서 목적**: 인파(Inpa)를 "코드 0줄 게이트(Phase 0) → 발굴 wedge MVP(Phase 1) → 리텐션·전환(Phase 1.5) → 성장 루프(Phase 2) → 브랜딩(Phase 3+)" 순으로 어떻게 짓는지의 실행 지도.
> **읽는 사람**: 투자자(언제 무엇이 검증되나) · 디자이너(어느 화면이 어느 스프린트에 떨어지나) · 개발자(무엇을 어떤 순서로, 무엇으로 검증하나).
> **전제**: foliio 코드 90% 재활용, 신규 의존성 0개, 별도 repo `~/Desktop/inpa`. 본 문서는 `dev/01~03`(아키텍처·데이터모델·포팅지도)을 빌드 일정으로 환산한 정본이다.
> **읽기 전 한 줄**: 인파의 출시 가부는 코드가 아니라 **컴플라이언스 게이트(병력 국외이전 동의 · §97 비교안내 정확요건 · 기준선 출처)** 가 쥔다. 그래서 Phase 0가 critical path다.

## 1. Phase 0 — 코드 0줄 게이트 (critical path)

> **원칙: 코드보다 법무가 먼저다.** 인파는 병력(민감정보)을 미국 Claude API로 보내고, 갈아타기(§97)를 자동 제안한다. 이 둘은 "설계가 끝나서 막힌" 게 아니라 "법무 회신이 안 와서 못 여는" 게이트다. Phase 0는 코드를 한 줄도 안 쓰고 **출시 가부 자체를 확정**하는 구간이다.

## 2. Phase 0 게이트 — 코어 담보 기준선 · 국외이전 동의 · §97 (심층)

> 이 셋이 Phase 0의 핵심 "폭탄 3종"이라 별도 절로 깊게 다룬다. 각각 **빌드에 미치는 물리적 영향**과 **막혔을 때 어떻게 우회하는지**를 명시한다.

### 2.1 코어 담보 기준선 (G1) — 히트맵 3색의 근거

**문제**: 히트맵 🟢충분/🟡부족/🔴없음 중 🟡🔴 판정은 "표준 보장 기준선"이 있어야 한다. 자체 컨센서스로 "부족"을 단정하면 부당권유 리스크.

**빌드 영향**:
- 임계값 코드는 **미리 짠다** — `status = none if actual==0 / short if actual < std_baseline*0.7 / enough otherwise` (`customers/heatmap.py` 신규).
- 단 `std_baseline` **데이터 출처**가 확정 전이면 코드는 `neutral(회색)` 폴백으로만 동작.

```
기준선 출처 확정?
  YES → 3색 풀 가동 (🟢🟡🔴)
  NO  → 중립 모드: none(0원 보유여부)만 🔴/⚪, enough/short 보류
        UI는 출처주석 자리(ⓘ)만 미리 확보
```

> **핵심**: M3 히트맵은 기준선 없이도 **"보유 0원" 스캐너**로 선출시 가능. 발굴 wedge의 80%는 "이 담보 아예 없네요"에서 나오므로 중립 모드만으로도 미팅이 잡힌다.

### 2.2 국외이전 동의 (G2) — 가장 단단한 폭탄

**문제**: 병력이 미국 Claude API로 나간다. 동의 없이는 `detect`(OCR 파싱) **자체를 못 연다**. 전 AI 기능의 물리적 게이트.

**빌드 영향(낙관 가정 금지)**:
- `Customer.consent_overseas_at` **별도 필드**(null=미동의)를 기본 전제로 설계.
- `ConsentLog(customer, type=overseas, agreed_at, ip, doc_version)` 감사추적 모델 신규.
- `detect` 호출 **전** `consent_overseas_at` 확인 → 미동의 시 **412** + FE 동의 모달.

```
[detect 호출] ──> consent_overseas_at 존재?
                    NO  → 412 게이트 + 동의 모달 (AI 차단)
                    YES → claude_parse 진행 + ConsentLog 적재
```

> **fallback**: G2가 끝까지 안 풀리면 → AI 미동작. 이때 OCR도 막히므로, 최악엔 **수기 입력 + 히트맵(보유여부 none 중립)** 만으로 선출시(분석 가치의 잔여분이라도 확보).

### 2.3 §97 비교안내 정확요건 (G3) — 합법 방패 = 발행 하드블록

**문제**: 갈아타기 비교안내서가 wedge인데, 부당승환(§97) 필수항목 누락 시 위법. AI 추출 누락분의 책임 경계가 모호하면 발행 자체가 리스크.

**빌드 영향**:
- §97 필수 6항목 체크리스트를 비교안내서 데이터 필드로 강제. **미완료 시 발행 하드블록**(다운로드/공유 버튼 비활성).
- 불리점 자동경고(해지손실·면책리셋·예정이율·갱신전환)를 자동 노출.
- 면책 고지 **상시 노출** + "AI 1차 보조, 최종책임 설계사".

```
[비교안내서 발행] ──> §97 필수 6항목 모두 충족?
                       NO  → 발행 차단(하드블록) + 누락 항목 하이라이트
                       YES → PNG/공유 발행 허용 + 면책 고지 강제 노출
```

> **가드레일 KPI**: §97 필수항목 누락률 = **0%** (악화 방지 지표). 1건이라도 누락 발행되면 회귀로 간주.

## 3. Phase 1 — MVP "발굴 wedge" (6주, 의존성 순서)

> **빌드 의존성의 핵심**: 정규화 OCR이 **모든 분석의 입력**이다. 정규화가 없으면 보험사별 담보명이 제각각이라 히트맵·비교안내서가 거짓이 된다. 그래서 빌드 순서는 강제된다:
>
> ```
> [M1 다건 OCR + M2 정규화] ──입력──▶ [M3 히트맵] ──공백──▶ [M4 비교안내서] ──문구──▶ [M6 메시지]
>        (선행, 모든 분석의 뿌리)
> ```

### 3.3 Phase 1 게이트 (정성)

```
[ ] 베타 설계사 10명 모집 + 온보딩
[ ] "비교안내서 → 카톡 발송"을 베타 설계사가 *실제 미팅*에 사용했는가 — 정성 인터뷰 5건
[ ] 5분 핵심 루프 도달 (증권 업로드 → 카톡 문구 복사)
[ ] 가드레일 회귀: §97 필수항목 누락률 = 0% / OCR 7필드 추출률 ≥ 85%
────────────────────────────────────────────
→ "실미팅 사용 5건"이 통과 조건. 분석 건수·가입자수가 아니라 *발굴 행동*을 본다.
```

> **투자자 노트**: Phase 1 게이트는 의도적으로 **허영지표(MAU·가입자수)를 배제**한다. "10명이 가입했다"가 아니라 "5명이 실제 고객 미팅에서 인파로 만든 비교안내서를 카톡으로 보냈다"가 검증선. 곱셈형 북극성(발송수×열람률×소개귀속)의 첫 항을 증명하는 단계.

## 7. Fallback 경로 — 게이트 막혔을 때

> **합의된 원칙: "막히면 중립기능부터."** 컴플라이언스 게이트가 안 풀려도 제품을 0으로 만들지 않는다. AI 게이트(동의·§97)에 종속되지 않는 기능부터 선출시한다.

```
컴플라이언스 게이트 막힘
        │
        ▼
[1차] M1 다건 OCR          ── 증권 디지털화(입력 자동화)는 동의만 있으면 가능
        │
        ▼
[2차] M3 히트맵 none 중립   ── 기준선 출처 미확정이어도 "보유 0원" 스캐너는 가능
        │                     (enough/short는 보류, 회색 중립)
        ▼
[3차] M2 정규화 사전        ── 데이터 복리는 백엔드라 출시와 무관하게 축적
        │
        ▼
[보류] M4 비교안내서 · M6 메시지  ── §97 정확요건(G3) + 국외이전 동의(G2) 확정 후 오픈
```

| 막힌 게이트 | 즉시 출시 가능 | 봉인 | 오픈 조건 |
|---|---|---|---|
| 기준선 출처(G1) | OCR + 히트맵 none 중립 + 정규화 | 히트맵 enough/short 판정 | G1 확정 |
| 국외이전 동의(G2) | (동의 동선 자체가 게이트) | detect = 전 AI | G2 동의서 개정안 |
| §97 정확요건(G3) | OCR + 히트맵 + 정규화 | M4 비교안내서 발행 | G3 6항목 확정 |

> **빌드 순서가 fallback과 일치하도록 설계됨**: M1→M2→M3가 앞에 오는 이유는 단지 의존성 때문만이 아니라, **법무가 막혀도 이 셋은 선출시 가능**하기 때문이다. M4·M6를 뒤에 둔 건 게이트 종속성을 흡수하기 위한 의도적 배치.
