# 인파 컴플라이언스 경계 — 중개·권유 금지 & 역할 분리

> 2026-06-19 사용자 피드백 반영. **인파는 보험을 중개·권유하지 않는다.** 이 한 줄이 고객뷰·히트맵·문구·코드 설계 전체를 규정한다. (기존 `05-ai-and-data`·`08-screen-specs` 보완)

## 1. 핵심 원칙
- 인파 = 보험설계사가 쓰는 **도구**. 라이선스 행위(모집·권유·중개)는 **라이선스 있는 설계사만**.
- 인파(비라이선스 SW)가 고객에게 "부족/충분/갈아타세요"를 **판정·권유하면 = 무자격 모집·부당권유 위반**.
- → 고객이 보는 화면은 **'사실'만**. 판단·권유는 설계사.

## 2. 고객 공유뷰(A) 규칙 — 사실만
- **보여줄 것**: 납입 현황(납입률 · 낸/남은 보험료 · 만기), 보유 담보(이름 + 보장금액).
- **금지**: 부족/충분/없음 등 **적정성 판정 라벨**, "갈아타기·리모델링·추천" 권유 문구, 비교 우열, 공포 조장.
- 판단은 설계사로 유도: *"내 보장 괜찮은지 → 담당 설계사와 확인하기"* + 면책 상시.
- (구현: `app/s/[token]` — status 판정 prop 자체가 없음. 납입 현황 + 사실 목록만.)

## 3. 히트맵·분석(설계사 도구) 규칙 — 설계사 기준
- 충족 표기(넉넉/적정/부족/없음)는 **'설계사가 정한 기준' 대비**. 인파가 자체 기준으로 단정하지 않음.
- 기준 = 설계사가 설정(연령/성별/상품군별 권장 보장 밴드). 화면에 **'기준 설정' 진입점** 제공.
- 이 화면은 **설계사 내부 분석용** — 고객에게 raw 노출 금지. 고객 공유는 사실(납입/보유)만.

## 4. UI 가드 (코드로 강제)
- [x] 고객뷰 컴포넌트에 status 판정 prop 미존재
- [x] 면책 푸터(`DisclaimerFooter`) 상시 — "판단·권유는 담당 설계사"
- [ ] '안전/심의완료/보장확정/추천' 카피를 디자인·타입에서 제외(컴파일 차단)
- [x] 자동발송 없음(클립보드/카톡 열기) · share_token noindex(BE)

## 5. 기준선 게이트(G4)와의 관계 — 리스크 하향
- 기존 'AI 진단의 표준 보장 기준선 출처' 게이트가 **인파의 부당권유 리스크**였음.
- **설계사가 기준 주체**가 되면서 → 인파는 판정 근거를 자체 보유하지 않고 **설계사 설정값**을 표시할 뿐. 인파 책임·부당권유 리스크 크게 하향.
- (단 기본 프리셋 제공 시 *"참고용 · 설계사가 조정 가능"* 디스클레이머 필요. 프리셋의 출처는 여전히 명시.)

## 6. 후속 기획 필요 (이 결정에서 파생)
- **설계사 기준 설정 기능 spec** (`dev/10` 예정): 기준 밴드 입력 UI · 저장 모델 · 히트맵 반영.
- **컴플라이언스 카피 룰**: 고객뷰 문구 화이트/블랙리스트 + 자동 검수.
- 데이터 모델에 `planner_baseline`(설계사별 기준) 테이블 추가 — 히트맵 status 판정의 단일 입력.
