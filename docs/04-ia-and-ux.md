# 인파(Inpa) — 화면 구성(IA) & UX 스펙

> 위촉직 보험설계사가 **고객 앞에서·이동 중에·자투리 시간에** 한 손으로 쓰는 모바일 우선 영업 OS.
> 모든 화면의 종착지는 **행동**(공유링크 발송·메시지 복사·갈아타기 비교안내서 발행)이다. 분석은 미끼, 발송이 KPI.
> 본 문서는 디자이너가 바로 그릴 수 있는 수준의 사이트맵·페이지별 스펙·와이어프레임·토큰을 정의한다.

**문서 전제**
- 본 IA는 foliio의 `layout`(헤더/탭/모달/토스트)과 `share-view`(공개뷰 헤더 숨김) 패턴을 **재배치**해 구성한다. 신규 라우팅 골격 외에는 컴포넌트를 새로 만들지 않는다.
- 정량 수치(전환율·한도·기준선 임계 등)는 모두 **(추정)** — 베타 90일 실측 후 토큰화한다.
- 제품명은 본 문서 전체에서 **인파(Inpa)** 로 표기한다.

## 1. 정보구조 (사이트맵)

### 1.1 하단 탭 5개 — 영업 동선에 맞춘 재배치

```
🏠 홈     /home        오늘 할 일(액션큐) · 이번 달 남은 횟수
👥 고객   /customer     목록·검색·정렬 / 상세(분석·갈아타기·공백·이력)
➕ 발굴   /prospect     갈아타기 즉석비교 · 무료진단 링크 · 상품 가이드   ← FAB 중앙 강조
📊 분석   /analysis     담보 한눈표 · 갈아타기 비교표 · 메시지 생성
🙋 내정보 /me           판촉물 · 추천코드 · 동의/발송 기록 · 설정
```

설계 의도:
- **발굴(➕)을 중앙 FAB로 격상** — 인파의 정체성(분석툴 → 발굴 OS)을 탭 구조 자체로 선언. foliio가 `+ 새 고객`을 보조 버튼으로 두던 것을 1급 동선으로 끌어올린다.
- **홈 = 액션큐**가 첫 화면 — 켜자마자 "오늘 누구에게 무엇을"이 0클릭으로 보인다(§4).
- **분석(📊)은 끝점이 아니라 경유지** — 모든 분석 화면의 하단 CTA는 항상 발송/제안 행동으로 수렴.

### 1.4 credit 게이트 매핑 (전 화면 공통 규칙)

| 차감 종류 | 발생 지점 | 베타 동작 |
|---|---|---|
| `insurance_credit` | OCR 증권 등록(건당) | `FREE_TIER_UNLIMITED=True` → `∞` |
| `ai_credit` | 분석/히트맵 / 갈아타기 비교 / 공백 / AI 메시지 | 동일 우회 |
| `download_credit` | PDF 저장 / 판촉물 다운로드 | 동일 우회 |

- 한도 소진 시 BE는 `402 {reason:'CREDIT_EXHAUSTED', limit, remaining}` → FE는 **업그레이드 모달**(foliio `UpgradeGuideModalComponent` 재활용).
- 헤더 크레딧바는 `GET /credit_status/` 폴링. **`remaining=0`이 아니라 `is_unlimited=True`/`remaining=null`을 `∞`로 렌더** (0은 무제한 센티넬, foliio §8 트랩 그대로 계승).

## 3. 페이지별 상세 스펙

> 각 페이지: **목적 · 컴포넌트 · 표시 데이터 · 액션 · 상태(빈/로딩/에러/한도) · 모바일 · AI 터치포인트**.

### 3.1 🏠 홈 `/home` — 액션큐 (핵심화면 외 최우선)

- **목적**: 켜자마자 "오늘 누구에게 무엇을" 0클릭 노출. 발굴·리텐션의 출발점.
- **컴포넌트**: 상단 크레딧바(`이번 달 남은 횟수: 분석 7 · 메시지 12`, foliio header credit indicator ♻) → 액션큐 카드 스택 → 하단 FAB `+ 새 고객`.
- **표시 데이터**: 각 행 = `고객명 · 사유태그 · D-day · [원클릭 액션]`.
- **사유 우선순위 정렬**:
  1. 🔴 만기/갱신 **D-30↓** (빨강)
  2. 🟠 **동의 미수신** (주황)
  3. 🎂 **생일 D-7**
  4. 📨 **미열람 공유링크** (발송 후 24h 미열람)
  5. 🔗 **인바운드 리드** (추천코드 신규 유입)
- **액션(사유별 분기)**:

  | 사유 | 우측 버튼 | 동작 |
  |---|---|---|
  | 만기 D-30↓ | [갈아타기 제안] | `/customer/:id/switch` |
  | 동의 미수신 | [동의 링크 복사] | 셀프동의 링크 클립보드 |
  | 생일 D-7 | [축하 메시지 복사] | message(목적=생일) |
  | 미열람 공유 | [리마인드 복사] | message(목적=리마인드) |
  | 인바운드 | [연락처 보기] | 리드 상세 |

- **상태**:
  - 빈(고객 0) = "첫 고객을 등록해보세요" 일러스트 + [증권 올리기] (콜드스타트).
  - 로딩 = 스켈레톤 3행. 에러 = [재시도].
  - 한도 = 크레딧바 `∞`(베타) 또는 업그레이드 안내 배너.
- **모바일**: 카드 1열, 사유태그 색칩, 액션버튼 풀폭. 스와이프 → [완료/스누즈].
- **AI**: 만기/생일은 룰 분류, 미열람은 이벤트 트리거. 메시지 액션은 목적 enum으로 Claude 호출(Haiku).

### 3.2 👥 고객 등록 `/customer/create` — 다건 OCR (M1)

- **목적**: 증권 사진/PDF 여러 장을 올리면 고객+보험을 자동 채움.
- **컴포넌트**: 드롭존(카메라/앨범/PDF) → 업로드 큐(썸네일+진행률) → 인식결과 빈칸채우기 폼(이름·생년월일·**성별**·연락처·직업위험) → 미인식 담보 수동추가.
- **표시 데이터**: OCR 7→15+필드 추출(`detect` / `detect_batch`), 보험사·증권번호·담보 리스트, 신뢰도 색칩.
- **액션**: [올리기] → [저장]. 여러 장이면 보험별 카드 누적.
- **상태**:
  - 업로드중 = 장당 진행률.
  - **부분실패 허용** — 한 장 실패해도 나머지 진행(foliio fallback chain). 응답 `partial_failed[]` 노출.
  - 암호화 PDF → 비번 입력 모달.
  - 인식률 낮음 → "직접 확인해주세요" 경고칩.
  - 한도 → `insurance_credit` 402 → 업그레이드 모달.
- **모바일**: 카메라 직접 촬영 우선, 멀티샷.
- **AI**: `claude_parse`(Haiku) + 보험사별 담보명 **정규화 사전**(`_add_coverage` 4→5단계). 추출 신뢰도 색칩(초록 확정/노랑 추정).

### 3.3 📊 담보 한눈표 `/customer/:id/analysis` — **핵심화면 1 (히트맵)**

→ 와이어프레임 §5.2. 상세 스펙은 아래.

- **목적**: 100+ 담보 '틀'에 고객 보장을 얹어 **충분·부족·없음**을 한눈에. 설득의 무기.
- **컴포넌트**:
  - 상단 고객요약(나이·성별·월보험료·동의상태칩)
  - **히트맵 그리드**(행=카테고리 15+, 열=세부담보 / 칸 3색: 🟢충분 🟡부족 🔴없음 / 회색=중립)
  - 칸 탭 → 하단시트(현재 보장액·권장 기준선·**출처주석 ⓘ**)
  - 하단 CTA `[갈아타기 제안] [공유링크 복사]`
- **표시 데이터**: `calculate_total_analysis` 결과 + 카테고리 트리(taxonomy 문서) + 충족률(현재/기준선).
- **액션**: 간략/상세 토글(foliio `summary-report` ♻), 히트맵↔표 전환, 공유링크 복사(카톡 자동발송 불가).
- **상태**:
  - **동의 미수신 = 히트맵 블러 + 잠금 오버레이 + [동의 받기]** (원칙 4).
  - 분석중 = 프로그레스. 빈(보험 0) = 등록 유도. 한도 → `ai_credit` 게이트.
- **기준선 중립 모드 (법무 게이트 종속)**: 표준 보장 기준선 출처 확정 전까지 **enough/short 판정 보류**, `none`(0원 보유여부)만 **회색 중립**으로 표기. 임계 코드(`std_baseline*0.7`)는 미리 작성하되 UI는 출처주석 자리(ⓘ)만 확보 → 출처 확정 시 3색 활성화.
- **모바일**: 가로 히트맵 = **가로스크롤 + 왼쪽 카테고리 고정열(sticky)**. 색은 **명도차 + 패턴 병행(색각 배려)**.
- **AI**: 공백 자동 탐지·기준선 비교, 내러티브 한 줄("암 진단비가 권장 대비 절반입니다").

### 3.4 🎯 갈아타기 비교안내서 `/customer/:id/switch`, `/prospect/switch` — **핵심화면 2 (해자 / §97 방패)**

→ 와이어프레임 §5.3.

- **목적**: 기존 vs 제안을 보험업법 §97 요건에 맞춰 **비교안내확인서** 자동생성. 부당승환 처벌 공포를 합법 방패로 전환.
- **컴포넌트**:
  - 좌(기존) · 우(제안, **인디고 `#3B5BDB`**) 나란히 비교표(담보·보험료·차액)
  - **불리점 자동경고 배너**: 해지환급 손실 · 면책기간 리셋 · 예정이율 차이 · 갱신형 전환
  - 고객 확인 체크리스트(필수)
  - 하단 면책고지 상시노출("본 자료는 1차 보조자료이며 최종책임은 설계사에게 있습니다")
- **표시 데이터**: 두 portfolio 비교, 담보별 차액, 손익 항목.
- **액션**: [비교안내서 발행] → 공유링크 `/s/:token`, [PDF 저장](`download_credit`).
- **상태**:
  - **§97 필수 불리고지 체크 미완료 → 발행 하드블록**(가드레일). 누락률 = 0% 가드.
  - 한도 → `ai_credit`.
- **모바일**: 좌우비교는 **세로 누적 + 담보별 펼침**. 차액은 빨강(불리)/초록(유리).
- **AI**: 불리점 후보 자동 추출(`ai_guardrail` 룰셋이 최종 검증), 안내 문구 생성(Opus — 정확도 critical). **'안전배지' 절대 표기 금지**.

### 3.5 💬 AI 메시지 `/customer/:id/message` — **핵심화면 3 보조**

- **목적**: 목적별 카톡 문구 생성(M6).
- **컴포넌트**: 목적 enum 칩(니즈환기/만기안내/생일축하/공백제안/소개요청/리마인드) → 톤 슬라이더(정중·친근) → 생성결과 미리보기 → `[복사]`.
- **표시 데이터**: 선택 목적 + 고객 컨텍스트(이름·만기·공백 요약).
- **액션**: [생성] → [복사](클립보드). **원탭 자동발송 불가 — 정직하게 클립보드**. 복사 이벤트는 발송 프록시 보조지표로만 계측.
- **상태**: 생성중 스켈레톤, [재생성], 한도(`ai_credit`). 가드레일 위반어 자동 치환.
- **AI**: 목적 enum → 프롬프트 매핑(Haiku), `ai_guardrail` 보험업법 룰셋 후처리(단정표현·수익보장·비교과장 플래그).

### 3.6 🙋 동의 동선 `/customer/:id/agree` — 컴플라이언스 게이트 (M7)

- **목적**: 병력 등 민감정보 Claude(미국, Anthropic Inc.) **국외이전 동의**. 全 AI 기능 잠금해제.
- **컴포넌트**: 쉬운말 설명(무엇을·왜·어디로) → 항목별 체크(민감정보 처리 / 국외이전 / 제3자) → 동의 로그 타임스탬프 저장 → 셀프동의 링크(고객 직접).
- **표시 데이터**: 동의 항목 · doc_version · 동의/회수 상태.
- **액션**: [동의 저장] → `consent_overseas_at` 기록 + `ConsentLog`(누가·언제·범위·doc_version·ip).
- **상태**: 미동의 = 상위 AI 화면 전부 잠금(블러+자물쇠). 동의완료 = 초록 배지. 만료/회수 동선 별도.
- **AI**: 없음(순수 게이트). detect 호출 전 `consent_overseas_at` 확인 → 미동의 시 BE 412.

### 3.7 고객뷰 `/s/:token` (진단·비교안내서 공개뷰)

- **목적**: 설계사가 발송한 진단결과/비교안내서를 고객이 열람. **북극성 = `share_view` 이벤트(서버 측정)**.
- **컴포넌트**: **글로벌 헤더·탭 숨김**(foliio share-view ♻) → 설계사 퍼스널 브랜딩 헤더(이름·연락처·사진) → 결과 본문(히트맵 또는 비교안내서) → 면책고지.
- **상태**: token 만료/회수 → 안내 페이지. robots **noindex**.
- **계측**: 진입 시 `share_view` 이벤트 + `?ref=<설계사코드>` 귀속 파라미터 기록(Day1 계측, 루프 활성화는 Phase2).

### 3.8 고객뷰 `/check/:token` (셀프진단·인바운드 발굴)

- **목적**: 무작위 발굴 금지 → **고객이 자발적으로 입력**(동의기반 인바운드). 추천코드 귀속.
- **컴포넌트**: 설계사 브랜딩 헤더 → 간단 입력(나이·성별·관심담보·현재가입 여부) → **동의 1탭** → "설계사가 분석해드려요" 리드캡처.
- **상태**: token 만료/회수 → 안내. robots noindex.
- **계측**: `selfdiag_start` → 제출 시 `referral_attributed`. (법무 Q4: 제3자 동의 1탭 충분성 선결 — 미확정 시 본 화면 Phase2 보류.)

### 3.9 브랜딩 노출 규칙 (고객뷰 공통)

> [미해소 쟁점] 바이럴 귀속(인파 노출 필요) vs 설계사 신뢰(본인 브랜딩 우선) 상충 → 퍼스널 브랜딩 필러와 비율 확정 대기.
