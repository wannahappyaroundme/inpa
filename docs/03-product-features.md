# 03. 제품 기능 카탈로그 (Product Features Catalog)

> 인파(Inpa) — 위촉직 보험설계사를 위한 AI 영업 파트너
> 본 문서는 영업 퍼널 8단계에 기능을 매핑하고, MVP(T0) → Phase 1.5(T1) → Phase 2(T2) → Phase 3(T3)의 출시 우선순위를 정의한다.
> 정량 수치는 베타 90일 실측 전 가설이며 **(추정)** 으로 표기한다. 제품명은 항상 **인파(Inpa)**.

## 0. 한눈에 보기 — 제품 철학

인파의 기능 설계는 단 하나의 원칙 위에 선다.

```
  분석은 풀고(미끼),  행동은 막는다(과금).
  ───────────────    ──────────────────
  OCR · 히트맵 ·       발송 · 복수생성 · AI메시지 ·
  비교안내서 1건 생성    워치독 · 귀속
  = Basic 무료          = Plus+ 유료
```

- **분석 = 무료 미끼**: 설계사를 끌어들이는 후크. 증권을 넣으면 5분 안에 가치를 본다.
- **갈아타기 비교안내서 = 발굴 wedge**: 부당승환(보험업법 §97)이라는 *처벌 공포*를 *합법 방패* 기능으로 뒤집는다. 범용 AI·캔바가 절대 못 주는 단 하나의 칸.
- **리텐션 = 복리**: 만기·갱신·생일을 자동으로 챙겨 방치 고객을 소개·추가계약으로 전환.

> **가장 무서운 단일 폭탄**: 병력(민감정보) 국외이전 동의 게이트. 이 게이트가 막히면 detect API(증권 OCR) 자체를 못 연다 → 전 AI 기능이 묶인다. 그래서 MVP에 `M7 국외이전 동의`가 포함되고, 막힐 경우 중립 기능(OCR·히트맵)부터 선출시하는 fallback 경로를 설계에 박아 둔다(§6).

### 2.1 MVP 핵심 루프 — 5분 안에 가치 도달

```
 [증권 사진/PDF N장]
        │  ① 일괄 OCR (M1, Haiku)
        ▼
 [원시 담보명 추출]
        │  ② 정규화 (M2, 보험사별 이름 → 표준 담보)
        ▼
 [표준 담보 매핑]
        │  ③ 공백 스캐너 (M3, 보장 0원 = 🔴 / 보유 = 🟢)
        ▼
 [담보 히트맵 한눈표]
        │  ④ 갈아타기 비교안내서 (M4, Opus + §97 6항목)
        ▼
 [기존 vs 제안 비교표 + 불리점 자동경고]
        │  ⑤ AI 카톡 문구 (M6) → [클립보드 복사]
        ▼
 [고객에게 카톡 발송 → 미팅 성사]
```

> 이 한 흐름이 인파의 정체성이다. foliio가 "본인 화면에만 머무는 분석"이었다면, 인파는 **분석을 영업 행동의 방아쇠**로 재배치한다.

### 2.2 MVP 의도적 제외 — 왜 빼는가

| 제외 기능 | 이동 Phase | 제외 이유 |
|---|---|---|
| 바이럴 루프 (셀프진단 인바운드) | Phase 2 (T2) | 발굴 wedge 검증 전에 폭을 넓히면 핵심 루프가 흐려진다 |
| 리퍼럴 귀속 캠페인 | Phase 2 (T2) | 단, **귀속 계측(`?ref=` + 이벤트 스펙)은 Day1에 깐다** — 사후 복원 불가 |
| 알림톡 (카톡 발송 자동화) | Phase 3 (T3) | 사전심의 필요 + 발송 인프라 가정 금지 |
| 만기 워치독 | Phase 1.5 (T1) | 리텐션 레버지만 *재방문 동기*가 약해 핵심 루프에서 분리 |
| 퍼스널 브랜딩 | Phase 3 (T3) | 발굴 검증 후 성장 단계 자산 |

> **계측은 1일차, 캠페인은 2단계** (충돌4 합의). `share_token`에 `?ref=<설계사코드>` 파라미터와 이벤트 계측은 첫 배포 전 확정하되, 바이럴 *루프 활성화*(더블사이드 보상)는 Phase 2로 미룬다.

## 3. 컴플라이언스 게이트 막힘 시 — Fallback 출시 순서

법무 선결(Phase 0)이 막히는 경우를 대비해, **중립 기능부터 선출시**하는 경로를 설계에 박아 둔다.

```
  게이트 통과?
  ┌─────────────────────────────────────────────┐
  │ NO (동의서 개정 미완 / §97 요건 미확정)        │
  └─────────────────────────────────────────────┘
        │
        ▼  중립 기능부터 (AI 부당권유 리스크 0)
   ① M1 OCR (증권 텍스트 추출 — 중립)
   ② M3 히트맵 (none=보유여부만, 회색 중립 모드)
   ③ M2 정규화 (백엔드, 분석 정확도 향상)
        │
        ▼  게이트 통과 후 개방
   ④ M4 갈아타기 비교안내서  ← §97 6항목 확정 필요
   ⑤ M6 AI 메시지            ← 국외이전 동의 확정 필요
```

- **기준선 중립 모드**: 표준 보장 기준선의 출처·권위(금감원/보험연구원/자체+면책)가 확정되기 전까지 히트맵은 `enough/short`(부족/충분) 판정을 **보류**하고, `none`(0원 보유 여부)만 회색 중립으로 표기한다. 임계값 코드(`std_baseline * 0.7`)는 미리 짜두되 데이터 권위는 Phase 0 법무 산출물로. UI에는 출처주석 자리(ⓘ)만 확보.
- **국외이전 동의 412 게이트**: 병력(민감정보)이 Claude API(미국, Anthropic Inc.)로 나가므로 `detect` 호출 *전* `consent_overseas_at`를 확인 → 미동의 시 **412** 반환 + FE 동의 모달. `ConsentLog`(누가·언제·범위·doc_version·ip)로 감사 추적.

## 4. 우선순위 Tier — T0 ~ T3

| Tier | 단계 | 기능 | 마일스톤 |
|---|---|---|---|
| **T0** | MVP (6주) | M0~M8 (위 §2) — 없으면 제품이 성립 안 함 | 베타 설계사가 비교안내서→카톡발송을 실제 미팅에 사용 (정성 인터뷰 5건) |
| **T1** | Phase 1.5 (4주) | 만기 워치독 · AI 갈아타기 내러티브 · 원클릭 니즈환기 카드 · **Plus 과금 오픈** | 첫 유료 전환 |
| **T2** | Phase 2 (6주) | 바이럴 진단 루프 · 추천코드 귀속 · 셀프진단 공유링크 | 소개 귀속 리드 발생 |
| **T3** | Phase 3+ | 퍼스널 브랜딩 · 알림톡(심의 후) · 팀/라이브 모드(Super) | — |

> **빌드 의존성 순서** (충돌 합의): `정규화 OCR(선행, 모든 분석의 입력)` → `히트맵` → `비교안내서` → `메시지`. 정규화 사전이 두꺼워질수록 OCR 정확도가 오르고, 그것이 분석·비교의 신뢰도를 결정한다 — 데이터 복리의 물리적 토대.

## 5. 해자 3종 vs 가드레일 — 명확한 구분

기능 카탈로그에서 **모방 어려운 해자**와 **위생(필수지만 차별점 아님)** 을 혼동하면 투자 서사가 흐려진다.

| 구분 | 항목 | 왜 |
|---|---|---|
| **해자 ①** | 갈아타기 자동 비교안내서 (§97 방패) | 범용 AI는 §97 6항목 합법 요건을 모름. 규제를 기능으로 뒤집은 단 하나의 칸 |
| **해자 ②** | 보험사별 담보명 정규화 사전 | `hit_count` 기반 데이터 복리 플라이휠 — 쓸수록 두꺼워지는 자산 (아래 도식) |
| **해자 ③** | 끊김없는 파이프라인 | 증권→진단→갈아타기→멘트→링크. 한 흐름 독점 |
| 위생 (해자 아님) | 3중 가드레일 (§97·광고심의·PII) | 전 티어 공통이어야 §97 방패가 신뢰받음 — 강제 과금 금지 |

```
  데이터 복리 플라이휠 (해자 ②)
  ┌──────────────────────────────────────────────┐
  │ OCR 미매칭 raw_name → unmatched_log            │
  │       ↓                                        │
  │ admin 1탭 매핑 → 사전 영구 추가                 │
  │       ↓                                        │
  │ 다음부터 자동 매칭 (hit_count++)               │
  │       ↓                                        │
  │ 사전이 두꺼워짐 → OCR 정확도↑ → 더 많은 사용   │
  └──────────────────────────────────────────────┘
              ↑___________________________________│
```

> **운영 미결(openQuestion)**: `hit_count` 5회+ 시 admin 검수 없이 자동승격 허용 여부는 미정. 자동매핑이 틀리면 비교안내서가 거짓이 되어 §97 위반 → 정확도 vs 운영비용 트레이드오프, 베타 후 결정.

```
  North Star = (미팅당 공유링크 발송수) × (열람률) × (소개 귀속 리드수)
               ─────────────────────────   ────────   ──────────────────
               share_token 발송 이벤트       서버 측정    referral_attributed
```

- **신뢰 KPI = `share_token` 열람 이벤트** (서버 측정 가능).
- **클립보드 복사 = 발송 프록시 보조지표** — 원탭 자동발송이 카카오상 불가하므로 복사를 *발송 의도*로만 본다 (충돌6 합의). 복사→실발송 전환율 가정은 베타 검증 대상.
- 가입자수·MAU·**분석 건수는 허영지표로 강등** — 지표 변경이 곧 정체성 변경(분석툴→영업OS)의 증거.
