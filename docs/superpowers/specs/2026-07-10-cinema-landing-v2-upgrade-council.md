# 시네마 랜딩 v2 업그레이드 — 페르소나 협의체 기록 (2026-07-10)

> 목적: new.inpa.kr 시네마 랜딩을 "지금보다 훨씬 더 고퀄리티"로 (PM 지시).
> 진행 방식: 연출가 페르소나 = 독립 서브에이전트 완주(코드 6파일 + 시안 PDF 실검토). 브랜드·UX·프론트기술·사운드·카피·그로스·대표 7개 렌즈 = 세션 한도로 오케스트레이터가 직접 수행(코드 전량 파악 상태). 필요 시 리셋 후 독립 재검증 가능.
> 구현 원칙: 라이브(new.inpa.kr) 불변, 업그레이드판은 **new.inpa.kr/version** 에 비교용으로 배포(proxy 예외 추가) → PM 확인 후 본편 교체.

## 핵심 진단 (전 렌즈 공통 결론)

콘티(장면 구성·카피·타자기 컨셉)는 강하다. 부족한 것은 **장면 사이의 시간 설계(전환 문법)와 마지막 카타르시스**. 현재 구현은 "슬라이드쇼 + 타자기"이고, 영화가 되려면 컷/디졸브/리듬/빛의 층위가 필요하다.

확인된 사실(코드 기준):
1. 스펙의 "불 켜짐" 엔딩이 실제로는 하드컷(`setMode("landing")` 한 줄, 파랑→흰색 1프레임).
2. 전환 문법 부재: 배경색 700ms 트랜지션이 실제 작동하는 곳은 S5→S6 한 곳뿐. 배경 이미지 mount/unmount 즉시 팝(S1→S2 설계 안 된 흰색 플래시). 텍스트 퇴장 애니메이션 0. 스펙 S5 "사진 서서히 페이드인" 미구현.
3. 카메라 무브 0(모든 장면 정지 정중앙 샷), 파랑 장면 신호등 점 애니메이션 누락(스펙 약속 미이행), 데스크톱에서 글자가 타이틀이 아니라 자막 크기.
4. `font-mono`가 한글에서 폴백되어 영문과 베이스라인·폭 불일치(타자기 정밀감 훼손). `line-rise`는 max-height 애니메이션이라 리플로우 유발.
5. 배경 이미지 1672px 원본이 대형/레티나 화면에서 업스케일되어 물러짐.
6. 계측 0: 장면별 이탈·완주율·게이트 선택을 측정할 수단 없음. CTA가 www로 넘어갈 때 UTM 유실(도메인 교차).
7. 재방문자도 매번 게이트부터(브랜드 경험 강제 반복).

## 제안 (티어별)

### 티어 1 — 체감 7할 (강추)
- **A. "불 켜짐" 엔딩 시퀀스** (난이도 중): `closing` 중간 상태 신설. 텍스트·워터마크 300ms 페이드아웃 → 흰 원이 화면을 밀어내는 900ms(transform scale, GPU 합성) → 밑에 미리 깔린 랜딩 노출. 종막 벨 2음(1318→1568Hz). reduced-motion은 즉시 전환 유지. 피크엔드 법칙상 전체 기억을 결정하는 1초.
- **B. 배경 레이어 상시 마운트 + 크로스 디졸브 통일** (중): 배경(검정/일러스트/군중/파랑)을 레이어 스택으로 전부 렌더하고 opacity만 700ms 전환. 텍스트는 200ms 페이드아웃 후 다음 비트. S1→S2 플래시가 "어둠이 걷히는" 의도된 장면이 됨.
- **E. 파랑 장면 신호등 점 소생** (소): 점 낙하(translateY -12→0, 400ms) + `.inpa-ping` 파동 1~2회 + 저음 타건 동기화. 게이트 빨간 점→진행 도트→클라이맥스 점→페르소나 신호등으로 모티프 회수.
- **F. 타이포 타이틀 승격** (소~중): 본문 `text-[26px] sm:text-[44px] lg:text-[54px]`. S1 완성 후 "人波"만 밝기 유지, 나머지 60%로 가라앉힘. 한글은 mono 폴백 대신 Pretendard + tracking, 영문만 mono. **(브랜드 렌즈 추가) 검정 장면 한정 명조(세리프) 서브셋 도입 검토**: 사전 정의 장면의 활자 무게가 크게 오름(Noto Serif KR 서브셋 프리로드, 시네마 전용이라 서비스 페이지 영향 0).

### 티어 2 — 완성도
- **C. 소등 의식 + 리듬 차등** (소): 게이트 400ms 페이드아웃 → 완전 암전 800ms 유지 → 첫 타건. 장면별 타속(S1 85ms/자 또박또박, S3 반전 55ms/자 단호). "화면을 눌러 계속" 힌트 1.2초 지연(문장 음미 침묵).
- **D. 슬로우 푸시인 + 비네트** (소): 군중 사진 12~20초 scale 1.05→1.12(transform만, will-change), 일러스트 미세 드리프트. 군중 하단 라디얼 비네트로 텍스트 대비 + 깊이. 저해상 업스케일 물러짐도 일부 상쇄.
- **사운드 3종** (소~중): ① 타건음 레이어링(노이즈 버스트 + 톤 핑 + 짧은 바디 공명, 글자별 세기 변주)으로 "삑"이 아닌 "타닥" ② 문장 완료를 딩 대신 **딩+캐리지 리턴**(타자기 서명 사운드) ③ 종막 벨(A에 포함). 전부 WebAudio 합성 유지(파일 0), 마스터 게인 낮게.

### 티어 3 — 운영·선택
- **계측 + UTM 연결** (소, 비가시·고가치): 게이트 선택(소리/무음/건너뛰기)·장면 도달·완주·CTA 클릭 이벤트 계측(Vercel Analytics custom events). CTA 링크에 `utm_source=new_landing` 부착해 www 가입까지 귀속 추적(현재 유실). 업그레이드 전후 완주율 비교 가능해짐.
- **재방문자 게이트** (소): localStorage 시청 기록 → 재방문 시 "서비스 소개 바로 보기"를 첫 버튼으로 승격(다시 보기 유지).
- **(선택) 필름 그레인 오버레이** (소): 검정 장면 한정 미세 그레인. 필름 질감 + 저해상 마스킹. 단, 연출가는 "질감 추가 절제" 입장이라 PM 취향 판단 필요.
- **(선택) 군중 장면 웅성임** (중): 군중 사진 위 2초 저레벨 앰비언스 스웰. 침묵의 긴장을 해칠 수 있어 순수 선택지.

### 하지 말 것 (전 렌즈 합의)
- 스크롤 하이재킹·타이머 자동 진행 금지(클릭=책장 넘기는 통제감이 정체성).
- 파티클·3D·글리치·BGM 루프 금지(절제가 힘, B2B 신뢰 톤, 오글거림 방지).

- 카피 대수술 금지(현 카피 리듬 양호; 게이트 "1분"→실측 "40초" 정도만 정직 보정 검토).
