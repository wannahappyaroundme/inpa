# 인파 콘텐츠 마케팅 전략 (블로그 · 가이드 · 사례) — 제안

> 2026-07-12. PM 질문: 자체 사이트 블로그가 인스타/외부 블로그보다 나은가? 토스·쏘카·닥터나우 형태가 검색에 유리한가? 콘텐츠(가이드·사례) 전략을 달라.
> 이 문서 = 전략 제안(SSOT). 실제 블로그 **구축은 별도 Phase 0**(페르소나 카운슬 → 구현 계획 → 빌드).

## 1. 자체 블로그 vs 소셜/외부 블로그

| 채널 | 강점 | 약점 | 역할 |
|---|---|---|---|
| **자체 블로그(인파 도메인 /blog)** | SEO 자산이 **내 도메인**에 축적, 도메인 권위↑(제품 페이지도 동반 상승), 내부링크로 가입 연결, AI가 "인파=보험 전문"으로 학습, 콘텐츠·CTA·제품 통합 | 초반 트래픽 느림, 직접 운영(발행) 필요 | **SEO 본진(원본)** |
| 인스타/유튜브 | 발견·확산 빠름, 시각·숏폼 | 자산이 플랫폼 소유(내 검색순위로 안 쌓임), 링크 제약, 알고리즘 종속 | **유입 깔때기** |
| 네이버 블로그 | 네이버 검색에서 자사 블로그 우대 경향 | 자산이 네이버 것, 구글/AI 기여 낮음 | **네이버 검색 보조** |

**결론(하이브리드, one source → multi channel):** 글의 **원본은 자체 블로그**에 두고(구글·AI·도메인 자산), 인스타/유튜브엔 요약·이미지·숏폼으로 재가공해 뿌려 자체 블로그로 유도. 네이버 검색을 노리면 네이버 블로그에 요약본 병행(원본 링크 포함). **핵심 자산은 자체 블로그.**

## 2. 토스·쏘카·닥터나우 형태 해석

그들이 검색을 장악한 이유 = **예쁜 블로그 디자인이 아니라, 제품 자랑 대신 "고객이 실제로 검색하는 질문"에 실용적으로 답하는 콘텐츠를 꾸준히** 낸 것. (토스=금융을 쉽게 / 쏘카=이동·라이프 / 닥터나우=증상·약)

인파에 적용: 청중이 둘.
- **(A) 보험설계사 = 우리 고객(B2B) — 최우선.** 설계사가 검색하는 실무 질문에 답 → 인파 잠재고객이 검색으로 유입 → 가입 전환.
- (B) 일반 소비자(설계사의 고객) — 부차. 셀프진단 유입엔 좋지만 검색 경쟁이 치열하고 전환이 약함.

→ **B2B(설계사) 중심 블로그 추천.** 형태는 토스식(쉬운 말 + 실용 가이드 + 자연스러운 제품 연결)을 따르되, 주제는 설계사 실무.

## 3. 콘텐츠 4대 기둥(Pillars)

1. **영업·발굴 노하우** (새내기 설계사 = 1순위 타깃): "신입 보험설계사 고객 발굴법", "지인 영업 없이 고객 늘리기", "상담 예약률 높이는 화법", "소개 늘리는 법"
2. **보장분석 실무** (인파 기능 직결): "보험 증권 읽는 법", "3대 진단비란", "갱신형 vs 비갱신형", "100대 담보 한눈에", "보험나이 계산법"
3. **규정·안전** (설계사 불안 해소 + 인파 차별점): "부당승환(§97) 쉽게 이해하기", "비교안내서 바르게 쓰는 법", "금소법 설명의무 체크리스트"
4. **사례·인터뷰** (신뢰·전환): "증권 분석 30분→3분 후기", "인파로 첫 계약한 새내기 설계사", before/after

## 4. 글 1편 SEO 공식

- **검색어 1개**를 겨냥 → 제목에 그 검색어 포함(예: "보험 증권 분석 방법").
- **답을 맨 앞에**(AI 답변 엔진 인용·발췌에 유리 = AEO).
- 실용 가이드 본문 + **자연스러운 인파 CTA**(강매 금지, "이 과정을 인파가 자동화" 식 fit-framing, §6 상태우선 카피).
- **Article/BlogPosting + (해당 시) FAQPage 구조화 데이터** + 내부링크(관련 글 · /faq · 가입).
- OG 이미지, sitemap 자동 포함 — **이미 깐 SEO 토대 재사용.**
- 정직성 레드라인: 조작 통계·"안전/1위" 배지 금지, 인파는 중개 안 하는 분석·정리 도구.

## 5. 운영

- **주기:** 주 1편(현실적, 품질 우선). 첫 3개월 = 기둥별 3~4편, 총 12~16편 초석.
- **측정:** 서치콘솔 검색어·유입, 글→가입 전환(UTM), AI 답변 인용 여부 스팟체크.
- **재활용:** 1글 → 인스타 카드뉴스 + 숏폼 스크립트 + 네이버 요약본.

## 6. 기술 형태 (비개발자가 직접 발행 — 필수)

- 인파 도메인 **`/blog`** 하위(도메인 권위 통합). 서비스 라이트 테마, 기존 디자인 시스템.
- **PM이 코드 없이 글 발행**(비개발자 접근 원칙). 옵션:
  - (a) **어드민 글 작성 화면**(제목·본문(마크다운/에디터)·대표이미지·발행/임시저장) → DB 저장 → `/blog/[slug]` 렌더. 인파 자체 완결.
  - (b) headless CMS/Notion 연동(PM이 Notion에 쓰면 블로그로 반영).
  - → 추천 (a): 외부 종속 없음, 어드민 SSOT, 우리 스키마·OG 재사용 쉬움.
- 글마다 Article/BlogPosting 스키마 + OG + sitemap 자동. 목록/태그/관련글.

## 7. 다음 단계 (별도)

블로그 **구축은 새 기능(Phase 0)**. 진행 결정 시:
1. 페르소나 카운슬(그로스·온라인 마케터 / SEO·콘텐츠 / 보험 도메인 전문가 / 기획)로 주제 우선순위·브랜드 보이스·B2B vs B2C 비중 검증.
2. 구현 계획(어드민 작성 화면 + `/blog` 렌더 + 스키마) 작성.
3. 초석 글 12~16편 주제 확정 + 발행 캘린더.
