# Spec: /analysis 보험별 카드 우선 보기 (PM 지시 2026-07-07)

> PM: "/analysis 에서 보험이 1개면 지금처럼, 여러 개(보유든 가입제안서든)면 **카드가 먼저** 보이고, 카드를 누르면 **그 보험의 상세 보장 정보**가 나타나게."

## Decisions (locked)

1. **BE — 히트맵 보험 필터**: 기존 히트맵 엔드포인트(analysis 조회)에 선택 쿼리 파라미터 `insurance_id` 추가. 주어지면 해당 CustomerInsurance(소유 검증: customer__owner)의 case_list만 집계한 트리/summary 반환, 없으면 기존 전체 합산 그대로(하위호환 100%). 잘못된/남의 insurance_id → 404. 기존 응답 형태 불변(`insurances[]` 포함 — 카드 데이터로 재사용).
2. **FE — /analysis 화면 흐름**:
   - 보험(보유, portfolio_type=1) **0~1개**: 기존 화면 그대로(변경 0).
   - **2개 이상**: 히트맵 위 영역에 **보험 카드 그리드**(기존 InsuranceCard/보험 카드 스타일 재사용 — 보험명·종류·월보험료·담보 수) + 맨 앞에 **'전체 한눈에 보기' 카드**(합산). 기본 선택 = 전체. 카드 클릭 → 그 보험 기준으로 히트맵·보험료 분리표를 다시 조회(`insurance_id`)해 아래 상세 영역 갱신 + 선택 카드 하이라이트. '전체' 카드로 복귀 가능.
   - 가입제안서(portfolio_type=2)는 /analysis 의 기존 노출 규칙을 그대로 따르되, 카드 그리드에 있으면 동일 동작.
3. **판정 규칙 불변**: 기준선 없으면 neutral, 있으면 graded — 필터 조회에도 동일 적용(BE가 이미 처리). 카드·상세 어디에도 새 판정어 추가 금지.
4. 마이그레이션 0. 고객 상세(분석 탭)는 이번 범위 밖(이미 보험별 카드 존재; 동일 패턴 확장은 후속).

## Tests (BE)
1. insurance_id 필터: 보험 2개 고객 → 각 id로 조회 시 트리가 해당 보험 담보만, summary 도 해당 보험 수치만.
2. 소유 격리: 남의 insurance_id → 404. 없는 id → 404.
3. 파라미터 없음 = 기존 전체 합산(기존 테스트 불변).
4. 전체 스위트(554+) 그린.

## Verification gates
- BE check + makemigrations --check(0) + 전체 스위트, FE lint:copy + build 실출력.
- 보고: 필터 응답 실측 JSON(보험 2개 중 1개 선택 시 트리 분리 확인), FE 상태 흐름 설명(전체↔개별 전환).
