# 제안 vs 제안 자유 선택 UI + ④ 비교안내서 설계사 직접 발송 — Spec

> 2026-07-09. PM 확정: (1) 비교탭에서 아무 보험이나 A/B에 자유 배정(제안 vs 제안·증권 vs 증권). (2) ④ 비교안내서 = 인파가 아니라 **설계사가 중립 비교를 복사/내보내 직접 발송**(인파는 편의만, §97 회피). FE-only(BE side_a_ids/side_b_ids 이미 지원, 마이그레이션 0).

## 현재 상태 (탐사)
- `app/customer/[id]/page.tsx` SwitchTab: 선택 풀이 `held`(portfolio_type==1, 왼쪽/A)·`proposed`(==2, 오른쪽/B)로 분리(:1347-1348). curSel/propSel 선택 후 **이미 `sideAIds/sideBIds` 전송**(:1327). 하지만 풀 분리 때문에 제안 2개를 A/B에 못 넣음.
- `copyText`(@/lib/clipboard) 기존 사용(동의 링크 복사 등). 클립보드 유틸 있음.
- 비교 응답: `data.rows`(담보별 현재/제안 금액·증감), `data.current/proposed`(보험료 요약), `switch_warnings`(설계사 내부 전용), verdict 없음(③에서 제거).

## 설계 (FE only)

### 1. 자유 A/B 선택 UI
- 선택 영역을 **단일 보험 목록**(고객의 모든 보험 = held+proposed 합침)으로 바꾸고, 각 행에 **A / 미포함 / B 3-state 토글**(세그먼트) 부여. 설계사가 각 보험을 A안·B안·제외로 배정.
- 기본값(회귀 방지 UX): 마운트 시 held→A, proposed→B 프리셋(현재 동작 유지). 이후 설계사가 자유 변경.
- 배정 결과 → `sideAIds`(A로 표시된 id들)·`sideBIds`(B) → 기존 `compareCustomer(customerId, {sideAIds, sideBIds})` 그대로.
- 라벨 중립(A안/B안). '보유/제안'은 각 보험의 태그로만 보조 표시(portfolio_type 배지).
- 제안 vs 제안: 제안 2개를 각각 A·B로 → BE가 이미 처리.

### 2. ④ 비교안내서 = 설계사 직접 발송(복사/내보내기)
- 비교 결과 하단에 **'고객에게 보낼 내용 복사'** 버튼. 클릭 시 **중립 비교를 텍스트로 클립보드 복사**(설계사가 카톡·문자에 붙여넣어 직접 전송).
- 복사 내용(★ 중립 사실만):
  - 헤더: '보장 비교 (설계사 {이름} 제공)' 정도(선택).
  - 담보별: `담보명 | 현재 금액 | 제안 금액` (data.rows). 증감(추가/삭제/변경/유지)은 중립 서술이라 포함 가능.
  - 보험료 요약: 현재/제안 월·총 보험료(data.current/proposed).
  - **제외(절대 미포함):** `switch_warnings`(설계사 내부), 판정(이미 없음). 어떤 권유·판단 문구도 없음.
  - 말미 정직성 1줄(기존 통일 문구 재사용): "인파가 등록된 보장 정보를 정리한 참고 자료입니다. 보장 판단과 안내는 담당 설계사님이 합니다."
- **인파가 보내지 않음** — 복사만. 자동 발송·인파 발신 없음(§97: 플랫폼이 승환 유통 안 함).
- 이미지 내보내기는 후속(html2canvas 등 의존성 추가 회피). 이번은 텍스트 복사(기존 copyText 패턴).
- §6 카피: 긍정·평이, em-dash 금지, '부당승환' 등 법적어 미노출(고객에게 갈 텍스트이므로 특히 중립·benefit).

### ★ §97 준수
- 복사 텍스트 = 중립 사실(담보·금액)만. 판정/권유/switch_warnings 없음. 설계사가 발신 주체. 인파는 정리·복사 편의만.
- `COMPARE_PUBLISH_ENABLED`(인파 공식 발행 하드블록)과 무관 — 이건 설계사 복사이지 인파 발행이 아님. 게이트 불필요(단, 복사 내용에 판정 없음을 코드로 보장).

### 검증
- FE `npm run build` + `lint:copy`(고객에게 갈 텍스트라 권유어·em-dash 특히 주의).
- 자유 선택: 제안 2개를 A/B에 넣고 비교 요청 → 정상(수동 확인/기존 BE 테스트가 side_a/b 커버).
- 복사 텍스트에 switch_warnings 문구·판정어 미포함 확인(코드 리뷰).
- BE 무변경. 고객 대면(/s·/d·/c·/b) 무변경.
