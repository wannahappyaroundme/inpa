# new.inpa.kr/test 서비스 중심 랜딩 설계

> 확정일: 2026-07-16
> 목적: `www.inpa.kr`의 빠른 이해·가입 구조와 `new.inpa.kr`의 브랜드·실제 화면을 결합한 최종 후보를 공개 테스트 경로에서 검증한다.
> 배포 주소: `https://new.inpa.kr/test`
> 기존 화면 보호: `https://new.inpa.kr/`와 `https://www.inpa.kr/`는 이번 작업에서 변경하지 않는다.

## 1. 결정 배경

라이브 화면, 데스크톱·모바일 렌더, 실제 랜딩 자산, 사업·제품·마케팅·UX·브랜드·프론트엔드·데이터·보안·QA·법무·CS 관점을 함께 검토했다.

공통 결론은 다음과 같다.

- `www`는 첫 화면에서 가치와 가입 버튼이 바로 보여 전환에 유리하다.
- `new`는 人波와 INPA를 연결한 브랜드 기억과 실제 제품 화면이 강하다.
- 현재 `new`의 영화 관문, 반복 섹션, 작은 제품 화면은 최종 서비스 랜딩에 맞지 않는다.
- 최종 후보는 `new`의 시각 언어를 유지하되 첫 화면부터 제품을 크게 보여줘야 한다.
- 브랜드 영화는 삭제하지 않고 사용자가 선택해서 보는 콘텐츠로 이동한다.

이번 `/test`는 이 결론을 실제 화면으로 확인하는 공개 후보판이다.

## 2. 목표와 성공 기준

### 목표

1. 방문 후 5초 안에 대상, 핵심 가치, 시작 방법을 이해한다.
2. 첫 화면에서 실제 인파 대시보드 또는 보장분석 화면을 확인한다.
3. 보장분석·비교분석·고객관리·일정·성과가 한 서비스 안에 있다는 점을 화면으로 이해한다.
4. 모바일에서도 데스크톱 화면의 축소판이 아니라 읽을 수 있는 제품 증거를 제공한다.
5. 현재 운영 랜딩과 서비스에는 영향 없이 후보안을 검증한다.

### 완료 기준

- `https://new.inpa.kr/test`가 200으로 열린다.
- 첫 화면에 실제 제품 화면과 `무료로 시작하기`가 함께 보인다.
- 실제 화면 전환 탭과 확대 보기가 키보드·터치로 작동한다.
- 390px, 768px, 1440px에서 가로 넘침과 잘림이 없다.
- `new.inpa.kr/`와 `www.inpa.kr/`의 기존 화면·동작이 유지된다.
- 테스트 페이지는 검색 결과에 노출되지 않는다.
- CTA는 현재 실제 서비스인 `https://www.inpa.kr`의 가입·로그인으로 연결된다.

## 4. 정보 구조

### 4.1 헤더

- 로고
- `실제 화면`, `주요 기능`, `요금`, `자주 묻는 질문` 앵커
- `로그인`, `무료로 시작하기`
- 모바일에서는 핵심 행동을 우선하고 보조 앵커는 메뉴 안에 둔다.

### 4.2 첫 화면

- 보조 문구: `내 손안의 인슈어 파트너`
- 제목: `설계사님은 클로징만 준비하세요`
- 설명: 고객 관리부터 보장분석, 일정, 성과까지 한곳에서 이어진다는 사실을 쉬운 말로 설명한다.
- 주 행동: `무료로 시작하기`
- 보조 행동: `인파 이야기 60초 보기`
- 화면 오른쪽 또는 아래에 실제 운영 데모 대시보드를 크게 배치한다.
- 현재 대시보드의 공유 링크·로컬 주소 영역은 촬영 범위에서 제외한다.

### 4.3 빠른 사실 정보

- `100개 이상 담보를 같은 틀로`
- `증권 한 장 자동 정리`
- `설계사님이 정한 기준 적용`
- 숫자 카운트업은 쓰지 않고 최종 문구를 즉시 표시한다.

### 4.4 실제 제품 둘러보기

한 번에 하나의 실제 화면을 큰 크기로 보여준다.

1. 대시보드: 목표·고객 단계·일정·성과
2. 고객관리: DB·TA·FA·청약 단계별 관리
3. 보장분석: 담보별 보유·공백 확인
4. 비교분석: 현재 보장과 제안의 사실 비교
5. 일정·예약: 캘린더와 상담 일정

각 화면은 다음 규칙을 따른다.

- 데스크톱 표시 너비 1,000~1,100px
- 화면 안의 핵심 글자가 인식 가능한 크기
- 2~3개의 설명 표시
- 탭 선택 상태를 색상뿐 아니라 테두리·텍스트로도 구분
- 누르면 원본 확대, ESC와 닫기 버튼으로 종료
- 모바일에서는 핵심 영역을 별도 편집하고 전체 데스크톱 화면을 무리하게 축소하지 않음
- 개인정보·실제 고객 연락처·살아 있는 토큰·로컬 주소를 포함하지 않음

판촉물 화면은 재촬영 품질이 기준에 못 미치면 이번 후보판의 제품 탭에서 제외한다.

### 4.5 사용 흐름

현재의 영업 지도, 증권 처리 6단계, 사용법 3단계를 하나로 합친다.

1. 고객과 증권 등록
2. 보장 자동 정리
3. 현재와 제안 비교
4. 고객 안내와 다음 일정 관리

모바일에서도 네 단계가 한 화면씩 지나치게 길어지지 않도록 짧은 카드 또는 가로 스크롤 없이 세로 2×2 구조를 사용한다.

### 4.6 차별점

- 보험사마다 다른 담보 이름을 같은 틀로 정리
- 비교할 때 확인할 항목을 빠뜨리지 않도록 도움

인파가 보장 판단 기준을 제공한다는 표현은 쓰지 않는다. `설계사님이 정한 기준을 한눈에 보여줍니다`로 통일한다.

### 4.7 대상별 가치

- 개인 설계사
- 관리직

일반 가입자 대상 카드는 이번 후보판에서 제외해 핵심 구매자 메시지를 유지한다.

### 4.8 요금·FAQ·신뢰·마지막 행동

- 운영 설정과 일치하는 베타·요금 정보만 노출한다.
- 미확정 가격·한도는 새로 만들지 않는다.
- FAQ는 증권 등록, 고객 동의, 비용, 로그인·데이터 처리 중심으로 구성한다.
- 공식 역할 안내는 한 곳에서만 짧게 노출한다.
- 마지막 CTA는 `무료로 시작하기`로 통일한다.

## 5. 카피 원칙

- 쉬운 말 사용: OCR·칸반·리드 같은 표현 금지
- 긍정적으로 다음 행동 안내
- em dash 사용 금지
- 근거 없는 `88%`, `30분→3분`, `40분→5분` 삭제
- `보험설계사의 모든 영업` 같은 절대 표현은 `핵심 영업`으로 낮춤
- 법적 검토로 닫힌 기능은 사용할 수 있는 기능처럼 소개하지 않음
- `Claude AI`처럼 사용자가 몰라도 되는 모델명은 노출하지 않음

## 6. 실제 화면 촬영

- 2026-07-16 운영 `www.inpa.kr`에서 기존 데모 계정 로그인을 확인했다.
- 데모 대시보드, 고객, 분석, 일정 화면에 접근 가능하다.
- 계정 비밀번호는 설계 문서·커밋·스크린샷 메타데이터에 추가로 기록하지 않는다.
- `[DEMO]` 데이터만 사용하고, 촬영 전 화면마다 개인정보와 링크를 재확인한다.
- 대시보드의 로컬 주소가 보이는 하단 영역은 제외하거나 운영 주소가 보이도록 상태를 정리한 뒤 촬영한다.
- 캡처 날짜와 화면 비율을 가능한 한 통일한다.

## 7. 기술 구조

### 라우팅

- 공개 주소: `https://new.inpa.kr/test`
- 내부 Next 라우트: `/new/test`
- `proxy.ts`에서 `new.inpa.kr`의 `/test`만 `/new/test`로 rewrite한다.
- 기존 `/` → `/new` rewrite와 기타 경로 → `www` redirect는 유지한다.
- `/new/test` 직접 접근은 `/test`로 정규화한다.

### 컴포넌트

- 테스트 페이지 전용 섹션 컴포넌트를 사용해 현재 랜딩 컴포넌트와 상태를 공유하지 않는다.
- 브랜드 토큰, 로고, 공통 버튼 스타일은 기존 자산을 재사용한다.
- 새 패키지는 추가하지 않는다.
- 실제 화면 데이터는 화면 이름·설명·이미지·강조점을 한 배열에서 관리한다.

### 링크

- 로그인: `https://www.inpa.kr/login`
- 무료 시작: `https://www.inpa.kr/register`
- 브랜드 이야기: `https://new.inpa.kr/`
- 교차 도메인 가입 링크에는 원래 유입 정보를 보존한다.

### 검색과 분석

- 테스트 페이지 메타데이터: `robots.index=false`, `robots.follow=false`
- CTA 위치, 제품 탭 선택, 화면 확대, 브랜드 이야기 열기를 구분해 계측한다.
- 원본 UTM을 덮어쓰지 않는다.
- 성공 지표는 방문 수가 아니라 `가입 시작 → 가입 완료 → 첫 고객 등록 → 첫 분석 완료` 흐름으로 본다.

## 8. 상태와 오류 처리

- 이미지 로딩 중에는 같은 비율의 브랜드 배경을 유지해 레이아웃이 흔들리지 않게 한다.
- 이미지가 실패해도 화면 이름과 설명, CTA는 남긴다.
- 확대 화면은 포커스를 내부에 두고 닫으면 원래 버튼으로 돌려보낸다.
- 움직임 줄이기 설정에서는 등장 효과를 생략한다.
- 로그인·가입은 외부 주소 이동임을 브라우저 기본 동작으로 처리하며 새 창을 강제하지 않는다.
