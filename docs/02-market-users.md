# 시장·타겟·문제·경쟁

> **인파(Inpa)** 신사업 기획 문서 02 — 시장·타겟·문제·경쟁
> 본 문서의 모든 정량값은 **베타 90일 실측 전 가설**이다. 출처가 협회 공시·공개 통계인 항목만 "공시"로 표기하고, 나머지는 전부 `(추정)` 라벨을 유지한다. 투자 의사결정의 중심은 절대 시장가치(ARR)가 아니라 **단위경제와 wedge의 검증 가능성**임을 먼저 못박는다.

## 0. 한 장 요약 (TL;DR)

| 질문 | 답 | 근거 |
|---|---|---|
| **누구를 파는가** | 원수사 전속 + GA 위촉직 설계사(개인사업자), **개인 직접 결제** | 시장구조 메모(`project_market_structure`) |
| **1순위 타겟** | 발굴 절박한 **신입**(고객 0·틀 없음) | JTBD J1·J2 |
| **2순위 타겟** | 관리·리텐션이 자산인 **중견** | JTBD J4 |
| **가장 큰 문제** | 매달 quota는 오는데 **만날 사람이 없다 (발굴 병목)** | 발굴 1위 = DB영업(승환) |
| **wedge** | §97 부당승환 공포를 **합법 비교안내서**로 뒤집기 | 규제 핫스팟 = 진입 틈 |
| **시장 규모** | TAM 38만(공시) → SAM 12만 `(추정)` → SOM 2,400~3,600명 `(추정)` | §3 |
| **경쟁 빈틈** | B2C앱·GA시스템·Canva·범용GPT 누구도 **'증권→진단→갈아타기→멘트→링크' 파이프라인을 못 잇는다** | §6 |

핵심 통찰 한 줄: **"설계사가 30분~1시간 들여 만든 분석의 가치 90%가 본인 화면에서 증발한다."** 인파는 그 증발 지점을 영업 행동(공유·발송·소개)으로 전환한다.

## 1. 타겟 — 누가 돈을 내는가

### 1.1 결제 주체: 위촉직 개인사업자, 개인 결제

인파의 고객은 **사내 도입(B2B 사내툴)이 아니라 설계사 개인이 자기 카드로 결제**하는 구조다. 이 한 줄이 BM·GTM·가격 전부를 규정한다.

```
원수사(보험회사)
   └─ 전속 설계사 ── 한 회사 상품만 판매 (생명 or 손해)
GA(법인보험대리점)
   └─ 위촉직 설계사 ── 여러 회사 상품 비교 판매
        └─ 공통점: 모두 "위촉직 = 개인사업자"
                   회사가 툴값을 대주지 않는다 → 본인이 산다
```

- **함의 1 — 가격 민감도가 높다.** 월 19,900원을 "커피 4잔"으로 체감하게 만들어야 한다. 89,900원(Premium)은 "DB공백 스캐너 전용가"로 가치를 명확히 묶어야 지불된다.
- **함의 2 — 의사결정이 빠르다.** 품의·구매팀이 없다. 15분 데모 → 즉시 카카오 가입이 가능하다(GTM의 door-to-door 근거).
- **함의 3 — 이탈도 빠르다.** 워터마크 강제 과금처럼 자존심 건드리는 BM은 즉시 해지로 이어진다 → 워터마크는 전 티어 위생(공통)으로.

### 1.2 1순위 페르소나 — 신입 설계사 (발굴 절박형)

```
┌─────────────────────────────────────────────────┐
│  "OO 설계사 (28~38세, 위촉 6개월차)"             │
├─────────────────────────────────────────────────┤
│  상태:  고객 0명 · 보장 설명 '틀' 없음            │
│         매달 quota 압박 · 첫 계약 못 만들면 이탈  │
│  공포:  부당승환(§97) 처벌 무서워 갈아타기 못 함  │
│  결핍:  "100+ 담보를 한눈에 펼쳐주는 틀"이 없음   │
│  골든타임:  위촉 직후 ~ 첫 3개월 (도구 채택 시점) │
└─────────────────────────────────────────────────┘
```

신입이 1순위인 이유:
1. **발굴이 절박하다.** 기존 고객 자산이 없으니 신규 발굴 외에 살 길이 없다.
2. **틀이 없다.** 중견은 머릿속에 보장 체계가 있지만 신입은 없다 → 인파의 담보 정규화 사전 + 히트맵이 그 틀을 즉시 제공.
3. **콜드스타트를 우리가 만들 수 있다.** "첫 고객 5명 분석 온보딩 콜 대행"으로 첫 aha를 설계사 손에 쥐여줄 수 있다.

### 1.3 2순위 페르소나 — 중견 설계사 (리텐션 자산형)

```
┌─────────────────────────────────────────────────┐
│  "OO 설계사 (40~55세, 위촉 5년차+)"               │
├─────────────────────────────────────────────────┤
│  상태:  관리 고객 수백 명 · 방치된 계약이 금광    │
│  결핍:  만기·갱신 A/S를 일일이 못 챙김             │
│         인수인계 실패로 생긴 '고아계약' 다수       │
│  지불의사:  워치독·DB공백 스캐너에 높게 형성        │
│         (만기는 시간압박 → 업셀 최강 지점)         │
└─────────────────────────────────────────────────┘
```

> ⚠️ **미해소 쟁점:** Plus 19,900 / Premium 89,900의 **4.5배 갭**에 신입 발굴 핵심기능(워치독·귀속)이 Premium에 몰려 있다. 신입이 Plus에서 막힐 위험 → 신입용 중간티어 신설 또는 기능 재배치를 **베타 90일 실측 후 재검토**(미결, openQuestion).

### 1.4 분리 축 — 생명 vs 손해

타겟을 하나로 뭉뚱그리면 안 된다. 보험료 계산 로직(8케이스)부터 담보명까지 생명/손해가 갈린다.

| 축 | 생명보험(insurance_type=1) | 손해보험(insurance_type=2) |
|---|---|---|
| 주력 담보 | 사망·종신·연금 | 실손·진단비·수술·운전자 |
| 갈아타기 핵심 | 예정이율·해지환급 손실 | 면책 리셋·갱신 전환 |
| 계산 override | 일반사망/재해사망 적용 | **미적용** (`calculate.py`에서 `insurance_type==1` 게이트) |

→ 비교안내서·히트맵 모두 두 축을 분리 처리해야 §97 정확요건을 충족한다.

## 2. 핵심 문제 — 발굴이 진짜 병목이다 (설득 < 발굴)

### 2.1 통념의 전복

대부분의 보험 SaaS는 "설득을 잘하게 돕는다(분석·제안서 품질)"에 집중한다. 그러나 현장의 진짜 병목은 설득이 아니라 **발굴**이다.

```
   [통념의 퍼널]              [현실의 퍼널]
   만날 사람 많음              만날 사람이 없음  ◀── 진짜 병목
        │                          │
   설득력 부족  ◀── 여기 투자       설득은 됨
        │                          │
       계약                       계약
```

매달 quota(목표)는 어김없이 온다. 그런데 **"이번 달에 만날 신규 고객"이 없다.** 이것이 위촉직 설계사가 가장 자주 이탈하는 이유다.

### 2.2 발굴 1위 채널 = DB영업(승환) — 그런데 막혀 있다

한국 설계사의 신규 발굴 1순위는 **기존 가입자를 갈아타게 하는 DB영업(승환)** + 지인·소개다.

```
발굴 채널 빈도 (현장 기준, 추정)
─────────────────────────────────────
DB영업 / 갈아타기(승환)   ████████████  ← 1위, 그러나 §97로 공포
지인 · 소개               ████████
콜드 발굴 (무작위)        ██            ← 비효율 + 위법 리스크
세미나 · 제휴             ███
```

문제: **갈아타기 = 부당승환(보험업법 §97)의 규제 핫스팟.** 부정확한 비교로 갈아타게 하면 영업정지·과태료. 그래서 설계사는 발굴 1위 채널을 무서워서 손도 못 댄다.

### 2.3 §97을 wedge로 — 공포를 합법 방패로 뒤집기

```
  규제(§97)를 "장벽"으로 보면        규제를 "기능"으로 뒤집으면
  ───────────────────────           ───────────────────────────
  갈아타기 = 무서워서 회피     →     갈아타기 = 자동 비교안내서가
  (막힌 1위 발굴 채널)               합법 방패로 남는다 (열린 채널)
```

이것이 인파의 정체성: **분석 툴이 아니라, 막힌 발굴 채널을 합법적으로 여는 wedge.** 범용 GPT·Canva가 절대 줄 수 없는 단 하나의 기능이 §97 비교안내서다.

> ⚠️ **레드라인:** 비교안내서는 "AI 1차 보조, 최종책임 설계사" 면책 고정. **'심의 안전/완료' 배지는 절대 금지**(보증책임 전가 = 소송 리스크). 필수 6항목 미완료 시 발행 **하드블록**.

### 2.4 세 번째 문제 — 분석 가치 90% 증발

```
설계사가 1시간 들여 만든 보장분석
        │
        ▼
   본인 PC 화면에만 머묾  ◀── 가치 90% 증발
        │
        ✕  고객에게 안 전달 / 지인 소개로 안 이어짐
```

생산은 끝났는데 **전달이 빠져 있다.** 인파의 공유링크(share_token)는 이 증발 지점을 막고, `?ref=` 귀속으로 소개 리드까지 자동 회수한다.

### 2.5 네 번째 구조 문제 — 고아계약

원수사의 관리 공백 + 설계사 퇴사 시 **인수인계 실패** → 주인 없는 '고아계약'이 대량 발생. 중견 설계사에게는 이것이 **방치된 금광**이다. 인파의 워치독(만기·갱신 감시)이 이 금광을 캐는 도구가 된다.

## 3. 시장 규모 — TAM → SAM → SOM

> 서사의 중심은 ARR이 아니라 단위경제다. 아래 ARR은 시장 크기 감각용일 뿐, 펀딩 논거는 §5(LTV/CAC, 별도 문서 06)에 둔다.

### 3.1 깔때기

```
┌──────────────────────────────────────────────────────────┐
│ TAM  생·손보 위촉 설계사 전체                              │
│      ≈ 38만 명  (공시: 생손보 협회 등록, 중복위촉·휴면 포함)│
│      ARPU 25,000원 가정 → 약 1,140억/년 (상한, 비현실)      │
├──────────────────────────────────────────────────────────┤
│ SAM  디지털 친화 + 발굴 절박 신입~중견                     │
│      ≈ 12만 명 (TAM의 31%)  (추정)                          │
│      = 스마트폰 활용 + DB영업층 + 5년차 이하 중심           │
│      ARPU 25,000원 → 약 360억/년  (추정)                    │
├──────────────────────────────────────────────────────────┤
│ SOM  3년 내 실제 유료 전환 (freemium 보수적)               │
│      ≈ 2,400~3,600명 (SAM의 2~3%)  (추정)                   │
│      ARPU 25,000원 → ARR 7.2~10.8억  (추정)                 │
└──────────────────────────────────────────────────────────┘
```

### 3.2 수치 가정 명세 (전부 검증 대상)

| 항목 | 값 | 출처 / 가정 | 라벨 |
|---|---|---|---|
| TAM | 38만 명 | 생손보 협회 등록 공시 (중복위촉·휴면 포함) | 공시 |
| SAM 비율 | 31% | 디지털 친화 + 발굴 절박 신입~중견 추정 | (추정) |
| SOM 전환율 | 2~3% | 초기 freemium 보수적 전환 가정 | (추정) |
| ARPU | 25,000원/월 | Plus 19,900 / Premium 89,900 **가중평균 가정값** (믹스 미검증) | (추정) |

> 💡 **읽는 법:** TAM 38만만 공시다. SAM·SOM·ARPU는 전부 가설 → 베타 90일 실측 후 토큰화. 투자자에게는 "1,140억 시장"이 아니라 **"단위경제가 흑자인 wedge를 보수적 2~3% 전환으로 검증한다"**로 서사를 건다.

## 4. JTBD — 설계사가 인파를 '고용'하는 4가지 일

```
J1 ─ 발굴   "quota는 오는데 이번 달 만날 사람이 없다"
            → 갈아타기 후보 자동 발굴 + 비교안내서로 새 고객 확보
            (1순위 신입의 생존 문제)

J2 ─ 신뢰   "보장을 설명할 '틀'이 없다. 특히 신입인 나는"
            → 100+ 담보 히트맵 3색으로 즉석에서 전문가처럼 보이게

J3 ─ 면책   "§97 부당승환 처벌이 무섭다"
            → 자동 비교안내서 = 합법 방패 (공포 → 효익 전환)

J4 ─ 복리   "방치된 고객·고아계약을 A/S로 다시 살리고 싶다"
            → 워치독(만기·갱신) + DB공백 스캐너 (2순위 중견의 금광)
```

각 JTBD는 MVP 모듈과 1:1 매핑된다(문서 03 참조): J1→M4 비교안내서, J2→M1 OCR+M3 히트맵, J3→M5 가드레일+M7 동의, J4→Phase1.5 워치독.

## 5. 경쟁 지형 — 우리만의 칸

### 5.1 4군 빈틈 분석

| 경쟁군 | 대표 | 하는 일 | 못 잇는 빈틈 | 인파 차별점 |
|---|---|---|---|---|
| **B2C 보험앱** | 굿리치·보맵·보험클리닉 | 소비자가 본인 보험 조회 | **소비자뷰** — 설계사 영업행동 미지원 | 설계사 측 발굴·제안·발송 파이프라인 |
| **GA 내부 시스템** | 각 GA 전용 CRM | 지점 고객·실적 관리 | **폐쇄형** — 외부 설계사·바이럴 없음 | 개인 결제·공유링크 귀속·바이럴 루프 |
| **디자인 툴** | Canva·미리캔버스 | 예쁜 판촉물 | **디자인 only** — 고객데이터·분석 단절 | 증권 데이터 → 분석 → 판촉물 연결 |
| **범용 GPT** | ChatGPT 등 | 멘트 작문 | **데이터·컴플라이언스·갈아타기 없음** | 담보 정규화 사전 + §97 비교안내서 |

### 5.2 한 칸으로 시각화

```
                  설계사 영업행동 지원
                          ▲
                          │
   범용 GPT ○             │      ★ 인파 (Inpa)
   (멘트만, 데이터X)       │      증권→진단→갈아타기
                          │      →멘트→링크 한 흐름
   Canva ○                │
   (디자인만)    ─────────┼──────────────────►
                          │             고객데이터 + 컴플라이언스
   B2C앱 ○                │
   (소비자뷰)             │      GA시스템 ○
                          │      (폐쇄, 외부X)
                          │
```

**우리만의 칸:** "증권 → 보장공백 진단 → 갈아타기 비교안내서(§97 방패) → 카톡 멘트 → 공유링크"가 **끊김 없이 한 흐름**으로 이어지는 영업 파이프라인. 어느 경쟁군도 이 5단계를 연결하지 못한다.

### 5.3 차별점이 '해자'인 이유 (모방 난이도)

- **갈아타기 자동 비교안내서** — 범용 AI·Canva가 못 줌. 규제를 기능으로 뒤집은 wedge. (모방하려면 §97 도메인 + 8케이스 계산 + 면책 설계 전부 필요)
- **보험사별 담보명 정규화 사전** — 같은 담보도 회사별 이름이 다르다(일반암 진단비 = 삼성 *암진단급부금* / 교보 *암진단보험금*). 쓸수록 두꺼워지는 **데이터 복리 플라이휠**.
- **끊김 없는 파이프라인** — 단계 하나가 아니라 연결 자체가 해자.

> 3중 컴플라이언스 가드레일은 **해자가 아니라 위생**(없으면 출시 불가)이다. 혼동 금지.
