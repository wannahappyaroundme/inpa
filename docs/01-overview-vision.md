## 0. 한 줄 정의

> **인파(Inpa)는 위촉직 보험설계사가 [발굴 → 분석 → 갈아타기 제안 → 성사]를 한 흐름으로 끝내게 돕는 AI 영업 파트너다.**

분석 툴이 아니다. **영업 OS**다. 증권 한 장을 넣으면 보장 공백 진단 → 갈아타기 비교안내서 → 카톡 멘트 → 고객 공유링크가 끊김 없이 한 줄로 흐른다. 그리고 그 흐름의 시작점은 "분석"이지만, 끝점은 "새 고객 발굴"이다.

```
[증권 PDF]  →  [일괄 OCR]  →  [담보 정규화]  →  [보장 히트맵]  →  [갈아타기 비교안내서]  →  [AI 카톡 멘트]  →  [고객 공유링크]
   업로드        Claude         정규화 사전        3색 공백 스캐너      §97 방패 자동 발행        목적 enum 생성        share_token + ?ref=
```

이 한 줄을 **5분 안에** 끝내는 것이 인파의 첫 약속이다.

## 1. 비전·미션

### 비전
> 위촉직 설계사가 "고객을 찾고, 보장을 풀어 보여주고, 갈아타기를 합법적으로 제안하고, 계약을 성사시키는" 전 과정을 AI가 곁에서 대신 준비해주는 세상.

### 미션
> 설계사의 가장 큰 병목인 **"새 고객 발굴"**을, 합법적 갈아타기(승환) 비교안내서를 무기로 풀고, **분석을 영업 행동의 시작점**으로 바꾼다.

### 정체성 전환의 증거
인파는 foliio의 검증된 파이프라인(증권 OCR → 8케이스 보험료 계산 → 공유링크)을 **새로 짓는 것이 아니라 영업 사이클로 재배치**하는 것이다. 코드 90% 재활용, 10% 신규.

정체성이 "분석 툴 → 영업 OS"로 바뀌었다는 가장 단단한 증거는 **우리가 세는 숫자가 바뀐다는 것**이다.

```
구(舊) foliio:  "분석 건수" 를 센다  →  우리는 계산기다.
신(新) 인파:    "발굴 액션 수" 를 센다  →  우리는 영업 파트너다.
```

## 2. 3축 정체성 — 분석=무기 / 콘텐츠=미끼 / 리텐션=복리

인파를 떠받치는 세 개의 기둥. 셋은 같은 흐름의 다른 구간이다.

| 축 | 역할 | 핵심 문장 | 비유 |
|---|---|---|---|
| **분석 = 무기(방아쇠)** | 영업 행동을 *시작*시킨다 | "분석은 끝점이 아니라 방아쇠다." | 증권 → 히트맵 → 비교표가 미팅을 잡는 *총알* |
| **콘텐츠 = 미끼** | 무료로 가치를 *보여준다* | "분석은 풀고, 행동은 막는다." | 1건 비교안내서 *생성* 무료 체험 → wedge를 맛보임 |
| **리텐션 = 복리** | 시간이 *쌓일수록* 강해진다 | "쓸수록 두꺼워지는 데이터 사전." | 정규화 사전 hit_count가 누적되는 플라이휠 |

> **핵심 통찰**: 설계사가 30분~1시간 들여 만든 보장분석이 **본인 화면에서만 머문다.** 생산은 끝났는데 *전달*이 빠져 있다 — 분석 가치의 90%가 증발한다. 인파는 그 분석을 "고객에게 보내는 행동"으로 직결시킨다.

## 3. 작명 근거 — 왜 "인파(Inpa)"인가

```
  Inpa  =  In(sure)  +  Pa(rtner)
           인슈어        파트너
```

- **실존 단어 + 약어 합성**: "당근 = 당신 근처"식 네이밍 패턴. 발음하기 쉽고 입에 붙는다.
- **'곁에 있는 파트너'라는 정서**: 차갑고 기능적인 "분석 시스템"이 아니라, **곁에서 일을 거들어주는 동료(partner)**.
- **포지션이 가격 저항을 낮춘다**: "도구(tool)를 산다"는 비용 인식보다 "파트너를 둔다"는 인식이 월 구독 결제 거부감을 줄인다.

> ⚠️ 단, '파트너'가 **무자격 모집 오해**를 일으키지 않도록 전 카피에서 "모집 보조 도구 · 최종 책임은 설계사" 면책을 일관 표기한다(§9, 정직성 레드라인). 상표 출원 사전 점검은 GTM 문서(07)에서 다룬다.

## 4. 포지셔닝 — 우리만 차지하는 한 칸

**"보험설계사 곁의 AI 영업 파트너."** 경쟁 지형에서 누구도 잇지 못하는 칸은 **'증권 → 진단 → 갈아타기 → 멘트 → 링크' 파이프라인**이다.

### 경쟁 비교표

| 경쟁군 | 하는 일 | 못 잇는 빈틈 |
|---|---|---|
| **B2C 보험앱** (굿리치·보험클리닉) | 소비자가 *본인* 보험 조회 | **소비자뷰** — 설계사의 영업 행동을 지원하지 않음 |
| **GA 내부 시스템** | 지점 전용 고객·실적 관리 | **폐쇄형** — 외부 설계사 진입·바이럴 불가 |
| **Canva·미리캔버스** | 예쁜 판촉물 디자인 | **디자인 only** — 고객 데이터·분석과 단절 |
| **ChatGPT 범용 LLM** | 멘트 작문 | 고객 데이터·분석·**컴플라이언스·갈아타기 비교안내 없음** |
| **인파(Inpa)** | **증권→진단→갈아타기→멘트→링크 한 흐름** | — (우리의 칸) |

### 포지셔닝 맵 (ASCII)

```
                 영업 행동 연결 강함 (발굴까지)
                          ▲
                          │
                          │            ● 인파(Inpa)
                          │              증권→진단→갈아타기→멘트→링크
                          │
   데이터/컴플라 ◀────────┼────────────────────────▶ 데이터/컴플라
      없음                │                              있음
                          │
        ● Canva           │      ● GA 내부 시스템
        ● ChatGPT         │        (폐쇄형, 외부 X)
                          │      ● B2C 보험앱(소비자뷰)
                          │
                          ▼
                 영업 행동 연결 약함 (조회/작문에서 끝)
```

> 범용 AI와 캔바가 **절대** 못 주는 것이 **갈아타기 자동 비교안내서**다. 이것이 우리 칸의 빗장이다.

## 5. Why Now — 왜 지금인가 (4개)

```
① 발굴이 진짜 병목이다        설득 < 발굴
② 발굴 1위 = 갈아타기인데 §97로 막힘   → 규제를 기능으로 뒤집는다
③ foliio 검증 파이프라인 이전비용 낮음   → 90% 재활용
④ Claude 단가가 임계점 아래로 하락       → 단위경제 감당 가능
```

1. **발굴이 진짜 병목이다 (설득 < 발굴).** 위촉직 설계사는 매달 quota 압박에 시달린다. 정작 어려운 건 "어떻게 설득하나"가 아니라 **"누구를 만나나"**다. 한국 발굴 1위 채널 = DB영업 = 기존 가입자 갈아타기(승환) + 지인·소개.

2. **발굴 1위가 §97로 막혀 있다 → 규제를 기능으로 뒤집는다.** 갈아타기는 부당승환(보험업법 §97) 규제의 핫스팟이라 설계사가 무서워서 손도 못 댄다. 인파는 이 공포를 **합법 방패(자동 비교안내서)**로 뒤집어, 막혀 있던 1순위 발굴 채널을 연다. *규제가 곧 wedge다.*

3. **이전 비용이 낮다.** foliio v1.2~v1.6 파이프라인이 이미 검증·고도화돼 있다. 신규 의존성 0개, 별도 repo `~/Desktop/inpa`로 포팅. 적은 투자로 정체성을 바꾼다.

4. **AI 단가가 임계점 아래로 내려왔다.** Claude로 멘트·진단 내러티브 생성이 단위경제상 감당 가능해졌다. Prompt caching + Batches로 호출당 원가를 수백 원대로 낮추고, 크레딧 차감으로 통제한다. **(추정)**

## 6. 해자(Moat) 3종 vs 가드레일(위생) — 정직한 구분

인파는 "해자"와 "위생"을 섞지 않는다. **위생은 없으면 출시 불가지만 차별점이 아니다.** 해자만이 후발주자를 막는다.

### 진짜 해자 3종 (모방 어려움)

| # | 해자 | 왜 모방 어려운가 |
|---|---|---|
| **① 갈아타기 자동 비교안내서** | §97 방패. 보험업법 정확요건(불리점 완전성)을 코드로 강제. **범용 AI·캔바가 절대 못 줌** — 규제를 기능으로 뒤집은 wedge |
| **② 담보 100+ '틀' + 보험사별 담보명 정규화 사전** | 같은 담보도 보험사별 이름이 다름(일반암 진단비 = 삼성 *암진단급부금* / 교보 *암진단보험금*). **쓸수록 두꺼워지는 데이터 복리.** 신규 설계사가 못 가진 '틀'을 제공 |
| **③ 끊김 없는 파이프라인** | 증권→진단→갈아타기→멘트→링크를 한 흐름으로 잇는 통합. 경쟁 4군 누구도 전 구간을 잇지 못함 |

### 가드레일 = 위생 (해자 아님)

- 3중 컴플라이언스 가드레일(단정표현·수익보장·비교과장 플래그)은 **있어야 출시할 수 있는 위생**이지 차별점이 아니다.
- ❌ **'심의 안전/완료' 배지 절대 금지** — 보증 책임을 떠안게 되어 소송 리스크. ✅ "AI 1차 보조, 최종 책임은 설계사" 면책 카피로만 포지셔닝.

### 데이터 복리 플라이휠 (해자 ②의 구조)

```
        OCR 미매칭 raw_name
              │
              ▼
        unmatched_log 적재
              │
              ▼
        admin 1탭 매핑 (검수)
              │
              ▼
   NormalizationDict 영구 추가 (hit_count++)
              │
              ▼
   다음 증권부터 자동 매칭 ──┐
              │             │
              ▼             │ 매칭 정확도 ↑
        분석 품질 ↑          │ → 발송·열람 ↑
              │             │ → 신규 증권 유입 ↑
              ▼             │ → 사전 두꺼워짐
   발송·열람·신규 증권 유입 ─┘
              │
              ▼
        (다시 OCR 미매칭으로 순환 — 후발주자 진입장벽)
```

> 정규화 사전은 사용량이 늘수록 매칭 정확도가 올라간다. **(추정)** 매칭 정확도 85% → 93%. 이것이 단순 복제로는 따라올 수 없는 데이터 플라이휠이다. (자동승격 임계 — hit_count 5회+ 자동 매핑 허용 여부 — 는 운영 미결, openQuestion에 있음.)

## 7. 6대 필러 (Six Pillars)

인파를 구성하는 6개 기능 축. 각 필러는 영업 퍼널의 한 구간을 책임진다.

```
┌─────────────────────────────────────────────────────────────────────┐
│  P1 분석-콘텐츠 파이프라인  →  P2 컴플라이언스 안전망                │
│         (무기 + 미끼)              (위생, 출시 게이트)                 │
│                                                                       │
│  P3 리텐션 엔진            →  P4 리드 성장 루프(바이럴 귀속)         │
│         (복리)                     (발굴 확장)                        │
│                                                                       │
│  P5 퍼스널 브랜딩          →  P6 측정·과금 거버넌스                  │
│         (차별화)                   (북극성 + 단위경제)                │
└─────────────────────────────────────────────────────────────────────┘
```

| # | 필러 | 무엇 | 역할 | MVP 단계 |
|---|---|---|---|---|
| **P1** | **분석-콘텐츠 파이프라인** | 증권→OCR→정규화→히트맵→비교안내서→멘트→링크 | 무기 + 미끼 (핵심 루프) | **MVP (T0)** |
| **P2** | **컴플라이언스 안전망** | §97 비교안내 정확요건 + 가드레일 + 국외이전 동의 | 위생 (출시 가부 게이트) | **MVP (T0)** |
| **P3** | **리텐션 엔진** | 만기·갱신·생일 워치독 + AI 내러티브 + 액션 큐 | 복리 (A/S로 소개·추가계약) | Phase 1.5 (T1) |
| **P4** | **리드 성장 루프** | share_token `?ref=` 귀속 + 셀프진단 인바운드 + 더블사이드 리퍼럴 | 발굴 확장 (바이럴) | Phase 2 (T2) — *계측은 Day1* |
| **P5** | **퍼스널 브랜딩** | 'Powered by 인파' 워터마크 + 14 판촉 템플릿 + 고객뷰 브랜딩 | 차별화 (입소문) | Phase 3 (T3) |
| **P6** | **측정·과금 거버넌스** | 북극성 KPI + 크레딧 차감 + 호출당 비용 로깅·예산 캡 | 신뢰 (단위경제) | **MVP (T0)** |

> ⚠️ **P4의 핵심 규칙**: 바이럴 *루프 활성화*는 Phase 2지만, **귀속 계측(`?ref=` 파라미터 + 이벤트 스펙)은 Day1에 깐다.** 계측은 사후 복원이 불가능하기 때문이다. "계측은 1일차, 캠페인은 2단계."

## 8. 북극성 KPI — 지표가 곧 정체성

> **North Star = (미팅당 공유링크 발송수) × (열람률) × (소개 귀속 리드수)**

**곱셈 구조**다. 셋 중 하나라도 0이면 전체가 0 — 발송만 늘려도, 열람만 좋아도 안 된다. 발송 → 열람 → 귀속이 모두 살아야 한다.

```
   North Star  =  발송수  ×  열람률  ×  소개 귀속 리드수
                  └─보냈나   └─봤나     └─새 고객으로 이어졌나
```

### 측정 위계 (대표가 매주 보는 대시보드)

| 레벨 | 지표 | 측정 방법 |
|---|---|---|
| **L0 북극성** | 발송수 × 열람률 × 소개 귀속 리드수 | 곱셈 |
| **L1 선행** | 비교안내서 생성 수, 진단 카드 생성 수, AI 멘트 복사 수 | 이벤트 카운터 |
| **L2 리텐션** | WAU/MAU, 무료→Plus 전환율, Plus→Premium 전환율, 4주 잔존율 | 코호트 |
| **L3 단위경제** | 사용자당 AI 호출비용, CAC(귀속 기반), LTV | 비용 로깅 |

### 신뢰 KPI vs 허영 지표 vs 프록시

- ✅ **신뢰 KPI** = `share_token` 열람 이벤트 (**서버 측정 가능**)
- ⚠️ **프록시(보조)** = 클립보드 복사 = '발송 의도'. 원탭 자동 발송이 카카오상 불가하므로 복사 시점만 잡힌다 — 신뢰 KPI로 쓰지 않고 보조 카운터로만.
- ❌ **허영 지표** = 가입자 수 · MAU · 분석 건수. 정체성이 영업 OS이므로 "분석 건수"는 끝점이 아니라 허영이다.

> **이 지표 변경이 곧 정체성 변경의 증거다.** "분석 건수"를 세면 계산기, "발송×열람×귀속"을 세면 영업 OS.

## 9. 회사 리스크 표 — 과장 금지 영역

각 리스크에 대해 **"이건 절대 안 한다"**는 안전 원칙을 못 박는다.

| # | 리스크 | 위험 | 안전 원칙 (확정) |
|---|---|---|---|
| **R1** | **컴플라이언스 과신** | 필터는 비교광고·특별이익·불완전판매 맥락 위반을 못 잡음. '안전 배지' = 보증 책임 = 소송 | "AI 1차 보조, 최종 책임 설계사" 면책으로만 한정. **안전 배지 절대 금지** |
| **R2** | **AI 진단 오진** | "암 진단비 X만원 부족" 단정이 틀리면 부당권유. 기준선 출처 불명확 = 판정 부당 | **표준 기준선 출처·면책 확정 전 enough/short 판정 보류, none(0원 여부)만 회색 중립 표기** |
| **R3** | **발송 인프라 거짓 전제** | 원탭 자동 발송은 카카오상 불가 | 개별 멘트는 **클립보드 복사로 정직하게 격하**. 자동 발송 사칭 금지 |
| **R4** | **입력 노동 함정** | 만기/소개 데이터가 비면 리텐션·소개가 공회전 | 수기 입력 의존 기능은 3주면 버려짐. OCR로 채우고 자동 이벤트 캡처로 끈다 |
| **R5** | **AI 비용 누수** | 무료 무한 호출 시 단위경제 붕괴 | 크레딧 차감 게이팅 + 호출당 로깅 + 월 예산 캡 + 임박 알림 필수 |
| **R6** | **콜드스타트** | 신입은 빈 상태로 진입, 첫 5분 이탈 | 본인 증권 셀프 시드 · 첫 고객 5명 온보딩콜 대행 |

### 🧨 가장 무서운 단일 폭탄 — 병력 국외이전 동의 게이트

> **병력(민감정보)을 Claude API(미국, Anthropic Inc.)로 보내는 국외이전 동의가 막히면 — `detect` API 자체가 안 열리고, AI 기능 *전체*의 출시 가부가 막힌다.**

- 이건 코딩 전 **법무 선결 1순위.** 낙관 가정(customer-agree 1필드로 커버) 금지.
- **설계 전제**: 별도 필드 `consent_overseas_at` + `ConsentLog`(누가·언제·범위·doc_version·ip) 분리. `detect` 호출 전 `consent_overseas_at` 확인 → 미동의 시 **412 게이트**.
- **막히면 fallback**: OCR(M1) → 히트맵 none 중립(M3) → 정규화(M2)부터 선출시. 비교안내서·AI 메시지는 §97·동의 확정 후 오픈.
