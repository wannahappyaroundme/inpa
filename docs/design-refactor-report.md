## 1. 한 줄 요약

레퍼런스 1번 이미지가 추구하는 시각 언어(**큰 숫자 · 컬러 아이콘 뱃지 · 컬러 파이프라인 · 넉넉한 여백/라운드/그림자**)를
대시보드는 물론 **앱 전반(서비스 페이지 34개)** 에 입혔습니다. 기능·데이터·라우팅은 한 줄도 안 바꿨고(프레젠테이션만),
대표님이 지키라고 한 **파랑·빨강·노랑·초록 4색은 그대로** 유지했습니다.

## 2. 제가 대신 내린 2가지 결정 (주무시는 동안, 지시 우선순위대로)

1. **색은 안 바꿨습니다.** 스펙 문서(`ui-redesign-spec.md`)는 새 파랑(#2E50F0) 등 *다른* 색을 제안했지만,
   대표님의 "절대 바꾸지 마"가 우선이라 **무시**하고 우리 4색을 유지했습니다.
   스펙에서 가져온 건 *구조*(큰 숫자·아이콘 뱃지·컬러 단계 카드·여백·라운드)뿐이고, 보조색(배경/글자/연한 틴트/단계색)은
   **우리 4색에서 다시 만들어** 입혔습니다.
2. **상단 네비를 유지했습니다(사이드바 전환 안 함).** 첨부 이미지는 사이드바지만, 스펙의 1번 규칙이 "상단 유지"이고
   사이드바는 *선택*(§8, "한 번에 바꾸지 말 것")으로 적혀 있었습니다. 27개 페이지의 레이아웃을 무인 상태로 갈아엎는 건
   리스크가 커서, **상단 네비를 더 예쁘게 다듬는(액티브 알약 버튼) 쪽**으로 했습니다.
   → **사이드바로 정말 바꾸고 싶으시면 "사이드바로 해줘" 한마디면 별도로 진행합니다.** (토큰·카드 작업은 그대로 재사용돼서 빠릅니다.)

## 3. 무엇이 바뀌었나 (눈에 띄는 순서)

- **통계 카드 5개**: 라벨+숫자만 있던 카드에 **컬러 아이콘 뱃지 + 큰 숫자(28px) + 증감 화살표(+75% 초록 ↑ / −16% 빨강 ↓)** 추가.
- **영업 단계 파이프라인**: 평평하던 4칸을 **단계별 색 카드 + 화살표**로(01 회색 / 02 빨강 / 03 노랑 / 04 초록). "흐름"이 보입니다.
- **상단 네비**: 현재 메뉴가 **파란 알약(pill)** 으로 강조, sticky 유지.
- **카드 전체**: 그림자를 부드러운 2단(`shadow-card`)으로 통일, 라운드·여백 일관화.
- **환수 레이더 배너**: 이모지 → 깔끔한 컬러 아이콘 뱃지.
- **하단 퀵액션 바** 신설: 고객 목록 / 보장 분석 / 상담 예약 / 상담 화법 / 일정 관리 바로가기(레퍼런스 1번 하단처럼).
- **차트**: 하드코딩돼 있던 파랑(#3b82f6 평균선)을 토큰으로 교체. 도넛·막대 색은 원래대로 4색 유지.
- **앱 전반 34개 페이지**: 고객·고객상세·분석·일정·화법·게시판·판촉물·알림·설정·온보딩·로그인·랜딩 등 전부
  공통 토큰/카드/네비가 자동 반영 + 페이지별로 하드코딩 회색·그림자를 토큰으로 정리.

2. **대시보드 "한눈에" 그리드** (`app/home/page.tsx`) — 세로 스크롤 → 12칼럼(좌 8 + 우 4). 좌: 목표카드+달성률 도넛 게이지 → 통계 4칸 → 단계 파이프라인 → 막대차트+캘린더. 우 레일: 오늘 일정 · 환수 레이더 · 유지현황 · 유지율 · 예약링크 CTA · 판촉물 CTA. 하단 퀵액션 바.
