# 인파 브랜드 & 로고

> **인파(Inpa)** — 인슈어(Insure) × 파트너(Partner) × 人波. 보험설계사가 고객 앞에서 전문가로 보이게 만드는 영업지원 AI OS의 브랜드 가이드.
> 본 문서는 디자인 시스템(`design/00-design-system.md`)의 하위 문서로, 로고·컬러 의미·보이스·톤앤매너·접점 적용을 다룬다. 토큰 전체 hex표는 `00-design-system.md` 참조.

---

## 0. 한 줄 정의

> **"내 편에서 가장 정확한 한 장을 만들어주는, 조용히 유능한 파트너."**

인파의 작명 자체가 약속이다. 'AI 툴'이 아니라 **'파트너(편)'**. 그래서 브랜드의 주인공은 인파가 아니라 **인파를 든 설계사**다. 도구는 뒤로 빠지고, 사용자의 전문가 정체성이 올라간다(당근이 '동네 이웃'을 주인공으로 세운 구조와 동일). 브랜드의 1순위 임무 단 한 문장:

> **"이걸 켜면 설계사가 전문가로 보인다."**

## 1. 브랜드 네이밍 — 인파의 삼중성(三重性)

'인파'는 한 단어에 세 의미를 포갠다. 이 중의성이 컨셉의 심장이다.

| 층위 | 원어 | 의미 | 브랜드 역할 |
|---|---|---|---|
| **인슈어** | Insure (보장) | 덮어주는 보장, 곡률의 호(arc) | 사업 본질(보험) |
| **파트너** | Partner (동행) | 곁에서 나란히 가는 두 획 | 포지셔닝(도구 아닌 편) |
| **人波** | 인파(사람 물결) | 신규 고객의 물결이 밀려옴 | 북극성(고객 발굴) |

```
  인슈어 ─ 무엇을 (보장 데이터)
  파트너 ─ 어떻게 (곁에서 동행)
  人波   ─ 무엇을 위해 (신규 고객 발굴 = 북극성)
```

### ⚠️ 의도적으로 버린 의미 — 방패/보장 보증

`人波`의 물결·동행은 살리되, **인슈어가 자칫 끌고 오는 '방패=보장 책임 보증' 함의는 정직성 레드라인 위반(보증 책임 전가)으로 의도적 폐기**한다. 인파의 의미는 오직 **동행(파트너)** 과 **충족 데이터(히트맵)** 둘뿐이다. 로고가 "우리가 당신을 지켜준다"고 말하는 순간, 그건 설계사의 책임을 인파가 대신 떠안는 거짓 약속이 된다.

## 2. 로고 컨셉 — 겹친 두 호(雙弧, Twin-Arc)

### 2.1 심볼의 정당성 — 한 도형이 세 의미를 다 품는다

심볼은 **굵은 호 위에 가는 호가 평행으로 겹친 형태**다.

```
       ╭──────────────╮          ← 가는 호 (굵기 1.6, 인디고 #1E40C4 = 인파의 제안)
     ╭────────────────────╮      ← 굵은 호 (굵기 3.0, 민트 #12B5A4 = 기존 보장)
              ●                   ← 충족 도트 (인디고, in의 점 = 충족셀 모티프)
```

- **굵은 호 (민트 #12B5A4)** = 고객이 이미 가진 **기존 보장**.
- **가는 호 (인디고 #1E40C4)** = 인파가 더하는 **제안**(갈아타기 해자).
- **두 호 사이 음각 공간** = 나란히 어깨를 겯은 **두 사람**(파트너·동행)이자 **물결(波)의 마루**.
- **음각 중앙의 인디고 충족 도트 1개** = `in`의 점(tittle)이자 히트맵의 **충족셀(데이터)** 모티프를 흡수.

### 2.2 비율이 곧 메시지 — 3 : 1.6

굵은 호 : 가는 호 = **3 : 1.6**. 이 비율은 장식이 아니라 정직성의 형태화다.

> **제안이 기존을 덮되 가리지 않는다.** 가는 인디고 호(제안)가 굵은 민트 호(기존) 위에 얹히지만, 기존을 완전히 가리지 않는다 — "당신이 이미 가진 건 그대로 둔다"는 정직성을 비율로 표현.

### 2.3 획·곡률 규칙

| 항목 | 값 | 이유 |
|---|---|---|
| 그리드 | 8pt 기반 | 시스템 그리드와 일치 |
| 호 굵기비 | 굵은 3.0 : 가는 1.6 | 덮되 가리지 않음(정직성) |
| 모서리 join | **round join** | 우산/방패의 부드러움 = 신뢰·비위협 |
| 곡률 | 두 호 곡률 일관 | 평행·동행 시각화 |
| 효과 | 그라데이션·드롭섀도·광택 **전면 금지** | 데이터 도구 절제(무신사 DNA) |

## 3. 워드마크 — 워드마크 우선, 심볼 종속

인파는 **워드마크 우선** 브랜드다(심볼은 종속). 보험 현장은 종이 명함·출력물 비중이 높고, 워드마크가 16px·흑백에서도 살아남아야 하기 때문이다.

```
   inpa   인파
   ▲  ▲   ▲
   │  │   └─ 한글 병기 (시장 친근감)
   │  └──── i의 tittle = 인디고 #1E40C4 도트 (심볼 없이도 브랜드 식별)
   └─────── 소문자 산세리프, weight 700, 자간 -2%, round join
```

### 3.1 구성 규칙

- **영문 `inpa`** — 소문자, weight 700, 자간 -2%, round join. 소문자 = B2B 절제·신뢰감.
- **한글 `인파` 병기** — 영문(절제 신뢰) + 한글(시장 친근감)의 이중 인상.
- **`i`의 점(tittle)을 인디고 #1E40C4 도트로 치환** — 심볼 없이도 브랜드 식별 가능 + 앱 액션큐 알림 도트와 시각 일관성.
- **`n → p` 베이스라인에 미세한 호(arc)** — 평행·동행의 암시. **장식은 여기서 멈춘다**(과장 금지 원칙).
- **색** — 무채색(회색조 후퇴) 또는 딥인디고 #1E40C4 단색. 다색 워드마크 금지.

## 5. 사용 금지 규칙 (Don'ts)

정직성 레드라인은 **로고 레벨에서 물리적으로 차단**한다. 디자인 라이브러리에서 슬롯 자체를 제거해 실수로도 못 넣게 한다.

| 금지 | 이유 |
|---|---|
| 🛡️ 방패 / ✓ 체크 / 👑 왕관 / ★ 별 / " 따옴표 | **보증·안전·심의 함의** = 보증 책임 전가(레드라인) |
| ✅ 그린 체크 / 🏅 금색 인증마크 | "심의 완료/인증" 거짓 신호 |
| 🐤 마스코트 / 캐릭터 | 보험 신뢰톤 부적합, 친근 과잉 |
| 색반전(민트↔인디고) | **민트=기존 / 인디고=제안 의미 고정** — 반전 시 메시지 오염 |
| 그라데이션 / 드롭섀도 / 광택 | 데이터 도구 절제(무신사 DNA) 위반 |
| "안심" / "보장 완료" 텍스트 결합 | 과장 카피 = 정직성 레드라인 |

> **체크리스트 (로고 QA)**
> - [ ] 방패·체크·인증마크 류 슬롯이 라이브러리에 **물리적으로 부재**한가
> - [ ] 16px·흑백 출력에서 워드마크가 읽히는가
> - [ ] 민트/인디고 색 의미가 반전되지 않았는가
> - [ ] 그라데이션·드롭섀도가 0인가
> - [ ] "안전·보장 완료" 류 카피가 결합되지 않았는가

## 6. 컬러 의미 — 왜 이 색인가

> 전체 토큰 hex표는 `00-design-system.md`. 여기선 **각 색이 왜 그 의미를 갖는가(심리·정당성)** 만 다룬다.

### 6.1 키컬러 분리 — 브랜드 인디고 vs 데이터 인디고

인파의 가장 중요한 컬러 결정: **인디고가 두 개다.**

| 구분 | 이름 | hex | 전용 영역 | 절대 금지 |
|---|---|---|---|---|
| **브랜드** | Inpa Indigo | **#1E40C4** | 헤더 · CTA · 로고 심볼 | 히트맵 셀에 사용 금지 |
| **데이터** | Proposal Indigo | **#3B5BDB** | 히트맵 충족셀 · 비교표 제안선 | 헤더 · CTA에 사용 금지 |

**왜 분리하나?** 히트맵 한 화면에 100+ 셀이 깔린다. 충족셀 색(#3B5BDB)과 브랜드 강조색이 같으면, 셀이 "충족 데이터"인지 "브랜드 강조"인지 **구분 불가**해진다. 그래서 브랜드는 한 톤 깊은 딥인디고 **#1E40C4**로 분리하고, 데이터 의미색 #3B5BDB는 셀·제안선 전용으로 **잠근다**.

> **조정 근거**: BI안 #2D4BC4(구분 약함)와 PM안 #1B3A6B(CTA 활력 부족) 사이에서 **#1E40C4**로 조정 — 명도차 충분 + 신뢰톤 유지. foliio 상속 teal `#00C5D1`·error pink `#FF60B6`는 deprecate.

```
역할 분리로 충돌 0:
  [브랜드 #1E40C4] ──→ 헤더 / CTA / 로고   (셀에 못 씀)
  [데이터 #3B5BDB] ──→ 히트맵 셀 / 제안선   (헤더·CTA에 못 씀)
```

### 6.2 색의 심리·정당성

| 색 | hex | 의미 | 정당성 |
|---|---|---|---|
| **인디고** | #1E40C4 / #3B5BDB | **신뢰** | 금융·보험의 정석 신뢰색. 딥블루(직방 DNA)에 보랏빛이 섞여 'B2B 전문 SaaS' 인상 — 보험권 밝은 코퍼릿 블루와 시각 분리 |
| **민트** | #12B5A4 | **명료·동행** | "내 것(기존)"을 차갑지 않게. 인디고 단색 피로 완화 + Before/After 좌우 시각분리(강남언니 DNA) |
| **포인트 오렌지** | #F76707 | **액션** | 직방식 액션 포인트. 발굴탭(엄지 영역 하단 중앙) 강조 — 1화면 1회만, 정적 화면의 의도된 단일 활력점 |
| **딥네이비** | #1B2A57 | **신뢰 앵커** | 푸터·헤딩·앱아이콘 배경. 인디고보다 무거운 딥블루 |
| **amber** | #F59E0B | **부족(개선 여지)** | 부족 ≠ 위험. 노란 경고 = "더 채울 수 있다" |
| **red** | #E03131 | **위험(손해)** | 해지 손해 · §97 불리점 · 부당승환. 실손해만 적색 |
| **회색** | #ADB5BD | **공백의 정직함** | 데이터 없음을 회색 점선으로 — 거짓 충족 표기 금지의 시각 언어 |

### 6.3 amber ≠ red — 위협 과장 금지의 시각 언어

```
  부족(amber #F59E0B)  →  "아직 더 채울 수 있어요"   (개선 여지)
  위험(red   #E03131)  →  "해지 시 환급 손해 가능"   (실제 손해)
```

설계사가 고객 앞에서 **공포 마케팅을 못 하게** 색으로 막는다. amber와 red를 섞어 모든 공백을 "위험!"으로 칠하는 순간 정직성 레드라인이 무너진다. 부족은 노랑, 위험은 빨강 — 이 경계는 절대 흐리지 않는다.

## 7. 브랜드 인상 비율 — 신뢰 65 : 혁신 25 : 친근 10

세 축 중 하나만 고르면 **신뢰**다. 단, 낡은 신뢰(보수적 딥블루 일색)가 아니라 **"데이터로 증명하는 신뢰"**(직방형)다.

```
신뢰  ████████████████████████████░░░░░░░░░░░  65%   데이터로 증명 (직방형)
혁신  ██████████░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  25%   AI·속도 양념
친근  ████░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  10%   심리장벽 완화 (의도적 억제)
```

> **친근은 왜 10%로 억제하나?** 친근이 과하면 정직성 레드라인과 충돌한다. "쉽고 편하게 보장 완료!" 같은 캐주얼 과장톤은 신뢰를 깎는다. 친근은 강남언니식 비교 UX의 심리장벽 완화에만 최소 투입.

## 8. 보이스 & 톤앤매너 — 정직성 레드라인을 보이스로 번역

정직 레드라인(안전배지 금지·면책 고정·자동발송 금지)은 규제가 아니라 **인파의 보이스 그 자체**다. 경쟁 보험 마케팅이 전부 "최고/보장/안심" 과장톤일 때, 인파는 **절제된 정직**으로 역설적 신뢰를 얻는다.

### 8.1 4대 보이스 원칙

| 원칙 | 설명 |
|---|---|
| **숫자로 말한다** | 형용사를 줄이고 수치로. "보장 공백 3건 발견" > "완벽 분석" |
| **설계사를 주어로** | 인파가 한 일도 설계사 공으로. "당신이 보낸 안내서를 고객이 열람했습니다" |
| **AI는 항상 '초안'** | 모든 생성물에 "AI 초안 · 최종 확인은 설계사" 면책. 약점 아닌 신뢰 시그널 |
| **면책을 신뢰로 카피화** | "인파는 책임을 넘기지 않는다" — disclaimer를 브랜드 자산으로 |

### 8.2 마이크로카피 — O / X 대조

| 맥락 | ✅ O (절제된 정직) | ❌ X (과장·공포) |
|---|---|---|
| 분석 완료 | 보장 공백 **3건** 발견 | 완벽 분석 완료! |
| 빈 히트맵 셀 | 아직 확인 안 됨 | 보장 없음 위험! |
| 비교안내서 경고 | 해지 시 환급 **손해가 발생할 수 있습니다** (사실 진술) | 지금 안 바꾸면 손해! |
| AI 멘트 | AI 초안 · 최종 확인은 설계사 | AI가 보장하는 최적 설계 |
| 공유 알림 | 고객이 안내서를 **열람**했습니다 | 고객 마음을 사로잡았어요! |

> **핵심**: "해지 시 환급 손해가 발생할 수 있습니다"는 **사실 진술**(가능성 + 조건)이지 공포 마케팅이 아니다. 단정("손해!")이 아니라 가능성("발생할 수 있습니다")으로 말하는 것이 정직성의 문법이다.

## 9. 당근식 작명 확장 — '인파-OO' 한국어 기능 네이밍

당근의 확장력은 친근한 모(母)브랜드 아래 자식 기능에 같은 정체성을 입힌 것이다(당근알바·당근채팅). 인파도 **'인파-OO' 네이밍 시스템**으로 확장한다. 기능명을 영문 기술어가 아니라 **설계사가 입으로 부를 한국어**로 짓는다.

| 기능 | 기술명(내부) | **인파 브랜드명(설계사용)** | 작명 의도 |
|---|---|---|---|
| 담보 히트맵 | coverage-heatmap | **인파 한눈표** | 한눈에 보장 전체를 본다 |
| 갈아타기 비교안내서 | compare-ledger | **인파 비교장** | 갈아타기 '한 장' |
| 홈 액션큐 | action-card | **오늘의 인파** | 오늘 챙길 고객 |
| AI 카톡 멘트 | ai-message-composer | **인파 한마디** | 고객에게 보낼 한마디 |
| 공유링크 귀속 | share-attribution | **인파 소개고리** | 소개가 고리처럼 이어짐(북극성) |

> **해자는 동사화·일상어화다.** 사용자가 동료에게 *"그거 인파 한눈표로 뽑아서 보내"* 라고 말하게 만든다. 북극성이 공유링크 발송·열람·귀속이므로, 공유 동선의 이름 **'인파 소개고리'** 를 특히 입에 붙게 짓는다(오프라인 명함 QR → 공유링크 → 귀속의 진입점).

## 10. 3대 접점 분리 대응 — 노출 거리·시간이 다르다

세 접점(방문영업 화면 · 앱아이콘 · 명함)은 노출 거리와 시간이 전부 다르다. 하나의 마크로 셋을 커버하려면 **극단적 단순화**가 답이다.

### 10.1 방문영업 화면 (고객이 옆에서 봄) — 로고 후퇴

```
┌──────────────────────────────────────┐
│ inpa(회색조)            🔔  크레딧 ∞   │ ← 헤더 로고: 작게·회색조 후퇴
├──────────────────────────────────────┤
│                                       │
│      [ 히트맵 데이터 = 주인공 ]        │ ← 고객이 보는 건 '내 보장 한 장'
│                                       │
└──────────────────────────────────────┘
```

- **로고가 콘텐츠를 가리면 안 된다.** 주인공은 히트맵 데이터(무신사 절제).
- 고객이 보는 건 '인파 광고'가 아니라 '내 보장 한 장'.
- **공유뷰(share_token)는 헤더 자체를 숨김** + 면책 푸터 고정. (foliio 공유뷰 헤더 숨김 토큰 `$global-header-h`와 일치 — 회귀 트랩 주의)

### 10.2 앱아이콘 — 딥네이비 단색 정사각

```
   ┌─────────────┐
   │             │   배경: 딥네이비 #1B2A57
   │   ╭──────╮  │   심볼: 흰 Twin-Arc
   │  ╭────────╮ │   텍스트: 0
   │     ●       │
   │             │   → 홈화면 레드/블루 아이콘 사이에서 즉시 분리(각인)
   └─────────────┘
```

- 딥네이비 #1B2A57 배경 + 흰 Twin-Arc, **텍스트 0**.
- 보험권 레드/블루 아이콘 포화 속에서 **딥 인디고 정사각**이 즉시 분리됨.
- 다크모드 대응: 배경만 invert.

### 10.3 명함 (종이·1:1 핵심 무기) — co-brand + 공유 QR

- 인파는 설계사 명함에 **공동 브랜딩(co-brand)** 된다 — `powered by inpa` + 공유 QR.
- 단색이라 **흑백 출력·금박·엠보싱** 어디에도 적용 가능(딥네이비가 금박 대체).
- 명함이 곧 **'인파 소개고리'의 오프라인 진입점**(QR → 공유링크 → 귀속).
- **컬러보다 QR 동선을 우선 배치** — 명함의 목적은 장식이 아니라 귀속 진입.

## 11. 브랜드 = 세 청중 동시 약속

브랜드는 장식이 아니라 **세 청중에게 동시에 거는 약속**이다.

| 청중 | 약속 | 메커니즘 |
|---|---|---|
| **사업(BM)** | "이건 내 무기다" | 설계사가 자기 돈으로 매달 결제 → 파트너 포지셔닝 = 정서적 **락인**(결제 이탈 방어) |
| **투자** | "분석 툴 → 영업 OS 피벗" | 일관된 디자인 시스템 = 실행력 증거. 브랜드 리뉴얼이 **피벗을 시각적으로 증명** |
| **설계사** | "고객 앞에서 전문가로 보인다" | 우리 화면 = 그의 **전문성 배지**. 과장톤(안전배지·"AI 보장")은 그의 신뢰를 깎음 |

> **정직성 레드라인은 비용이 아니라 자산이다.** 보험은 불신 도메인. 모두가 과장할 때 **과장 안 하는 절제**가 곧 차별화다. 브랜드의 주인공은 끝까지 **인파를 든 설계사**이고, 도구는 후퇴한다.
