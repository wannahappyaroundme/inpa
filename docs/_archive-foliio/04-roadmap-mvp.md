# 문서 4: 로드맵 & MVP

> Foliio를 '분석 계산기'에서 '한국형 영업지원 운영체제'로 재배치하기 위한 단계별 실행 로드맵.
> 핵심 원칙: **이미 깔린 데이터 파이프라인(OCR→8케이스→공유링크) 위에 행동 레이어만 얹는다. 신규 개발 최소, 자산 재활용 최대.**

---

## 0. 이 문서를 읽는 법 (PM용 30초 요약)

| 질문 | 답 |
|---|---|
| MVP는 몇 개 기능인가? | **7개.** 좁게 잡았다. "분석을 행동으로 바꾸는 최소 고리" + "그걸 켜기 위한 법적·컴플라이언스 선결과제"만 묶었다. |
| 왜 이 7개인가? | 분석은 이미 있다. MVP는 분석 결과를 **카톡 멘트 + 만기 후속 큐**로 바꾸는 데까지만. 단, 그 전에 **가드레일 + 국외이전 동의**가 없으면 출시 자체가 불가하므로 MVP에 강제 포함. |
| 바이럴(소개 루프)·리퍼럴·파이프라인 뷰는? | **Phase 2.** 데이터 모델 변경 + 법무 선결(제3자 수집)이 걸려 MVP에서 뺐다. |
| 브랜딩·SNS·라이브 모드는? | **Phase 3 / Later.** 파이프라인 신뢰가 쌓인 뒤 얹는 후행 기능. |
| PM이 지금 결정해야 할 것은? | §7 오픈 퀘스천 9개. 특히 **표준 보장 기준선 출처**와 **민감정보 국외이전 동의**는 AI 기능 전체의 출시 가부를 가른다. 코딩 전에 답이 나와야 한다. |

## 1. MVP 범위 — 좁고 현실적으로

### 1.1 MVP에 포함된 7개 (그리고 뺀 이유)

MVP의 목표는 한 문장이다: **"증권 한 장을 넣으면, 검수만 거쳐 카톡 멘트가 나오고, 만기 고객이 자동으로 오늘의 할 일에 뜬다."** 그 이상은 다음 단계로 미룬다.

| # | 기능 | 어느 기둥 | impact / effort | MVP에 넣은 이유 |
|---|---|---|---|---|
| M1 | 사전준비형 원클릭 니즈환기 (Analysis-to-Kakao) | 분석-콘텐츠 파이프라인 | high / **low** | 90% 재활용. 분석을 카톡 멘트+공유링크로 바꾸는 핵심 고리. 데일리 훅. |
| M2 | AI 보장공백 진단 + 상담 내러티브 | 분석-콘텐츠 파이프라인 | high / med | "왜 공백인지"를 자동 구성. 신입도 분석을 해석. **단 표준 기준선이 숨은 난관(§6).** |
| M3 | 보험 컴플라이언스 가드레일 (공유 검열 미들웨어) | 컴플라이언스 안전망 | high / med | **이게 없으면 다른 AI 기능 전부 출시 불가.** 모든 생성 텍스트의 단일 게이트. |
| M4 | 만기·갱신 자동 워치독 (리텐션 엔진) | 리텐션 엔진 | high / med | ROI 최상 구간. 데이터·cron·발송 인프라 이미 보유. 트리거+큐만 신규. |
| M5 | AI 카톡 메시지 엔진 (목적 enum + 가드레일) | 분석-콘텐츠 파이프라인 | high / med | 5개 페르소나가 재발명하던 멘트 생성을 **단일 엔진으로 통합.** 리텐션·소개가 이걸 호출. |
| M6 | 증권 다건 일괄 OCR + 보장 매핑 강화 | 분석-콘텐츠 파이프라인 | med / **low** | 미팅 전 증권 취합 수기 제거. 알려진 오매칭·double-count 트랩 정리. |
| M7 | PII 동의·민감정보 국외이전 동의 동선 | 컴플라이언스 안전망 | high / med | **병력=민감정보. 국외이전 동의 없으면 AI 기능 출시 법적 불가.** 전 기능 선결. |

### 1.2 MVP에서 의도적으로 뺀 것 (스코프 보호)

- [ ] **바이럴 진단 루프 / 리퍼럴 체인** → Phase 2 (제3자 수집 법무 선결 + `source`/`referred_by` 신규 필드)
- [ ] **파이프라인 카드피드 / lead_stage 자동전이** → Phase 2 (상태머신 + 자동 ActivityLog 필요)
- [ ] **비교 매트릭스 → 상담 내러티브** → Phase 2 (비교광고 규제 리스크, 배포용 자동생성 금지)
- [ ] **알림톡 사전심의 발송** → Phase 2 (kakao_biztalk.py pending, 심사·수신거부 거버넌스)
- [ ] **AI 비용 게이팅 + North Star 대시보드** → Phase 2 (단, MVP에서 **AI 호출 크레딧 차감의 최소 골격**은 깔아둔다 — §6 리스크)
- [ ] **후기 수집 / 성과 브랜딩 / SEO 콘텐츠 / 라이브 모드 / 콜드스타트** → Phase 3 / Later

### 1.3 MVP 정직성 원칙 (거짓 전제 금지)

> 마케팅 카피에서 절대 약속하지 않는다:

- [ ] **"원탭 자동발송" 금지.** 카카오 인프라상 개별 멘트는 **클립보드 복사 + 카톡 앱 열기**가 한계. 정직하게 "복사" 기본으로 출시.
- [ ] **"심의 안전 배지" 금지.** 가드레일은 "1차 보조, 최종책임 설계사"로만 포지셔닝. 보증 책임을 떠안지 않는다.
- [ ] **"라이브 현장 OCR" 금지.** OCR 오류·지연이 현장 신뢰를 즉사시키므로 **"사전 준비"**로 포지셔닝.

## 2. Phase별 단계 계획 & 산출물

> 각 Phase는 "출시 가능한 단위"로 끊는다. Phase 0(기획·법무)은 코드 0줄이지만 **MVP 출시의 게이트**다.

### Phase 0 — 선결과제 (코드 전, 법무·데이터 정의)

이 단계가 끝나지 않으면 M2·M3·M7은 착수 불가. **PM이 가장 먼저 닫아야 할 단계.**

- [ ] 표준 보장 기준선 **출처·면책 문구 확정** (금감원 / 보험연구원 / 자체 컨센서스 중 택1 + 법적 면책)
- [ ] 민감정보(병력) **국외이전 동의** 법무 검토 → 현 customer-agree 커버 여부 판정
- [ ] 보험업법 준법감시 자문 → 가드레일 **면책 카피·동의 로그 구조** 확정
- [ ] 가드레일 **위반 카피 골든셋 100건** 수집 (회귀 테스트 기준선)

**산출물:** 기준선 데이터셋 사양서 / 동의서 개정안 / 가드레일 룰셋 v0 / 골든 100건

## 3. 기존 자산 재활용 맵 (무엇을 그대로 / 개조 / 신규)

> 핵심 메시지: **MVP의 대부분은 "이미 있는 것"이다.** 신규 개발은 프롬프트·게이트·UI 중심으로 얇게 얹는다.

| 자산 | 어디에 쓰이나 |

## 4. 요금제 / 수익화 재설계 (영업지원 가치 정렬)

### 4.1 가격 철학의 전환

> **기존:** "분석을 잘 해주는 값을 받는다."
> **전환:** "**분석은 무료로 풀고, '발송·자동화·귀속'을 과금한다.** 가치가 행동에 있음을 가격이 증명한다."

분석을 무료로 개방하는 이유는 두 가지다. (1) 분석 계산기는 이미 경쟁사가 흉내낼 수 있는 영역이라 과금 방어력이 약하다. (2) **무료 분석 = 바이럴 진단 루프의 연료** — 공유링크가 많이 퍼져야 소개 리드가 들어온다.

### 4.2 4-tier 재설계

| 티어 | 가격 | 포함 기능 | 포지셔닝 |
|---|---|---|---|
| **Basic** | 무료 | 증권 OCR · 8케이스 분석 · 공유링크 생성 (= 현 분석 계산기) | **미끼.** 바이럴 루프 연료. |
| **Plus** | 19,900원 | + AI 카톡 메시지 엔진 · 진단 내러티브 · 사전준비 니즈환기 | **데일리 훅** — 매일 켜는 이유. |
| **Premium** | 89,900원 | + 리텐션 워치독 자동 큐 · 바이럴 추천코드 귀속 · 리퍼럴 체인 · 성과 브랜딩 · 알림톡 발송 | **영업 복리·성장 엔진 = 진짜 돈 버는 구간.** |
| **Super** | 견적 | + 팀/지점 일괄 도입 · 관리자 대시보드 · B2B 팀 바이럴 | 지점장·GA 단위 도입. |

### 4.3 과금 4대 원칙 (체크리스트)

- [ ] **분석은 무료, 발송·자동화·귀속은 유료.** 가격이 "행동에 가치가 있다"를 증명.
- [ ] **AI 호출은 monthly_credit 차감 연결.** 비용 누수 차단, Premium은 크레딧 대폭 상향.
- [ ] **컴플라이언스 가드레일은 전 티어 공통 위생.** 과금 차별화 X — 면책 책임만 명시. (안전을 돈 받고 파는 구조 금지)
- [ ] **워터마크 강제 과금 모델 금지.** 설계사가 싫어해 이탈. **추천코드 귀속이 진짜 바이럴 수익원.**

### 4.4 가격-기능 매핑의 단계 연동

| 단계 | 과금 상태 |
|---|---|
| 베타 (현재) | 무료 (`SUPER_BETA_EXPIRY=2099-12-31` placeholder), `FREE_TIER_UNLIMITED=True` |
| MVP 출시 | 베타 유지하되 **AI 호출 크레딧 차감 계측 시작** (전환 데이터 수집) |
| Phase 2 이후 | 정식 출시 시 4-tier 활성화 — Premium에 워치독·귀속·알림톡 집중 |

> ⚠️ 전환율 가설(§7 Q7)은 **베타 데이터로 검증 후** 가격을 확정한다. "분석 무료 → Plus/Premium 전환"이 실제 지불의사로 이어지는지가 핵심.

## 6. 숨은 난관 경고 (PM이 일정에 반영할 것)

> "low effort"로 보이지만 실제로는 리스크가 큰 지점. 일정 산정 시 버퍼를 둔다.

- [ ] **표준 보장 기준선 데이터셋은 effort 폭탄이다.** 코드는 단순하지만 **"누가 어떤 권위로 적정 보장을 정의하는가"**가 법적·기획적 난제. Phase 0에서 닫지 못하면 M2 전체가 지연.
- [ ] **가드레일은 "med effort"지만 골든 100건 수집이 병목.** 위반 카피 샘플 확보가 코드보다 오래 걸릴 수 있음.
- [ ] **국외이전 동의는 법무 외부 의존.** 코드는 동의 항목 추가뿐이지만 **법무 검토 결과가 출시 가부를 결정** — 일정의 critical path.
- [ ] **AI 크레딧 차감을 MVP에서 미루지 말 것.** 베타 무료라도 계측을 시작하지 않으면 정식 출시 시 단위경제를 못 세운다.

## 7. 오픈 퀘스천 — PM 결정 필요 (코딩 전 닫을 것)

> 각 질문 옆 **[차단 기능]**은 "이 답이 안 나오면 못 만드는 것"이다. 우선순위 순.

- [ ] **Q1. [M2 진단 출시 가부] 표준 보장 기준선을 누가 어떤 권위로 정의하는가?**
  연령/성별/직업위험별 적정 보장 밴드의 출처(금감원 가이드 / 보험연구원 / 자체 설계 컨센서스)와 면책 문구 확정 필요.
- [ ] **Q2. [전 AI 기능 출시 가부] 민감정보(병력) 국외이전 동의를 현 customer-agree로 커버 가능한가?**
  아니면 별도 동의서·국내 처리 대안(온프레미스/마스킹 후 전송)이 필요한가? 법무 검토 결과가 전체 출시 가부 결정.
- [ ] **Q3. [Phase 2 알림톡] 알림톡 사전심의로 만기/생일 정형 알림이 승인 가능한 범위는?**
  kakao_biztalk.py pending을 언제 풀고, 가변 AI 카피와 정형 알림톡의 경계를 어디에 긋는가?
- [ ] **Q4. [Phase 2 바이럴] 지인 B 셀프진단의 제3자 수집 동의를 어떻게 1탭으로 받는가?**
  share_token 만료/회수/조회로그/robots 차단을 어느 수준으로 구현하는가?
- [ ] **Q5. [M3 가드레일] "심의 안전 배지" 대신 어떤 면책 카피·동의 로그 구조가 Foliio를 법적으로 보호하는가?**
  보험업법 준법감시 전문가 자문 필요.
- [ ] **Q6. [North Star 계측] 발송이 클립보드 복사 기반일 때 발송 이벤트를 어떻게 정확히 계측하는가?**
  "미팅당 공유링크 발송수 × 열람률"의 발송 시점 측정 방식 정의.
- [ ] **Q7. [수익화 검증] 분석 전면 무료 시 Plus/Premium 전환율 가설은?**
  "분석=무료, 발송/자동화=유료" 매핑이 실제 지불의사로 이어지는지 베타 데이터로 검증.
- [ ] **Q8. [Phase 3 우선순위] 라이브 프레젠테이션 모드 수요가 실제 차별화 가치가 있는가?**
  사전준비형만으로 충분한지 상위 설계사 인터뷰로 확인 후 Phase3 우선순위 재조정.
- [ ] **Q9. [Phase 3 우선순위] 신입 콜드스타트 부트스트랩이 실제 첫 리텐션을 끌어올리는가?**
  아니면 "진짜 고객 데이터"가 있어야만 aha가 발생하는가? — 마케팅이 가장 절박한 신입층 이탈과 직결.
