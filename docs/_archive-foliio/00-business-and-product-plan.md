### 1.2 문제 — 진짜 병목은 "설득"이 아니라 "찾기"

위촉직 38만 설계사(생·손보 협회 공시 기준, 추정)는 quota 압박 속에서 **엑셀·수기·범용 GPT**로 영업한다. 세 가지가 동시에 깨져 있다.

- **발굴 공백** — 발굴 1위는 DB영업(갈아타기). 그러나 비교 작업을 엑셀로 손수 하면 담보 누락·중복이 일상.
- **규제 공포** — 갈아타기엔 **부당승환(보험업법 §97)** 리스크가 따라붙어, 발굴 동기 자체를 얼어붙게 만든다.
- **도구 부재** — GA 본사 사내툴은 전속·폐쇄라 개인이 못 산다. 범용 GPT는 담보 정규화·8케이스 계산·컴플라이언스 게이트가 없어 **부당승환 카피를 그대로 토한다(리스크 증폭)**.

> 핵심 통찰: **공포를 구매 이유로 전환한다.** 부당승환이 무서워서 발굴을 못 하던 설계사에게, 자동 비교안내서를 **합법 방패**로 쥐여준다. 이것이 단일 최강 wedge.

```
 ┌────────┐   →    ┌────────────────┐   →   ┌──────────┐   →   ┌──────────────┐   →   ┌────────────────┐
```

### 2.1 한 줄 요약

> **38만 위촉직 설계사가 매달 quota 공포 속에서, '발굴'과 '합법적 갈아타기'라는 두 병목을 엑셀·수기·범용 GPT로 버티고 있다. Foliio는 이 둘을 도메인 데이터 + 컴플라이언스 가드레일로 동시에 푸는 첫 개인용 영업 OS다.**

### 2.3 핵심 문제 — 설계사의 3대 병목

설계사 현장에서 검증한 결과, 통증은 '설득'이 아니라 그 **앞단계**에 몰려 있다.

#### 문제 ① — 매달 리셋되는 Quota 공포 (리텐션 엔진)

```
  1일 ──────────────── 말일
  │                      │
  실적 0에서 시작     "이번 달 몇 건 했어?"
  │                      │
  └─ 매달 처음부터 ──────┘   ← 누적이 안 되는 구조적 압박
```

- 위촉직은 **개인사업자**다. 고정급이 아니라 매달 0에서 시작하는 성과급 구조 → quota 미달 = 소득 0 = 생존 위협.
- 이 공포가 **월말 한도 소진 → 유료 전환 골든타임**으로 직결된다. 제품이 quota를 '덜어주는 무기'로 인식되는 순간, 가격 저항이 사라진다.

#### 문제 ② — 발굴(Lead-gen) 병목 (진짜 1번 통증)

- 신입의 죽음의 계곡은 "팔 줄 몰라서"가 아니라 **"만날 사람이 없어서"**다.
- 발굴 1위 채널 = **DB영업(기존 계약 갈아타기/승환)**. 그런데 여기에 **부당승환(보험업법 §97)**이라는 공포가 깔려 있다 → 잘못하면 제재.
- 결과: 가장 효과적인 발굴 채널을 **공포 때문에 못 쓰거나, 모르고 쓰다 사고**친다.

> **Wedge 포인트**: Foliio의 **기존 vs 제안 비교엔진 + 비교안내확인서 자동생성**이 이 공포를 *방패*로 전환한다. 규제를 회피 대상이 아니라 **구매 이유**로 바꾸는 것 — 이것이 우리의 단일 최강 진입 쐐기다.

#### 문제 ③ — '틀(Framework)'의 부재

- 신입은 고객 보장을 **체계적으로 진단할 표준 틀이 없다.** 엑셀에 손으로 담보를 옮겨 적다 누락·중복이 발생한다.
- 보험사마다 담보명이 다르다(삼성 "암진단급부금" = 교보 "암진단보험금"). **정규화 사전 없이는 비교 자체가 거짓말**이 된다. 특히 유사암(갑상선·경계성) 포함 여부, 뇌혈관(I60–69) vs 뇌졸중(I63–64) 같은 **보장범위(KCD 코드)** 차이를 못 잡으면 잘못된 제안이 나간다.
- Foliio는 **100+ 담보 표준 틀 + 보험사별 alias 정규화 사전 + 보장범위 플래그**로 "30분 만에 첫 제안서"를 가능케 한다.

#### 문제 요약표

| 병목 | 현장 증상 | 기존 대응(한계) | Foliio 해법 |
|---|---|---|---|
| ① Quota 공포 | 매달 0에서 시작, 월말 압박 | 정신력·야근 | 발굴·제안 자동화로 시간당 처리량↑ |
| ② 발굴 병목 | 만날 사람 없음 / 승환 공포 | DB영업 회피 or 무지성 강행 | 비교엔진 + §97 비교안내서 = 합법 무기 |
| ③ 틀 부재 | 엑셀 누락·중복·담보명 혼동 | 수기·범용 GPT | 100+ 담보 정규화 사전 + 보장범위 플래그 |

### 2.4 타겟 세그먼트

핵심 타겟은 **원수사·GA에 위촉된 개인사업자(개인 결제)**. 본사가 사주는 사내툴이 아니라, **설계사 개인이 자기 돈으로 사는 무기**를 판다. 이 점이 경쟁사(GA 사내툴)와의 근본적 분기점이다.

#### 세그먼트 분해

```
┌──────────────────────────────────────────────────────────┐
│ 위촉직 설계사 (개인사업자 · 개인 결제)                       │
│                                                            │
│  ┌─────────────────────┐   ┌──────────────────────────┐  │
│  │ 신입 (≤ 3년차)        │   │ 중견 (4년차 ~)            │  │
│  │ 핵심 통증 = 발굴      │   │ 핵심 통증 = 관리·리텐션   │  │
│  │ "만날 사람·틀이 없다" │   │ "묵힌 DB·만기 놓침"       │  │
│  └─────────────────────┘   └──────────────────────────┘  │
└──────────────────────────────────────────────────────────┘
```

| 항목 | **신입 (Primary)** | **중견 (Secondary)** |
|---|---|---|
| 연차 | 입문 ~ 약 3년 | 4년차 이상 |
| 핵심 통증 | **발굴** — 만날 사람·진단 틀 부재 | **관리·리텐션** — 묵힌 DB, 만기·갱신 누락 |
| Aha 모먼트 | 소개받은 1명 증권 촬영 → 5분 내 "보장공백" 시각화 | DB 일괄 스캔 → 공백·만기 워치독 알림으로 재방문 명분 |
| 매일 쓰는 기능 | 원클릭 니즈환기 / 보장공백 진단 / 정규화 비교표 | 만기·갱신 워치독 / DB 공백 스캐너 / 비교안내서 |
| 가격 적합 | Plus 19,900 (커피 4잔 체감가) | Premium 89,900 (DB 스캐너 전용가 포지셔닝) |
| 획득 채널 | 교육원·자격학원(입문 시점) / 단톡방 바이럴 | 지점 조회 / 추천 리더보드 |
| 진입 동선 | 빈손에서 첫 제안서 30분 | 잠자던 고객 자산화 |

> **생명 vs 손해 분리**: 손해(실손·자동차·운전자)는 진입 미끼·갈아타기 빈도가 높아 신입 발굴 채널로 적합. 생명(종신·사망담보)은 거절 1순위라 신입에게 과대평가 금물 — 중견 리텐션(납입완료·실효 알림) 쪽이 금광이다.

> **안티-타겟(과투자 경계)**: 퍼스널 브랜딩·성과카드 자랑·14종 판촉템플릿은 *신입은 자랑할 실적이 없고 중견은 쓰지 않는다.* 후행 기능으로 두고, 초기 리소스를 발굴·진단·컴플라이언스에 집중한다.

### 2.5 경쟁 지형

#### 경쟁 지형 한눈에 — 우리가 서 있는 빈칸

```
                도메인 깊이 (담보 정규화·8케이스 계산)
                  낮음 ◀──────────────────────▶ 높음
        개인 │                                   │
        구매 │  범용 GPT     엑셀                 ★ Foliio

             │                                   │  + 행동 트리거 교집합)
   ──────────┼───────────────────────────────────┼──────────
        본사 │                          굿리치     │
        전속 │                          인카        │
        폐쇄 │                          보험클리닉  │
             │                                   │
```

> **빈칸의 의미**: '개인이 살 수 있다' × '보험 도메인이 깊다' × '컴플라이언스 가드레일이 있다'의 교집합에 선 플레이어가 현재 없다. 이 빈칸이 곧 우리의 진입 공간이다.

#### 경쟁사 상세 비교표

| 경쟁자 | 정체 | 강점 | 치명적 빈틈 | **Foliio 차별점** |
|---|---|---|---|---|
| **굿리치** | GA 본사 사내툴 | 자사 DB·시책 연동, 전속 조직 보급 | **전속·폐쇄** — 개인이 못 사고, 타 GA 설계사 못 씀 | 소속 무관 **개인 결제**. 어느 GA든 자기 무기로 휴대 |
| **인카** | GA 본사 사내툴 | 대형 조직 인프라 | 동일 — 폐쇄, 개인 미판매 | 동일 — 개인 소유, 이직해도 따라옴 |
| **보험클리닉** | 상담 플랫폼/사내툴 | 상담 동선·브랜드 | 폐쇄·소속 종속, 개인 발굴 무기 아님 | **발굴 무기**로 포지션 — 분석 끝점을 영업 시작점으로 |
| **엑셀** | 범용 스프레드시트 | 무료·자유도 | **담보 정규화·틀 없음** → 누락·중복·승환 리스크 | 100+ 담보 정규화 사전 + 보장범위 플래그로 비교 정확성 보장 |
| **캔바(Canva)** | 범용 디자인 툴 | 예쁜 결과물·템플릿 | **보험 도메인 0** — 진단·계산·컴플라이언스 전무 | 도메인 데이터가 채워진 템플릿 + 자동 KPI |
| **범용 GPT** | 범용 LLM 챗봇 | 화법 생성·범용성 | 담보 정규화·8케이스 계산·**컴플라이언스 게이트 없음** → 부당승환 카피 그대로 토함(법적 리스크) | 도메인 + **§97 가드레일(정규식→Haiku→사람 3중)** + 위반 골든셋 회귀로 합법 보증 |

#### 차별점 = 3중 해자 (요약)

1. **담보 정규화 사전** — 보험사별 alias + 보장범위(KCD) 매핑. OCR·분석·승환비교의 전제. 데이터가 쌓일수록 두꺼워지는 **복리 자산**. 카피에 6개월+ 소요 추정.
2. **100+ 담보 표준 틀** — 경쟁 엑셀·범용툴의 '몇 항목' 비교를 압도하는 진단 깊이.
3. **컴플라이언스 가드레일** — 부당승환 자동 차단 + 비교안내서 자동생성. 범용 GPT가 절대 못 넘는 **법무 진입장벽**이자 신뢰 자산.

#### 우리만의 교집합 체크리스트

- [x] 소속 무관 **개인이 구매**할 수 있다 (vs 굿리치·인카·보험클리닉)
- [x] **보험 도메인**이 깊다 — 담보 정규화·8케이스 계산 (vs 엑셀·캔바·GPT)
- [x] **컴플라이언스 게이트**가 있다 — §97 비교안내서 자동화 (vs 엑셀·GPT)
- [x] **행동 트리거**로 연결된다 — 진단 → 화법 → 발송 한 동선 (vs 정적 분석툴)

> **결론**: Foliio는 "규제(부당승환)를 회피 대상이 아니라 *기능*으로 바꿔, 38만 시장에 합법적으로 진입한 유일 플레이어"다. 도메인 + 컴플라이언스 + 행동 트리거의 교집합은 현재 비어 있고, 데이터 사전의 복리 효과가 시간이 갈수록 격차를 벌린다.

## 3. 솔루션·포지셔닝·6대 필러·차별점

> 이 섹션의 모든 시장·재무 수치는 **추정치**이며, 협회 공시·내부 가정 기반임을 명시한다. 확정 수치는 §시장분석·§재무계획을 따른다.

---

### 3.1 솔루션 개요 — "분석의 끝점을 영업의 시작점으로"

기존 Foliio는 증권을 받아 **분석으로 끝났다**. 영업지원 에디션의 단일 명제는 그 끝점을 **행동의 시작점**으로 옮기는 것이다.

```
[ 기존 Foliio: 분석 = 종착역 ]
  증권 OCR ─→ 담보 정규화 ─→ 보장공백 진단 ─→ [분석 리포트] ⏹ 끝

[ 영업지원 에디션: 분석 = 트리거 ]
  증권 OCR ─→ 담보 정규화 ─→ 보장공백 진단 ─→ AI 화법 생성 ─→ 비교안내서 자동생성 ─→ [발송·계약] ▶
                  └────────────── 한 동선, 컴플라이언스 게이트 상시 동반 ──────────────┘
```

핵심은 **병목의 재정의**다. 위촉직 설계사의 진짜 병목은 '설득'이 아니라 **'찾기(발굴)'**다. 발굴 1위 채널인 DB영업(계약 갈아타기)을, Foliio가 이미 보유한 `기존 vs 제안` 비교엔진(8케이스 계산 + `share_token` 공유)으로 그대로 무기화한다. 단, 갈아타기에는 **부당승환(보험업법 §97)** 이라는 공포가 따라붙는다 — 우리는 이 공포를 **자동 비교안내확인서 = 합법 방패**로 전환해 구매 이유로 바꾼다. 이것이 단일 최강 wedge다.

**한 줄 가치제안**: "갈아타기를 보완설계로, 공백진단을 5분 안에, 숫자로." — 신입은 고객이 0명이어도 지인의 증권 1장으로 30분 내 첫 합법 제안서를 만든다. 중견은 쌓인 고객 DB를 야간 배치로 스캔해 '오늘 케어할 N건'을 자동으로 받는다.

#### 솔루션을 떠받치는 3개의 자산 레이어

| 레이어 | 내용 | 출처 |
|---|---|---|
| **데이터 파이프라인** | 증권 OCR(Claude→pdfplumber→PyMuPDF) → 담보 정규화 사전 → 8케이스 보험료 계산 → 보장공백 진단 | 기존 Foliio 재활용 (불변) |
| **행동 레이어 (신규 얇은 층)** | AI 화법 엔진 · 갈아타기 비교안내서 생성 · 케어 큐 · 워치독 알림 | 프롬프트·게이트·UI만 신규 |
| **위생 레이어 (전 티어 공통)** | 컴플라이언스 가드레일(정규식→Haiku→사람) · PII 국외이전 동의 · §97 면책·자필서명 동선 | content_filter 개조 + 동의 동선 신규 |

> 제품 원칙: **신규 개발은 프롬프트·게이트·UI의 얇은 레이어만 얹고, 검증된 데이터 파이프라인은 건드리지 않는다.** 이것이 빠른 출시와 회귀 안전성을 동시에 보장한다.

---

### 3.2 포지셔닝 — 분석판(베테랑) vs 영업지원 에디션(일반/신입)

Foliio는 하나의 데이터 파이프라인 위에 **두 개의 제품 인격**을 올린다. 동일 엔진, 다른 진입점·다른 동선이다.

| 축 | **기존 Foliio (분석판)** | **영업지원 에디션** |
|---|---|---|
| 핵심 사용자 | 베테랑·분석 지향 설계사 | 일반/신입 (발굴 절박형) |
| 사용자 JTBD | "보유 고객을 정밀하게 분석·관리하고 싶다" | "이달 quota를 채울 **첫 계약/리드**를 만들고 싶다" |
| 분석의 위치 | **종착역** (리포트가 산출물) | **트리거** (행동·발송이 산출물) |
| 입력 전제 | 이미 확보된 고객 DB | 고객 0명에서 시작 가능 (콜드스타트) |
| 핵심 동선 | 증권 → 분석 리포트 → 상담 | 증권 → 진단 → **AI 화법·비교안내서** → 발송 |
| 대표 화면 | 분석 대시보드·레이더 차트 | 보장공백 카드 · 갈아타기 비교표 · 케어 큐 |
| North Star | 분석 건수 | **검수·발송된 영업 액션 수** |
| 부담 지점 | 학습 곡선 (정밀함) | **법적 공포(§97)** → 가드레일로 해소 |

#### 세그먼트별 진입 wedge

```
┌─────────────────────────────────────────────────────────────┐
│  신입 (1순위 타겟)                                            │
│   콜드스타트 → 지인 증권 1장 → 보장공백 진단 5분 → "구멍" 시각화 │
│   = aha 모먼트.  화법·틀 부재를 talk_track 이 대체.            │
│   wedge: 갈아타기 비교안내서 (빈손에서 첫 합법 제안)          │
├─────────────────────────────────────────────────────────────┤
│  중견 (2순위 타겟)                                            │
│   보유 DB 야간 배치 스캔 → '오늘 케어 N건' 큐 자동 생성       │
│   = 실효·이탈 방어 + 객단가 업셀 리드.                        │
│   wedge: 보장공백 스캐너 (Premium / DB 스캐너 전용가)         │
└─────────────────────────────────────────────────────────────┘
```

> **현장 검증 주의**: 베테랑이 즐기는 정밀 분석(레이더 차트 등)은 **고객 설득에는 잘 먹히지 않는다.** 현장의 설득 언어는 "월 3만원 더 내면 암진단비 3천→5천"처럼 **막대 + 금액 숫자**다. 영업지원 에디션의 고객 대면 화면은 레이더가 아니라 **막대 타임라인 + 통화값**을 1순위로 배치한다(기존 analysis/compare uplift의 막대 타임라인 자산 재활용).

### 3.3 6대 필러 (제품 골격)

영업지원 에디션의 모든 기능은 6개 필러로 정렬된다. 각 필러는 퍼널 단계와 1:1 또는 N:1로 매핑된다.

| # | 필러 | 한 줄 정의 | 대표 기능 | MVP 포함 |
|---|---|---|---|---|
| **①** | **발굴·니즈환기** | 분석을 행동 트리거로 — 공백을 찾고 화법으로 환기 | 보장공백 진단 · 원클릭 니즈환기 · AI 카톡 메시지 엔진 · 다건 OCR | **✅ 코어** |
| **②** | **합법 전환(승환)** | 갈아타기를 §97 안전판 위에서 합법·정직하게 | **갈아타기 자동 비교안내서** · 컴플라이언스 가드레일 | **✅ 코어** |
| **③** | **리텐션·케어** | 만기·갱신·실효를 놓치지 않는 자동 워치독 | 만기·갱신 워치독 · 보장공백 스캐너(배치) | **✅ 코어** |
| **④** | **바이럴·소개** | 진단 경험을 소개·리퍼럴 체인으로 확산 | 바이럴 진단 루프 · 리퍼럴 귀속(`referred_by`) | ⬜ P2 |
| **⑤** | **퍼스널 브랜딩** | 설계사 개인 신뢰 자산화 (판촉물·후기) | 카드피드 · 후기카드 · 14종 판촉 템플릿 | ⬜ P2~Later |
| **⑥** | **측정·성장** | North Star 기반 행동 계측과 그로스 루프 | North Star 대시보드 · 전환 퍼널 계측 · SEO | ⬜ P2~Later |

#### 필러별 우선순위 근거

- **①②③ = MVP 코어**: 발굴이 병목이므로 ①(찾기)·②(합법 전환)·③(케어 리텐션)을 첫 출시 중심에 둔다. 단 ②는 가드레일·PII 동의·§97 면책 동선 없이는 **출시 불가** → 위생 4종(가드레일·동의·워치독·정규화 OCR)이 강제 동반된다.
- **④ 바이럴**: 제3자 정보 수집이 따라붙어 **법무 미결** → 의도적으로 P2로 미룬다.
- **⑤ 브랜딩**: 현장 검증상 **신입은 자랑할 실적이 없고 중견은 잘 안 쓴다.** 과투자 위험 영역 → 후행 배치. 14종 판촉 템플릿 자산은 보유하되 마케팅용으로만.
- **⑥ 측정**: North Star = **"주간 활성설계사당 검수·발송된 영업 액션 수"**(비교안내서 + 화법 + 케어 발송 합산, 클립보드 복사 이벤트로 계측). 분석 건수가 아니라 **행동**을 가치로 정의한다.

```
퍼널 ↔ 필러 매핑
  발굴 ─── ①  ┐
  니즈 ─── ①  ├─ MVP 코어 (찾기→합법전환→케어)
  전환 ─── ②  │
  리텐션 ── ③ ┘
  소개 ─── ④  ┐
  브랜딩 ── ⑤ ├─ P2+ (그로스 레이어)
  측정 ─── ⑥  ┘
```

### 3.4 해자(Moat) — 카피 불가능한 3중 방어선

경쟁자가 외형(UI·기능 리스트)은 6주면 모방하지만, 아래 3중은 **데이터·법무·시간**이 결합돼 카피 비용이 비대칭적으로 크다.

#### 해자 1 — 전체 담보 '틀' + 데이터 정규화 사전

범용 GPT·엑셀·캔바가 절대 못 만드는 핵심 자산.

- **표준 담보 틀**: 경쟁/사내툴의 7-factor 비교를 압도하는 100+ 담보 표준 틀. 단, 현장 검증상 고객 대면에는 **3대진단비(암·뇌·심장) + 수술비 + 입원일당 + 실손 + 사망**의 코어 8~10개를 1순위로 노출하고, 100개는 백엔드 정규화·내부 비교용으로 둔다. (100개를 다 들이밀면 고객이 도망간다.)
- **정규화 사전(alias 매핑)**: 보험사별 담보명 별칭을 표준 담보로 매핑. OCR·분석·승환비교 **모든 기능의 전제** → 사전이 없으면 비교가 작동하지 않는다.
- **⚠ 정규화 정밀성 — 거짓 비교 방지의 핵심 함정**:

| 함정 | 잘못된 처리 | 올바른 처리 |
|---|---|---|
| 뇌혈관(I60-69) vs 뇌졸중(I63-64) vs 뇌출혈 | 이름만 같으면 동일 담보로 매칭 | **KCD 보장범위 코드까지 매핑** |
| 삼성 "암진단급부금" = 교보 "암진단보험금" | alias만 매칭하고 동일 처리 | **유사암(갑상선·경계성) 50%/100% 포함여부 플래그** |
| 표적항암·고액암 포함 차이 | 무시 | **보장범위 플래그로 분기** |

> 정규화는 단순 문자열 alias가 아니라 **보장범위(KCD 코드 + 유사암/표적항암 플래그)** 까지 내려가야 비교가 거짓말이 되지 않는다. 이 사전은 협회 공시·청약서를 따라 **갱신될수록 두꺼워지는 복리 자산**이다. 카피 시도 시 **6개월+ 추정**의 데이터 구축 비용이 든다.

#### 해자 2 — 갈아타기 자동 비교안내서 (§97 합법 엔진)

규제(부당승환)를 **기능으로 전환**한 유일한 플레이어가 되는 지점.

- **§97 부당승환 추정요건** (실무 정밀): "기존계약 소멸 후 6개월 이내 신계약 + 신계약 후 6개월 이내 기존 소멸"의 **양방향** 룰.
- **제품에 박는 안전판** (비교표 생성 시 강제):

  - [ ] 기존 청약일 · **예상 해지환급금(원금손실액)** 표기
  - [ ] 예정이율 · 갱신주기 차이 명시
  - [ ] **면책·감액기간 재기산 경고** (암 90일·감액 1~2년 리셋)
  - [ ] **"승환이 불리할 수 있음" 강제 체크**
  - [ ] 비교안내확인서 **자필서명 동선** + 6개월 룰 회피 가이드
  - [ ] 4세대 실손 전환 **손익분기 자동판정** (병원 이용 多 → 3세대 유지 유리 판정)

- **합법·신뢰를 동시에 잡는 현장 화법**: "갈아타세요"가 아니라 **"기존 거 살리고 빈 곳만 채우죠"(보완설계)**. 제품 카피·talk_track의 기본값을 보완설계로 둔다.
- **정직성 레드라인 (절대 양보 불가)**: 원탭 자동발송 금지 · 심의 배지 사칭 금지 · 라이브 OCR 금지. 신뢰가 곧 리텐션이다.

#### 해자 3 — 컴플라이언스 가드레일 + 위반 골든셋 회귀

전 티어 공통 위생이자 법무 진입장벽.

```
[ 3중 가드레일 ]
  ① 정규식 (PII·금칙어 1차 차단, 즉시)
        ↓ 통과
  ② Haiku 분류기 (부당승환·과장 표현 의미 판정, 저비용)
        ↓ 통과
  ③ 사람 검수 (정직성 레드라인 최종 확인)
        ↓
  위반 골든 100건 회귀셋 ── 룰셋 변경 시마다 재검증
```

- **과금 차별 없음**: 컴플라이언스는 무료~Premium **전 티어 공통**(생명선). 비교안내서 생성도 무료에서 무제한(컴플라이언스 차별 금지 + 바이럴 효과).
- 범용 GPT는 가드레일·8케이스 계산·정규화 사전이 없어 **부당승환 문구를 그대로 토한다** → 리스크. 우리는 **도메인 + 컴플라이언스 + 행동 트리거**의 교집합을 단독 점유한다.

> **단독 점유 명제**: "규제(부당승환)를 기능으로 바꿔 38만 시장(추정)에 합법 진입한 유일 플레이어 + 쓸수록 두꺼워지는 데이터 사전 복리 해자." 이 교집합 — **도메인 정규화 × 컴플라이언스 × 행동 트리거** — 에 동시에 서 있는 경쟁자는 현재 없다.

#### 차별점 3줄 요약

1. **해자 1 (정규화)**: 100+ 담보 틀 + KCD/유사암 플래그까지 내려간 정규화 사전 → 거짓 없는 비교, 카피 6개월+.
2. **해자 2 (비교안내서)**: §97 안전판을 박은 갈아타기 자동 생성 → 규제를 구매 이유로 전환.
3. **해자 3 (가드레일)**: 정규식→Haiku→사람 3중 + 위반 골든 100건 회귀 → 범용 AI가 못 넘는 법무 장벽.

## 4. 제품 — 기능 카탈로그

> 이 섹션은 Foliio 영업지원 에디션의 전체 기능을 **퍼널 단계 × 우선순위 tier**로 정리한다. 설계 원칙은 단 하나 — **분석은 끝점이 아니라 영업 액션의 트리거다.** 따라서 모든 기능은 "설계사가 오늘 무엇을 발송할 것인가"로 수렴한다. 발굴(신규 계약 발굴)이 시장의 병목이므로, **갈아타기 자동 비교안내서**와 **기존고객 보장공백 스캐너** 두 발굴-우선 기능을 카탈로그의 중심에 둔다.
>
> 표의 모든 숫자(개발 규모, 비율 등)는 **추정치이며 별도 표기**한다. 재활용자산은 현재 Foliio 코드베이스(v1.6.1) 기준 실재하는 모듈을 가리킨다.

### 4.0 읽는 법 — 범례

| 표기 | 의미 |

### 4.1 발굴 (Acquisition) — 카탈로그의 중심

| 기능 | 해결 문제 | AI 역할 | Tier | 재활용 자산 |
|---|---|---|---|---|
| **① 갈아타기 자동 비교안내서** | 신입이 지인 증권 1장으로 합법적 첫 계약을 만들 콜드스타트 wedge 부재. "갈아타세요" 화법은 §97 부당승환 위험 | **생성**: 비교표 초안 + talk_track 멘트 / **판정**: 6개월 룰·면책 재기산 경고 | **MVP★** | `customer/:id/compare/` + 8-case `calculate()` + `share_token` 공유링크 **♻** / §97 비교안내확인서 양식·면책 카피 **✦** |

| ③ 원클릭 니즈환기 (Analysis→Kakao) | 분석 화면이 끝점에 머물러 고객 접촉으로 이어지지 않음 | **생성**: 공백 요약 → 카톡 공유 문구 | MVP | `customer/:id/analysis/` + Kakao Share **♻ (~90%)** |
| ④ 다건 OCR + 담보 정규화 사전 | 증권마다 담보 명칭이 달라(삼성 "암진단급부금" vs 교보 "암진단보험금") 비교가 거짓말이 됨 | **추출**: PDF→담보 + **판정**: 유사암/표적항암 보장범위 플래그 | MVP | `core/ocr/claude_parser.py` **♻** / 담보 정규화 사전 + KCD 보장범위 매핑 **✦** |
| ⑤ 바이럴 진단 루프 · 리퍼럴 체인 | 발굴을 제품 자체에 내장 (소개받은 사람이 또 진단받게) | **생성**: 진단 카드 talk_track | **P2** | `promotion` 카드 렌더 **♻** / `source`·`referred_by` 귀속 필드 **✦** |

#### ① 갈아타기 자동 비교안내서 — 상세 (MVP★ 1번 wedge)

현장 검증 핵심: **"갈아타세요"가 아니라 "기존 거 살리고 빈 곳만 채우죠"(보완설계)** 가 합법·신뢰를 동시에 잡는다. 따라서 산출물의 디폴트 프레임은 "교체"가 아니라 "보완"이다.

비교표 생성 시 **강제 표기 항목** (§97 안전판):

```
┌─────────────────────────────────────────────────────────┐
│  [기존계약]              [제안]              [차액/경고]   │
├─────────────────────────────────────────────────────────┤
│  암진단 3,000만           5,000만           +2,000만      │
│  월 보험료 4.2만          5.1만            ▲ 월 +9,000원  │
│  ─────────────────────────────────────────────────────  │
│  ⚠ 예상 해지환급금 (원금손실)   : -310만원 (추정)        │
│  ⚠ 암 면책 90일 / 감액 1~2년 재기산 (해지 시 리셋)      │
│  ⚠ 예정이율·갱신주기 차이        : 2.5% → 변동형         │
│  ☐ "승환이 불리할 수 있음을 고지하고 이해함" (자필 체크) │
```

- **6개월 룰 판정**: 기존계약 소멸일 ↔ 신계약일 양방향 6개월 추정 매칭 → 부당승환 추정요건 진입 시 적색 경고.

- **AI 역할 한계**: AI는 *초안 생성*과 *규칙 위반 경고*만 한다. **"이 갈아타기는 적법하다"는 판정·심의 배지는 절대 출력하지 않는다** (정직성 레드라인).

#### ② 보장공백 스캐너 — 상세 (MVP★ 2번 wedge)

- **동작**: 야간 cron이 설계사의 전체 고객 DB를 순회 → 각 고객의 보유 담보를 **코어 8~10개 기준선**(3대진단비[암·뇌·심장] + 수술비 + 입원일당 + 실손 + 사망)과 대조 → 공백 점수화.
- **산출**: 설계사 대시보드 상단에 **"오늘 케어할 고객 N명"** 큐. 각 행에 공백 유형 + ④의 talk_track 1줄.
- **현장 정확성 함정 (반드시 반영)**: 뇌혈관(I60–69 전체) vs 뇌졸중(I63–64) vs 뇌출혈은 **KCD 코드 범위까지 매핑**해야 한다. alias만 매칭하면 "있다고 떴는데 실제론 보장 안 됨" 사고가 난다 → 보장범위 플래그 필수.

### 5.1 디자인 원칙 (이 섹션 전체에 적용되는 5계명)

1. **분석은 끝점이 아니라 트리거다.** 모든 분석 화면의 1차 CTA는 "보기 좋은 차트"가 아니라 "다음 영업 행동"(제안 만들기 / 진단 카톡 / 비교안내서)이다.
2. **수기 입력 의존 화면 금지.** 빈 폼을 사람이 채우게 두지 않는다. OCR 자동 캡처 → 정규화 사전 → 빈칸만 보완하는 "빈칸 채우기" UX로 설계한다.
3. **정직성 레드라인은 UI로 강제한다.** 원탭 자동발송 버튼·"금감원 심의 완료" 류 안전 배지·라이브 OCR은 **디자인하지 않는다**. 모든 AI 생성물 하단에는 면책 푸터가 고정으로 박혀 나온다.
4. **컴플라이언스는 전 티어 공통 위생이다.** 동의 게이트·면책·가드레일은 유료/무료를 차별하지 않는다. 과금 게이트는 "행동의 양(크레딧)"에만 건다.
5. **모바일 퍼스트.** 설계사는 고객 미팅 자리·이동 중에 쓴다. 데스크톱은 모바일 레이아웃의 확장으로 설계하되, 히트맵·비교표 등 정보밀도 높은 화면만 데스크톱 전용 멀티컬럼을 허용한다.

#### 5.4.3 갈아타기 비교안내서 — `/analysis/:id/compare` ★방패(필러②)

```
                monthly_premium 델타), §97_checklist[6](충족 여부), signature_status
4. CTA           1차: [비교안내서 PDF 생성] / 2차: [카톡 복사 📋] ※자동발송 불가

6. 모바일        델타 컬럼 가로스크롤, 체크리스트는 세로 스택, 서명은 풀폭 버튼
```

### 5.5 100개+ 담보 한눈 시각화 — 설계 디테일

#### 5.5.1 두 가지 뷰: 히트맵(개별 담보) ↔ 충족률(카테고리 롤업)

| 뷰 | 축 | 셀/막대 | 용도 |
|---|---|---|---|
| **히트맵(상세)** | 세로 15카테고리 × 가로 담보 leaf | 셀 색 = 충족도 3단계 | "정확히 어느 담보가 비었나" |
| **충족률(간략)** | 카테고리 1열 | 가로 막대 % + ⚠ 펑크 뱃지 | "어느 영역이 약한가" 빠른 스캔 |

상단 `[간략 | 상세]` 세그먼트 토글로 전환(기존 customer-analysis 토글 패턴 계승). 기본값=상세, 모바일 기본값=간략(정보밀도 고려).

#### 5.5.2 충족률 간략 뷰 와이어프레임

```
보장 충족률 (간략)            [간략 | 상세]
암      ████████░░  78%
실손    █████████░  92%
뇌혈관  ████░░░░░░  45%  ⚠
심장    ███░░░░░░░  30%  ⚠ 펑크
사망    ██████░░░░  61%
운전자  ░░░░░░░░░░   0%  □ 공백
... 9개 더
           [공백 큰 순 정렬 ▾]
```

#### 5.5.3 시각화 규칙

- **충족도 3단계**: 충분(≥80%, 인디고 `#3B5BDB`) / 부족(30~79%, amber) / 공백(<30%, 회색). 임계치(80/30)는 **(추정)** — 기준선 데이터셋(법무 Q-기준선) 확정 후 토큰화.
- **정렬 기본값**: "공백 큰 순" — 영업 우선순위와 일치.
- **⚠ 펑크 뱃지**: 핵심 카테고리(뇌·심장·암 등) 충족률이 임계 미만일 때 강조. 어떤 카테고리를 "핵심"으로 둘지는 PM·기준선 확정 사항.
- **모바일 가로스크롤**: 담보 leaf가 6칸 초과 시 가로스크롤, 첫 컬럼(카테고리명)은 sticky 고정.
- **부분 매칭 처리**: 정규화 사전이 매핑 못 한 담보는 셀을 점선 테두리 회색으로 두고 "미분류 N건" 칩 노출 → 신뢰성 과장 방지.

### 5.6 Freemium 한도 노출 UX

> **전제**: 베타는 무제한(`FREE_TIER_UNLIMITED=True`)이라 한도 UI는 비활성. 아래는 **정식 출시 시 4-tier(무료/Plus/Premium/Super) 활성화 가정 (추정)**. `0`은 "소진"이 아니라 **무제한 sentinel**(헤더 `∞`)임을 디자인 전반에서 일관 유지.

#### 5.6.1 한도 노출 3계층 (점진적 마찰)

| 계층 | 노출 위치 | UX | 트리거 |
|---|---|---|---|
| **상시(passive)** | 헤더 크레딧 게이지 | `이번 달 분석 2/3` 또는 `∞` | 항상 |
| **임박(nudge)** | 액션 실행 직전 인라인 | "이번 달 1건 남았어요" 노란 칩 | 잔여 ≤1 |
| **차단(block)** | 402 응답 시 모달 | UpgradeGuideModal | 크레딧 0 + 차감 액션 |

#### 5.6.2 크레딧 게이지 (헤더 상시)

```
┌──────────────────────────────┐
│ Foliio   고객 2/3 · 분석 8/10 │  ← 무료 티어 (추정)
└──────────────────────────────┘
┌──────────────────────────────┐
│ Foliio                    ∞   │  ← 베타/Super (무제한 sentinel)
└──────────────────────────────┘
```

- 데이터: `GET /api/v1/user_membership/credit_status/` 폴링. `remaining=null` 또는 `is_unlimited=true` → `∞`. **`remaining==0`을 무제한으로 오해 금지**(기존 트랩 — `monthly_*_credit=0`은 무제한 sentinel).
- 잔여 ≤1 시 게이지 색 amber로 전환.

#### 5.6.3 한도 초과 모달 (402 공통)

```
┌─ 이번 달 분석 한도를 다 썼어요 ─────┐
│  무료 플랜: 월 분석 3건 (추정)       │
│  Plus로 올리면 월 20건 · 비교안내서   │
│  무제한, 열람 추적까지.               │
│                                       │
│  [Plus 업그레이드]   [다음 달까지 대기]│
│  ⓘ 컴플라이언스(동의·면책)는 모든     │
│    플랜 공통 — 한도와 무관합니다.     │
└───────────────────────────────────────┘
```

- **과금 게이트 원칙**: 한도는 **"행동의 양"(분석·비교안내서·카피 생성 = 크레딧 차감)에만** 건다. 동의·면책·가드레일은 절대 유료화하지 않는다(원칙 4).
- 402 응답 shape: `{reason:'CREDIT_EXHAUSTED', limit, remaining}` → 모달이 `limit`/티어별 혜택을 동적 바인딩.
- 모달은 전 화면 공통 단일 컴포넌트(`<upgrade-guide-modal>`). 화면별로 다시 디자인하지 않는다.

#### 5.6.4 티어별 한도 (정식 출시 가정 — 추정)

| 티어 | 월 고객 등록 | 월 분석/AI | 비교안내서 | 열람추적 | 비고 |
|---|---|---|---|---|---|
| 무료 | 3 (추정) | 10 (추정) | 제한 | × | 콜드스타트용 |
| Plus | 20 (추정) | 100 (추정) | 무제한 | ○ | 핵심 BM 검증 타깃 |

## 6. AI(Claude) 활용 설계 & 데이터

> 이 섹션은 영업지원 에디션의 AI 레이어를 "어디서·왜·어떻게·얼마에" 관점에서 정의한다. 투자자에게는 비용 구조와 컴플라이언스 게이트가 곧 사업 리스크의 핵심이고, 디자이너에게는 AI 산출물이 화면 어디에 어떤 신뢰 신호와 함께 노출되는지가, 개발자에게는 입력→처리→출력→가드레일의 데이터 흐름이 구현 명세가 된다.

### 6.0 설계 원칙 (3줄 요약)

1. **AI는 끝점이 아니라 트리거다.** Claude는 "분석"이 아니라 설계사가 *발송할 영업 액션*(비교안내서, 멘트, 케어 큐)을 만든다 — North Star 지표(검수·발송된 액션 수)에 직접 연결된다.
2. **컴플라이언스는 전 티어 공통 위생이다.** 국외이전 동의 게이트와 3중 가드레일은 과금 차별 대상이 아니라 *출시 게이트*다 — 닫히지 않으면 AI 기능을 켤 수 없다.
3. **신규 코드는 얇은 레이어다.** 모델 라우팅·프롬프트·정규화 사전·게이트만 신규. 기존 OCR(`claude_parser`)·8케이스 계산·credit 엔진은 그대로 재사용.

---

### 6.1 AI 터치포인트 표 (기능 × 입력 → Claude 처리 → 출력 × 가드레일)

각 AI 기능을 "무엇이 들어가서, 어떤 모델이, 어떻게 처리하고, 무엇이 나오며, 어떤 가드레일이 걸리는가"로 한 줄씩 정의한다. 모델 ID·단가는 모두 검증된 현행값이다(§6.5).

| # | 기능 (필러) | 입력 | Claude 처리 (모델) | 출력 | 가드레일 |
|---|---|---|---|---|---|
| M1 | **갈아타기 비교안내서** (①+②) | 기존증권 OCR 결과(정규화 담보) + 제안 포트폴리오 + 고객 컨텍스트 | 담보 매칭·차이 산출 보조 + §97 양식 문안 생성 (`claude-opus-4-8`) | 비교표 초안 + 면책·고지 문안 (편집가능 draft) | **부당승환 차단** = 사실은 8케이스 계산값 인용, AI는 *서술만*; "심의완료" 배지 금지; 워터마크 "1차 보조·최종책임 설계사" |
| M2 | **보장공백 스캐너** (①+③) | 고객 DB 전체(야간 배치) + 표준 보장 기준선 | 고객별 공백 진단 + 우선순위 talk_track (`claude-haiku-4-5`, Batches) | "오늘 케어 N건" 큐 + 고객별 1줄 멘트 | 기준선 출처 고정(추정·근거 표기); 자동발송 금지(설계사 검수 후 클립보드 복사) |
| M3 | **AI 카톡 메시지 엔진** (①) | 목적 enum(7종) + 고객 요약 + 톤 | 목적별 메시지 생성 (`claude-haiku-4-5`) | 메시지 후보 1~3개 (복사용) | 가드레일 3중(§6.4) 통과 후에만 표시; 발송은 항상 수동 |
| M4 | **만기·갱신 워치독** (③) | CustomerInsurance 만기일 + cron | (대부분 규칙 기반) 알림 문안만 AI 보조 (`claude-haiku-4-5`) | 인앱/이메일 알림 + 케어 멘트 | 알림 문안에도 가드레일 적용; PII 마스킹 |
| M5 | **다건 OCR + 담보 정규화** (①) | 증권 PDF(텍스트 추출/스캔) | 담보 추출 → 표준담보 매핑 (`claude-opus-4-8`) | 정규화된 CustomerInsuranceDetail 행 | 국외이전 동의 게이트(§6.4) 통과 필수; 병력 마스킹; 정규식 느슨화 금지(명시리스트만 확장) |
| M6 | **니즈환기 원클릭** (①) | 기존 분석(analysis) 결과 | 분석 요약 → 환기 메시지 (`claude-haiku-4-5`) | Kakao 공유용 멘트 | 가드레일 3중; 공유 share_token은 noindex |

> **모델 라우팅 근거 (2티어 + 폴백)** — *정확도가 합법성을 결정하는* 곳(M1 비교안내서, M5 정규화)은 `claude-opus-4-8`($5/$25 per MTok, 1M context, adaptive thinking). 고빈도·저위험 콘텐츠 생성(M2~M4, M6)은 `claude-haiku-4-5`($1/$5 per MTok). 두 모델 모두 본 문서 작성 시점 검증값. M1에서 Claude의 거절(`stop_reason: "refusal"`)에 대비해 **서버사이드 폴백**(`fallbacks: [{model: "claude-opus-4-8"}]`)을 켜 두면 보안·생명과학 인접 작업의 오탐 거절도 같은 호출 안에서 복구된다.

### 6.2 담보 전체체계 데이터 모델 + 보험사별 담보명 정규화 파이프라인

갈아타기 비교의 *합법성*은 "삼성생명의 〈암진단급부금〉과 한화의 〈일반암진단비〉가 같은 담보인가"를 정확히 판정하는 능력에 종속된다. 이 판정의 단일 출처가 **담보 표준체계 + 정규화 사전**이다.

#### 6.2.1 데이터 모델 (신규 테이블)

```
CoverageCategory   (대분류 15+: 사망/암진단/뇌혈관/심장/CI/수술비/입원일당/실손/치료비/후유장해/...)
   id, name, life_or_loss(생/손 구분), sort_order

CoverageStd        (표준담보 leaf 100+ — 비교의 기준 단위)
   id, category_id(FK), std_name, payout_type(진단/수술/입원/사망/실손...),
   life_or_loss, description

CoverageAlias      (보험사별 표기 → 표준담보 매핑)
   id, coverage_std_id(FK), insurer_code(samsung/hanwha/db/...), alias_name,
   source(공시실/청약서/수동), confidence(정확/유사/보류)
```

설계 의도:
- **CoverageStd가 비교의 원자 단위.** M1 비교표의 행(row)은 표준담보 1개 = 양사 동일 기준에서만 "있다/없다/금액차이"를 말한다 → 부당승환 방어선.
- **CoverageAlias는 append-only 사전.** 보험사가 새 상품명을 내도 표준담보는 불변, alias만 추가. 협회 공시실 분기 갱신을 admin 배치로 흡수.
- **gender/age 연계.** 기존 `Customer.gender`(1=남/2=여)와 나이별 막대 타임라인이 보장공백 진단(M2)의 입력이 된다 — BE 무변경 원칙 유지(막대 도출은 FE).

#### 6.2.2 정규화 파이프라인 (M5 OCR 후처리)

```
OCR 추출 담보명("삼성 암진단급부금Ⅱ")
        │
        ▼
① alias 정확매칭   ──hit──▶ CoverageStd 확정
        │miss
        ▼
② fuzzy 매칭(3글자+, 명시 키워드 리스트) ──hit──▶ confidence=유사, 검수 플래그
        │miss
        ▼
③ 수동 입력 행(설계사가 표준담보 직접 선택)
```

**불변 규칙 (failures.md 등록 트랩 회귀 방지):**
- 진단비 카테고리에서 **처치/치료 키워드 제거** — "진단"만 매핑(2026-05 post-v1.6 픽스 계승). 신규 키워드는 *명시 리스트에 추가*, 정규식 느슨화 금지.
- 표준담보당 1개씩만 추출, 중복 매칭 시 최대값 유지.
- 손해보험에서 일반사망/재해사망 override 비활성(`insurance_type==1`에서만 적용).

Phase 0 산출물: **정규화 사전 v1 seed**(`sales-enablement/06-coverage-taxonomy-reference.md`) — 출시 게이트의 일부.

### 6.3 OCR 아키텍처 (기존 자산 재사용 + 정규화 레이어 추가)

기존 v1.2 OCR 파이프라인을 **그대로 재사용**하고, 그 출력 위에 §6.2.2 정규화만 신규로 얹는다.

```
PDF ──┬─ Claude 파싱(claude_parser.py, claude-opus-4-8) ── 1순위
      ├─ pdfplumber(테이블 구조 보존) ───────────────── 2순위
      └─ PyMuPDF ──────────────────────────────────── 폴백
                         │
                         ▼  (추출 텍스트, 최대 30,000자)
                  담보명 raw 리스트
                         │
                         ▼  ★신규 정규화 레이어(§6.2.2)
                  표준담보 매핑 + confidence
```

- **모델:** `claude-sonnet-4-20250514`(현행 OCR) → 정규화 정확도가 합법성에 직결되므로 신규 정규화 호출은 `claude-opus-4-8`로 승격 검토(투자 demo 핵심 경로).
- **암호화 PDF:** password 파라미터 지원(기존).
- **타임아웃:** 60초, 다건은 stream + idempotent(중복 차감 방지). 야간 일괄은 Batches API.
- **음수 보험료 guard:** `monthly_non_renewal_premium = max(0, …)` 계승(OCR 오류 방어).

### 6.4 컴플라이언스 게이트 (국외이전 동의 — 출시 게이트)

병력은 **민감정보**이고 Claude API는 미국(Anthropic, Inc.)에서 처리된다. 따라서 *개인정보 국외이전 동의*가 없으면 AI 호출 자체가 불가 — 이것이 영업지원 에디션의 가장 단단한 법무 게이트다.

#### 6.4.1 동의 게이트 (`ConsentGate` 미들웨어)

```
AI 호출 요청(M1/M5 등 병력·민감정보 포함)
        │
        ▼
[동의 레코드 존재? (버전·timestamp·IP 기록)]
   │No                          │Yes
   ▼                            ▼
402-style 차단                 Claude 호출 허용
{reason:'CONSENT_REQUIRED'}     (PII 마스킹 후 전송)
   │
   ▼
FE: 동의 유도 모달(customer-agree 확장)
```

- **재사용:** 기존 `customer-agree` 컴포넌트 확장 — 새 동의 항목(민감정보 국외이전) 추가.
- **레코드 스키마:** `consent_version`, `agreed_at`, `agreed_ip`, `scope(민감정보_국외이전)`. 동의 철회 시 AI 기능 자동 차단.
- **PII 마스킹:** 정규화 입력에서 주민번호/연락처 마스킹 후 API 전송(`content_filter.py` 재사용). **마스킹은 system 프롬프트 캐시 prefix를 깨지 않도록** 고객별 가변값은 캐시 breakpoint *뒤*에 배치(§6.5).
- **공개 카피 주의:** 개인정보처리방침은 "Anthropic, Inc. (US)" 명시 유지 — 법적 컴플라이언스, 제거 금지.

#### 6.4.2 컴플라이언스 가드레일 3중 (전 티어 공통)

설계사가 생성·발송하는 모든 AI 문안(M1·M3·M4·M6)에 적용:

```
1차: 정규식 block   (content_filter.py — PII/금칙어, 즉시 차단 400)
        │pass
2차: Claude 맥락검수 (claude-haiku-4-5 — "과장·오인·부당승환 소지" warn)
        │pass
3차: 사람 승인       (설계사 검수 → 클립보드 복사 → 수동 발송)
```

- **레드라인(정직성):** 원탭 자동발송 금지 · "심의완료/안전" 배지 금지 · 라이브 OCR 금지.
- **가드레일 골든셋:** Phase 0에서 위반 케이스 100건 작성 → 회귀 테스트.

### 6.5 비용 · 캐싱 (현행 단가 기준)

검증된 현행 단가(per MTok):

| 모델 | 입력 | 출력 | 컨텍스트 | 용도 |
|---|---|---|---|---|
| `claude-opus-4-8` | **$5** | **$25** | 1M | M1 비교안내서, M5 정규화 (정확도 critical) |
| `claude-sonnet-4-6` | $3 | $15 | 1M | (선택) 중간 난이도 |
| `claude-haiku-4-5` | **$1** | **$5** | 200K | M2~M4, M6 (고빈도·저위험) |

#### 6.5.1 캐싱 전략 (prompt caching)

- **정규화 사전을 system 블록에 고정** → `cache_control: {type:"ephemeral"}` breakpoint를 사전 끝에 배치. 캐시 **읽기 ≈ 0.1×** 입력단가, 쓰기 ≈ 1.25×(5분 TTL).
- **불변성 검증:** 사전 bytes 불변 유지(JSON `sort_keys`, 타임스탬프/UUID를 prefix에 넣지 않음). `usage.cache_read_input_tokens`가 반복요청에서 0이면 silent invalidator 존재 → 감사.
- **가변값은 breakpoint 뒤로:** 고객별 데이터·질문은 캐시 prefix 이후에 배치(고객마다 사전 전체가 재처리되지 않도록).
- **경제성:** 5분 TTL 기준 2회 호출이면 손익분기(1.25× + 0.1× < 2×). 정규화 사전처럼 매 OCR마다 동일 prefix를 쓰는 워크로드에 최적.

#### 6.5.2 Batches API (야간 일괄 50% 절감)

- **M2 보장공백 스캐너**(야간 전체 DB 배치)·**M4 만기 워치독 문안**은 latency 비민감 → **Batches API로 표준단가 50% 할인**. 최대 100,000 요청/배치, 대부분 1시간 내 완료(최대 24h).
- 단, Batches에는 서버사이드 폴백(`fallbacks`)·프롬프트캐시 효과가 제한되므로 *비교안내서(M1)는 실시간 경로 유지*.

#### 6.5.3 비용 가드 (운영)

- **호출당 비용 로깅** + 월 예산 캡 + 임박 알림(설계 원칙 ⑥). North Star "발송 액션당 AI 비용"을 보조지표로 추적.
- **토큰 카운팅:** `messages.count_tokens`로 모델별 사전 측정(tiktoken 금지 — Claude 토크나이저와 불일치). Opus 4.8과 Sonnet/Haiku는 토크나이저가 달라 모델별로 재기준선.
- **재시도:** SDK 자동 백오프(429/5xx). 401/403/400은 재시도 금지.

### 7.1 포지셔닝 · 메시징

#### 한 문장 포지셔닝
> **"빈손 신입이 첫 달부터 고객 앞에 앉는 무기 — 증권 사진 한 장이면 보장공백 진단부터 합법 비교안내서까지 자동완성."**

#### 타깃별 메시지 분기

| 세그먼트 | 핵심 통증 | 메시지 | 1차 가치 |
|---|---|---|---|
| **신입(0~2년)** | 고객·DB 자체가 없음, quota 압박 | "고객을 찾는 게 아니라, 지인 1명 증권만 받아도 상담이 굴러간다" | 증권 1장 → 30분 내 첫 제안서 |
| **중견(3년+)** | 묵힌 DB 방치, 재방문 명분 부재 | "묵힌 고객의 보장공백을 스캔해 만기·갱신 알림으로 재방문 명분을 만든다" | DB 재활성화 |
| **갈아타기 영업(공통)** | 부당승환(§97) 적발 공포 | "엑셀 비교는 누락·중복·부당승환 리스크. Foliio는 비교안내서를 자동으로" | 합법 방패 |

#### 안티-경쟁 프레이밍
- vs **엑셀/수기**: "담보명 정규화 없는 비교 = 누락·중복 = 적발 리스크"
- vs **범용 GPT**: "8케이스 보험료 계산·담보 정규화·컴플라이언스 게이트 없이 부당승환 화법 그대로 토함"
- vs **GA 사내툴(굿리치·인카·보험클리닉)**: "전속·폐쇄형이라 개인은 못 산다. Foliio는 위촉직 개인이 직접 결제하는 유일 무기"

**브랜드 톤**: "신입의 든든한 사수." 권위적이지 않고, 옆자리 선배가 알려주는 꿀팁의 톤.

### 7.2 1:1 방문영업 전략 (door-to-door)

설계사는 **지점 단위로 물리적으로 군집**한다(원수사 지점·GA 지점). 한 명 전환이 동료 5~10명 노출로 이어지는 구조라, 초기엔 1:1 방문영업이 광고보다 효율적이라는 게 핵심 가설이다.

#### 어디서 만나나 — 채널 우선순위 (CAC 추정 낮은 순)

| 우선 | 채널 | 접근 액션 | 한 번에 노출 | 근거 |
|---|---|---|---|---|
| ① | **GA/원수사 지점 조회(朝會)** | 지점장 1명 설득 → 화/목 아침 조회 5분 데모 슬롯 | 20~50명 | 지점장은 "신입 생산성" KPI 보유 — 도구를 사주진 않아도 무대는 내줌 |
| ② | **설계사 단톡방·네이버 카페** | "증권 1장→비교표" 30초 GIF + 추천코드. 모집글 아닌 '동료 꿀팁'으로 | 방당 수십~수백 | 바이럴 귀속의 본진 |
| ③ | **교육원·자격취득 학원** | 신입 입문 시점 = aha 골든타임. 강사 제휴 데모 | 기수당 20~40명 | 콜드스타트 타깃 정조준 |
| ④ | **보험박람회·MDRT 행사** | 부스 대신 로비·동선 QR 스티커 + 라이브 비교 시연(게릴라) | 행사 규모 | 저비용 |

#### 3분 방문 스크립트 (ASCII 플로우)

```
[훅 10초]  "증권 사진 한 장만 주시면 30초 안에 보장공백 보여드릴게요"
              └─ 본인 증권으로 즉석 시연 (말 X, 화면 O)
                    │
                    ▼
[아하 30초] 화면에 빨간 '공백 칸' 노출
              └─ "이게 고객 앞에선 곧 추가계약 명분입니다"
                    │
                    ▼
[방패 20초] "갈아타기 하실 때 비교안내서가 자동생성돼요
              — 부당승환으로 걸릴 일이 없어요"
                    │
                    ▼
[오퍼 + 클로징] "무료로 월 3건 분석 됩니다. 추천코드에 본인 이름 넣고
              지인 1명 가입시키면 두 분 다 분석 한도 +5건"
              └─ 그 자리에서 QR → 가입 (마찰 0)
```

#### 방문 오퍼 설계 원칙
- **현금 인센티브 금지, 한도(credit) 인센티브 사용** → 원가 낮고 제품 재방문 유도(기존 `monthly_credit` 구조 그대로 활용).
- **즉석 가입 마찰 제로**: 카카오 OAuth 단일 로그인 + QR로 그 자리 전환.
- **데모는 말이 아니라 화면**: 본인 증권으로 시연하는 게 가장 강력한 전환기.

### 7.3 디지털 채널

방문영업이 0→100을 만들면, 디지털 채널이 100→1000의 인바운드 엔진이 된다.

| 채널 | 콘텐츠 형식 | 빈도(목표) | KPI |
|---|---|---|---|
| **유튜브/블로그 (SEO)** | "신입 첫 달 계약 만드는 법" 류 롱테일 20편, 말미에 Foliio 자연 노출 | 초기 20편 시드 | 인바운드 가입(CAC 최저 가설) |
| **단톡방·카페 시딩** | 30초 시연 GIF + 추천코드 | 시드 50개 방 | K-factor 기여 |
| **인스타/카톡(성과카드 바이럴)** | 설계사가 14종 성과카드 템플릿으로 SNS 자랑 → 추천코드 노출 | 유저 주도 | 무료 노출수 |
| **카카오 채널** | 만기·갱신·온보딩 넛지 알림(기존 알림 인프라 활용) | 트리거 기반 | 리텐션 |

**콘텐츠 = 제품 재활용**: 분석 결과로 만든 성과카드(워터마크가 아닌 **추천코드**를 새김)가 곧 무료 바이럴 광고가 된다. 별도 광고 제작비 없이 유저가 광고주가 되는 루프.

> ⚠️ 컴플라이언스 레드라인: 디지털 어떤 채널에서도 "원탭 발송"·"금융위 심의 배지" 류의 오해 소지 카피는 절대 쓰지 않는다(신뢰=리텐션). 메시지는 항상 '초안 보조'로 한정한다.

### 7.4 초기 100 → 1000명 획득 플랜

North Star Metric은 **가입수가 아니라 "주간 활성 발송수"**(공유링크 생성 × 열람)로 둔다. 발굴 무기로서 실제 고객 앞에 쓰였는지가 가치의 척도다.

#### Phase A — 0→100 (1~6주, 손수 / hand-to-hand)

```
지점 조회 10곳  ×  평균 10명 전환  =  100명
       │
       ├─ 전원 1:1 온보딩 (증권 1장 업로드 → 공유 1회까지 손잡고)
       ├─ 주 1회 콜: "이번 주 그걸로 계약 났나요?"
       └─ 목표 aha rate 60% (= 증권 1장 업로드 + 공유 1회 달성 비율)
```
- 이 구간의 목적은 매출이 아니라 **PMF 시그널 + 추천 종잣불 확보**.

#### Phase B — 100→1000 (7~20주, 루프 가동)

| 레버 | 목표(추정) | 산출 |
|---|---|---|
| 추천 바이럴 | K-factor 0.4 (1명 → 추가 0.4명) | 복리 성장 시드 |
| 단톡방 시딩 | 50개 방 × GIF | — |
| 유튜브 인바운드 | 20편 → 월 ~300 유입(가설) | 검색 롱테일 누적 |
| 교육원 정기 슬롯 | 3곳 기수당 데모 | 콜드스타트 지속 공급 |

> 본 플랜의 전환율·K-factor·인바운드 유입수는 전부 **가설**이다. Phase A 실측으로 보정한 뒤 Phase B 투입 강도를 조정한다.

#### 획득 단계별 체크리스트

- [ ] 지점장 콜드 리치아웃 리스트 50개 작성 (원수사·GA 지점)
- [ ] 3분 데모 스크립트 + 본인 증권 시연 영상 준비
- [ ] 추천코드 = 가입 설계사 이름, 양방향 +5건 한도 로직 검증
- [ ] 성과카드 추천코드 각인 기능 확인
- [ ] 유튜브/블로그 SEO 키워드 20개 + 초안 5편 선제작
- [ ] 단톡방·카페 시드 50곳 매핑
- [ ] aha rate / 주간 활성 발송수 계측 대시보드(기존 analytics·pmf-admin 활용)

### 7.5 추천 · 바이럴 루프

수익성의 핵심은 광고비가 아니라 **두 겹의 바이럴 루프**다.

#### 루프 1 — 설계사 → 설계사 (추천코드)
```
설계사 A 가입
   │ 추천코드 = "A이름"
   ▼
동료 B가 코드 입력 후 가입
   │
   ▼
A·B 양쪽 monthly_credit +5건  (현금 X = 원가 낮음)
   │
   ▼
B가 또 동료에게 추천 …  (K-factor 누적)
```

#### 루프 2 — 설계사 → 일반 소비자 → 설계사 (셀프진단 역유입)
```
설계사가 고객에게 share_token 분석 링크 전송
   │
   ▼
링크 본 지인/잠재고객 "나도 내 보험 진단받고 싶다"
   │  └─ 분석 결과 하단 CTA: "내 보험도 무료 진단"
   ▼
일반인 가입 → 본인이 설계사거나, 설계사 지인에게 연결
```

#### 게이미피케이션 — 지점 리더보드
- 지점별 추천 랭킹 노출 → **지점장 자존심·경쟁 심리 자극** → 조회 재초청 명분 확보(②번 채널 강화 피드백).

### 7.6 Freemium 전환 퍼널

기능은 전면 개방하되, **무료는 `monthly_credit` 횟수 캡**으로 선을 긋는다. 컴플라이언스(비교안내서·OCR 위생 기능)는 **전 티어 공통 무료** — 위반 리스크가 과금에 걸리면 신뢰가 무너지므로, 과금은 'AI 콘텐츠 생성량·자동화'에만 건다.

#### 퍼널 (ASCII)

```
 가입(카카오 OAuth)
      │
      ▼
 [무료]  고객분석 월 3건 / OCR 10건   ← 기존 크레딧 구조 그대로
      │
      │  4번째 분석 시도
      ▼
 ┌─────────────────────────┐
 │ 402 CREDIT_EXHAUSTED     │ → UpgradeGuideModal
 │ 업그레이드 모달          │   (앵커: Premium 89,900 먼저,
 └─────────────────────────┘    Plus 19,900 "가장 인기")
      │
      ├─→ Plus 19,900  (전환 가설 8%)
      │        │
      │        └─→ Premium 89,900 (가설 3%)
      │
      └─→ 이탈(무료 유지) → 추천 루프로 잔존
```

#### 전환 트리거 & 골든타임
- **신입은 월말(quota 압박기)에 한도가 소진**된다 → 4번째 분석 시도 모달이 가장 잘 먹히는 골든타임.
- 전 티어 무료 구간: **비교안내서·기본 OCR·담보 정규화**(생명선이라 과금 차별 금지).

#### 가격 심리
| 기법 | 적용 |
|---|---|
| 앵커링 | Premium 89,900 먼저 노출 → Plus 19,900을 "가장 인기"로 |
| ROI 프레임 | "월 1건만 더 계약해도 시책 30만+ → 구독비 15배 회수" |
| 체감가 환산 | "Plus = 커피 4잔/월" |
| LTV 방어 | 연결제 2개월 무료 |
| 초기 락인 | **베타 가입자 평생 Plus 동결** → 초기 1,000명 묶기 |

> 전환율(무료→Plus 8% / Plus→Premium 3%)·무료 한도(3건이 맞는지 5건인지)·가격은 모두 **가설**이며, 베타 90일 데이터로 확정한다.

### 7.7 단위경제 · CAC/LTV 논리 (추정)

| 지표 | 추정값 | 비고 |
|---|---|---|
| ARPU | 월 25,000원 (가설) | Plus/Premium 혼합 |
| LTV | ~60만원 | 24개월 × ARPU 가정 |
| CAC | 3~5만원 | 지점 군집 전환 효과로 1:1 영업비 분산 |
| **LTV/CAC** | **12~20x** | 펀딩 서사 성립 임계선 |

핵심 논리: door-to-door가 표면상 비싸 보이지만, **지점 단위 군집 + 추천 루프**가 한 명 전환의 한계비용을 빠르게 떨어뜨린다. quota=생존인 직군 특성상 리텐션이 일반 SaaS보다 높다는 게 LTV 가설의 근거다.

```
        │
        ▼
[베타 90일]          무료 운영, PMF·전환·한도·가격 실측
  └ North Star: 주간 활성 발송수 / aha rate 60% / K-factor 0.4
        │
        ▼
[Wave 2 — 2차]       법무 클리어 후
  └ AI 보장 진단 + AI 카톡 화법 생성 (국외이전 동의·기준선 권위 확보 전제)
        │
        ▼
[정식 출시]          FREE_TIER_UNLIMITED=False 전환 → 4-tier 과금 개시
  └ 베타 가입자 평생 Plus 동결 약속 이행 (초기 1,000명 락인)
```

#### 출시 직전 체크리스트

```
펀딩       :  Seed ₩12억 / 런웨이 18개월 (BEP M12 + 6개월 버퍼)
레드라인   :  컴플라이언스(부당승환 방패)는 전 티어 공통 — 과금 대상 아님
```

핵심 논리: **Claude API 단가 급락(Batches 50% 할인·prompt caching read ~0.1배)** 으로 설계사 1인당 월 AI 원가가 수천 원대로 떨어졌고, 비로소 "개인이 월 2.5만 원 내는 AI 영업비서" 가 흑자 단위경제로 성립한다. 동시에 4세대 실손 전환·계약 갈아타기 수요가 구조적 피크여서 **승환 비교**가 본업이 되고, 그 합법 방패인 비교안내확인서 자동생성이 단일 최강 wedge다.

### 8.1 가격 설계 (3-tier · 개인 월구독)

| 티어 | 월가격 | 무료/포함 한도 (`monthly_credit`) | 타겟·근거 |
|---|---|---|---|
| **Free** | ₩0 | 분석 5건/월 · 카톡카피 10건/월 · OCR 10건/월 · 비교안내서 1건/월 | aha(첫 제안서)까지 무료. 헤비유저만 과금으로 넘어옴 |
| **Pro** | **₩24,900** | 분석 무제한 · 카피 200 · OCR 200 · **비교안내서 무제한** · 컴플라이언스 가드 | 가격민감 신입~중견 메인. 계약 1건 수당(15~40만)의 6~16% → 즉시 ROI |
| **Team** | **₩19,900 / 석** (3석+) | Pro 전체 + 공유 대시보드 + 지점 KPI | GA 지점 번들. 좌석당 20% 할인으로 군집 전환 유도 |

**기존 Foliio 가격과의 관계 (재포지션).** 베타기 4-tier(Basic 무료 / Plus 19,900 / Premium 89,900 / Super 견적)는 "포트폴리오 분석 도구" 가격선이었다. 영업지원 에디션은 타겟이 **가격민감 신입**으로 이동하므로, 베타 Plus 가설(19,900)을 "발굴 도구" 로 재포지션해 **Pro 24,900에 검증**한다. 89,900 Premium은 이 에디션의 단독 상품이 아니라, 기존 Foliio 사용자 중 분석 고도기능을 쓰는 상위 세그먼트로 분리 유지한다.

**과금 철학.** 기능은 전부 열되 `monthly_credit` **횟수 캡**으로만 과금한다(코드상 이미 구현된 크레딧 엔진 재활용 — `membership/credit.py`). **컴플라이언스 가드(부당승환 방패)는 전 티어 공통**으로 절대 과금하지 않는다 — 신뢰가 리텐션의 생명선이고, 규제 방패에 가격을 매기면 정직성 레드라인이 깨진다.

공시단가(per MTok): **Opus 4.8 $5/$25 · Sonnet 4.6 $3/$15 · Haiku 4.5 $1/$5 · Batches 50%↓ · prompt caching read ~0.1배.** (₩환산 1,400원/$ 가정)

| 작업 | 모델 | 입력/출력(토큰) | 건당 원가 | 최적화 후 |
|---|---|---|---|---|
| 분석 / 비교안내서 1건 | Sonnet 4.6 | 8K / 2K | $0.054 ≈ **₩75** | 담보사전 caching → **₩30** |
| 카톡 화법 카피 1건 | Haiku 4.5 | 1K / 0.5K | **₩4** | — |
| 증권 OCR 1건 | Sonnet 4.6 | 12K / 1.5K | **₩90** | Batches 야간 → **₩45** |
| 컴플라이언스 가드(warn) | Haiku 4.5 | 0.5K / 0.2K | **₩2** | — |

**Pro 헤비유저 월 AI 원가(추정).** 분석 40 + 카피 150 + OCR 30 가정:

```
분석   40 × ₩30  =  ₩1,200   (caching 적용)
카피  150 × ₩4   =  ₩600
OCR    30 × ₩90  =  ₩2,700   (caching/Batches 전)
                  ─────────
                   ≈ ₩4,500 / 월
```

매출 24,900 대비 **AI 원가율 ≈ 18%**. caching(담보사전 입력 90%↓)·Batches(야간 OCR 50%↓) 전면 적용 시 **10%까지 압축**이 목표. 이 압축은 마진 옵션이 아니라 **must** — 미적용 시 18%가 그대로 마진을 잠식한다(§8.7 리스크 3).

> 핵심 설계: **모델 티어링**. 비싼 Opus는 쓰지 않는다. 정형 추출·요약은 Sonnet, 짧은 화법·가드 판정은 Haiku로 내려 건당 원가를 한 자릿수 원까지 떨어뜨린다.

### 8.3 비용구조 ② — 인프라·CS·영업 (월, 추정)

| 항목 | 월 비용(추정) | 비고 |
|---|---|---|
| 인프라 | ≈ ₩0 (증분), 증설 시 +₩200k | 기존 IDC 자가호스팅 재활용 → 한계비용 거의 0 |
| CS | ₩1.5M (0.5 FTE) | 인앱·커뮤니티 셀프서브로 억제 |
| **영업(핵심 비용)** | ₩4M/인 (고정) + 성과급 | door-to-door 1인 = 월 신규 유료 30명 가정 |

영업이 비용구조의 무게중심이다. 설계사는 **지점 단위로 군집**하므로 1인 전환이 동료 5~10명에게 노출되는 구조적 레버가 있고, 이것이 door-to-door의 비싼 CAC를 정당화하는 유일한 근거다. 군집 노출이 추천코드 바이럴로 귀속되지 않으면 단위경제가 무너진다.

### 8.4 유닛이코노믹스 (설계사 1인, 전부 추정)

```
ARPU            = 24,900 × (Pro 80% + Team 20%)        ≈ ₩24,000 / 월
─ AI 원가                                                 ₩3,500
─ PG 결제수수료 (3%)                                       ₩750
─ CS 배분                                                  ₩1,000
─────────────────────────────────────────────────────────────────
월 기여이익                                               ≈ ₩18,750

Churn 6%/월 (신입 이직률 높음 → 보수적) → 평균 수명 16.7개월
LTV  = 18,750 × 16.7                                     ≈ ₩313,000
CAC  = door-to-door ₩4M ÷ 30전환 = ₩133k
        └ 디지털(콘텐츠·추천코드) 혼합 → 블렌디드 ₩90k
```

| 지표 | 값(추정) | 판정 |
|---|---|---|
| **LTV / CAC (블렌디드)** | **3.5x** | 목표 3x 충족 |
| LTV / CAC (door-to-door 단독) | 2.4x | 미달 — 디지털 혼합 필수 |
| **CAC 회수기간** | **4.8개월** | 12개월 내 목표 충족 |

**민감도 경고.** door-to-door 단독(2.4x)은 한계선이다. churn 5%·전환 35명으로 끌어올리지 못하면 LTV/CAC가 3x 아래로 깨진다. **추천코드 바이럴 귀속**이 CAC를 90k → 60k로 낮추는 유일한 레버이며, 이게 작동해야 단위경제가 안정 구간(>4x)에 든다.

```
LTV/CAC 민감도 (블렌디드 CAC 기준)
            churn 9%      churn 6%      churn 5%
CAC 90k     2.3x          3.5x          4.2x
CAC 60k     3.5x          5.2x          6.3x   ← 추천루프 작동 시
```

## 9. 기술 아키텍처·개발 로드맵·법무·리스크

> 이 섹션은 영업지원 에디션을 "기존 Foliio 자산 위에 얇은 신규 레이어"로 올리는 기술 전략, 단계별 개발 로드맵, 출시를 막는 법무 게이트, 그리고 PM이 코딩 전에 닫아야 할 오픈퀘스천을 정리한다. 모든 정량 수치는 **추정(estimate)** 임을 명시하며, 검증 전 의사결정 근거로만 사용한다.

### 9.1 아키텍처 — 기존 Foliio 재활용 맵

#### 9.1.1 결정: 분리 모놀리스 (monorepo, 코드 공유 / 배포·도메인 분리)

신규 Django 앱(`sales/`)을 기존 `Foliio_be`에 추가하되, FE는 `sales.foliio.co.kr` 서브도메인으로 분리하는 **분리 모놀리스** 구조를 채택한다. 근거:

- 기존 코드베이스가 100+ 파일(글로벌 룰 기준 "stack 유지, 마이그레이션 금지" 구간) → 별도 레포 분기·언어 전환 없이 **import 재사용**으로 중복 0.
- 타겟 사용자(분석 SaaS 사용자 vs 영업·발굴 절박 설계사)·가격·온보딩이 달라 **UI/도메인은 분리**해야 두 제품의 브랜드·핵심 동선이 섞이지 않음.
- 배포·롤백·관측을 도메인 단위로 끊을 수 있어 영업 에디션 장애가 본진 Foliio로 전파되지 않음.

#### 9.1.2 재활용 맵 (Reuse vs New)

| 영역 | 기존 자산 (그대로 import) | 신규 (얇은 레이어) |
|---|---|---|
| 보험료 계산 | `insurances/models.py` 8케이스 `calculate()` + `customers/calculate.py` 집계 | 없음 (그대로) |
| 증권 파싱 | `core/ocr/claude_parser.py` (Claude → pdfplumber → PyMuPDF 폴백) | 담보 정규화 후처리(§9.1.3) |
| 공유 링크 | `Customer.share_token` + 공개 retrieve/analysis/compare | 지인 셀프진단 동선·만료·회수·noindex 강화 |
| 비교 엔진 | `customer/:id/compare/` (기존 vs 제안 매트릭스) | 비교안내확인서 4종 산출(§9.4) |
| 크레딧/과금 | `membership/credit.py` (consume/sync), monthly_credit 엔진 | 영업 에디션 티어·횟수 캡 매핑 |
| 인증 | Kakao OAuth + DRF Token, dormancy/withdrawal 파이프라인 | 그대로 |
| 콘텐츠 필터 | `community/content_filter.py` (PII 정규식) | 메시지/광고 카피 가드레일로 확장(§9.4) |
| i18n/관측 | ngx-translate 6 locales, Sentry BE+FE, GA4 | 그대로 |

> 신규 코드는 **담보 정규화 사전 + 행동 트리거(니즈환기/공백진단/만기워치독) + 컴플라이언스 가드레일 + 비교안내서 생성**에 집중. 핵심 계산·OCR·과금 엔진은 0 라인 신규.

#### 9.1.3 데이터 모델 — 담보 표준 트리 + 정규화 사전 (신규)

영업 에디션의 단일 최강 해자이자 모든 합법성의 전제. 보험사별 담보명(alias)을 표준 담보(std)로 매핑하는 사전이 OCR·분석·승환 비교의 정확도 게이트다.

```
CoverageCategory(id, name)              # 15+ 대분류: 사망/암진단/뇌/심장/CI/치료비 …
CoverageStd(id, category_id, std_name,  # leaf 100+ 표준 담보
            life_or_loss, payout_type)
CoverageAlias(coverage_std_id,          # 삼성 "암진단급부금" → 표준 "일반암진단비"
              insurer_code, alias_name)
```

OCR 후처리 파이프라인:

```
원문 담보명
   │  ① alias 정확매칭 (CoverageAlias)
   ├─ hit ──────────────► 표준 담보 확정
   │  ② fuzzy (3글자+ 부분일치)
   ├─ hit ──────────────► 후보 제시 → 설계사 확정
   │  ③ miss
   └─────────────────────► 수동 입력 행
```

> **트랩 주의(§8 회귀)**: 처치/치료 과매칭으로 진단비가 오염되던 사례 → 정규식 느슨화 금지, **명시 리스트 확장**으로만 대응. 표준 담보는 1개씩만 추출, 중복 매칭 시 최대값 유지. 협회 공시실 기준 분기 갱신을 admin 배치로.

#### 9.1.4 AI 통합 — 모델 라우팅 3티어

> **단가 출처**: claude-api 스킬 기준 모델 단가. 입력/출력 단가는 스킬에서 확인한 값을 기재하되, 실제 청구 전 콘솔에서 재확인 필요(**확정 전 추정**으로 취급).

| 용도 | 모델 (codename) | 근거 |
|---|---|---|
| OCR 정규화·비교안내서 생성 | Opus 4.8 (`claude-opus-4-8`, 1M ctx) | 증권 파싱→표준담보 매핑은 부당승환 방패 = 정확도 게이트. 고비용 감수 |
| 메시지/콘텐츠 생성 (목적 다종) | Haiku 4.5 (`claude-haiku-4-5`) | 고빈도·저위험. 단가 최소 |
| 가드레일 2차(맥락 위반 warn) | Haiku 4.5 | 1차 정규식 block → 2차 Haiku warn → 3차 사람 승인 |

비용/지연 최적화 3종 (글로벌 외부 API 규칙 — 캐싱·rate limit·예산캡 준수):

1. **Prompt caching**: 담보 정규화 사전을 system block에 고정 → cache read(0.1×). 사전 byte 불변 유지, `cache_read_input_tokens`로 적중 검증.
2. **Batches API(50%↓)**: 야간 만기 워치독·콘텐츠 일괄 생성은 배치 처리.
3. **Stream + idempotent**: OCR 다건은 스트리밍, 재시도 시 중복 청구 방지.
4. **예산 가드**: 월 예산 캡 + 호출별 토큰·비용 로깅(글로벌 "API cost 로깅·월 예산 알림" 룰).

### 9.2 개발 로드맵 — 단계·Effort·마일스톤

> **Effort는 추정**(난이도 ★1~5). 인원·기간은 팀 규모 미확정이라 산정 보류 — PM이 인원 확정 시 환산. P0는 코드가 아닌 **법무·데이터 선결(critical path)**.

| Phase | 산출물 | 난이도(추정) | 마일스톤 게이트 |
|---|---|---|---|
| **P0** (코드 0) | 정규화 사전 seed + 동의서 개정(민감정보·국외이전 2단) + 가드레일 룰셋 v0 + 위반 골든 100건 + 보장 기준선 출처 확정 | ★★ (법무 critical) | Q1·Q2 닫힘 → 코딩 착수 가능 |
| **P1** (펀딩 demo) | 담보트리 모델 + OCR 정규화 + 니즈환기(M1) + 가드레일(M3) + **비교안내서 원클릭** | ★★★ | OCR→정규화→비교안내서 1동선 데모 가능 |
| **P2** | 공백 진단(M2) + 만기 워치독(M4) + 리드루프 귀속(추천코드 바이럴) | ★★★ | 발굴→제안→귀속 풀 루프 |
| **P3** | 정식 과금 전환(`FREE_TIER_UNLIMITED=False`) + 워터마크 활성 + 광고/통지 enum 분리 운영 | ★★ | 유료 출시 게이트 |

#### 마일스톤 의존 그래프

```
P0(사전·동의·룰셋·골든) ──┬──► P1: M1 니즈환기 ─┐
   Q1 기준선 ─────────────┤    P1: OCR 정규화 ──┼─► P1: 비교안내서(데모 핵심)
   Q2 국외이전 동의 ───────┘    P1: M3 가드레일 ─┘        │
                                                          ▼
                              P2: M2 공백진단 ── P2: M4 만기워치독 ── P2: 리드루프
                                                          │
                                                          ▼
                              P3: 과금전환 · 워터마크 · 광고/통지 분리
```

> **펀딩 데모 최소셋 = P1**: 증권 업로드 → 담보 정규화 → 비교안내서 원클릭 생성. 신규 코드가 얇은 레이어라 P1까지 빠르게 도달 가능. 회귀 안전망: 8케이스 골든 테스트(`pytest …test_premium_calculation_8cases.py`) + 위반 카피 골든 100건을 CI 게이트로.

### 9.3 법무·컴플라이언스 체크리스트 (출시 게이트)

#### 9.3.1 부당승환 (보험업법 §97③, 시행령 §44)

- [ ] **법문**: 신계약 청약 전후 6개월 내 기존계약 소멸 시 부당승환 추정. 예외요건 둘 중 하나 — ① 소멸 전 6개월 내 보장·보험료·해지불이익 **비교안내**(서면교부+확인), 또는 ② 계약자가 1개월 내 손실가능성 **자필서명** 확인.
- [ ] **합법 경로**: 비교엔진 산출 → 「비교안내확인서」 4종 자동 생성 — ⓐ보장차이표 ⓑ보험료차이 ⓒ해지불이익(면책기간 재적용·감액기간·해지환급금 손실) ⓓ고객 요청 자발성 확인.
- [ ] **시스템 강제**: 산출물에 생성일시·share_token·전자서명 로그 DB 영구보관(감독원 검사 대비). "승환 위험 D-180 타이머"로 6개월 윈도 시각화.
- [ ] **레드라인**: 시스템이 "승환하세요" 권유 카피 생성 금지 — 비교'안내'까지만, 권유는 설계사 책임.

#### 9.3.2 민감정보 국외이전 (개보법 §23 민감정보 별도동의 + §28-8 국외이전 별도동의) — **최우선 게이트**

- [ ] **법문**: 병력=민감정보 → 일반 동의로 커버 불가. 2단 별도동의 필수 — (A) 민감정보 처리동의, (B) 국외이전 동의(별도 체크박스 + 4대 고지: 이전받는 자=Anthropic Inc., 항목=병력 등, 국가=미국, 보유기간·거부권리).
- [ ] **합법 경로(우선순위)**: ① **PII 마스킹 선처리**(이름·RRN·연락처 정규식 마스킹 후 전송, `content_filter` 재사용) → ② **온프레미스 1차 파싱**(pdfplumber/PyMuPDF로 추출, Claude엔 비식별 텍스트만) → ③ 불가피한 민감정보 전송은 2단 동의 로그 없으면 **API 호출 자체 차단**.
- [ ] **시스템 강제**: `ConsentGate` 미들웨어 — 동의 레코드(버전·timestamp·IP) 없으면 AI 기능 차단(402-style). 동의 거부 = AI 비활성, 수기 입력 폴백.

#### 9.3.3 보험광고 사전심의 (보험업법 §95-4, 협회 심의)

- [ ] **법문**: AI 생성 판촉물·카톡 카피가 광고면 협회 사전심의 대상.
- [ ] **합법 경로**: ① 정형 알림(만기·생일=사실 통지)과 ② 가변 AI 카피(광고)를 **분리 enum**. 광고성 산출물엔 면책 워터마크: "본 콘텐츠는 1차 보조자료이며 최종 표현·심의책임은 설계사 본인에게 있습니다".

- [ ] **레드라인**: '심의 안전 배지' 절대 금지(보증책임 발생).

#### 9.3.4 무자격 모집·제휴 (보험업법 §83 무자격모집 금지)

| Q6 | 추천코드 **바이럴 귀속 모델** (CAC 절감 핵심) 설계 범위 | 리드루프(P2) | 귀속 규칙·보상 정책 확정 |
| Q7 | 만기 워치독 **알림 채널·빈도·정형/광고 경계** | M4 + 광고심의 | 통지/광고 enum 정책 확정 |

> **결론**: Q1(기준선 권위)·Q2(국외이전 동의)가 닫히기 전까지 P1 본 개발은 보류. P0 산출물(정규화 사전 사양·동의서 개정·룰셋 v0·골든 100)을 법무와 병렬 선결하면, 펀딩 데모(P1: OCR→정규화→비교안내서 원클릭)까지는 기존 자산 재활용 덕에 얇은 레이어로 빠르게 도달 가능하다.
