## 2. Context

- **Target users:** individually-contracted planners (sole proprietors) at carriers/GAs. Priority 1 = new planners (desperate for lead-gen); priority 2 = mid-career (management).
- **Value:** prospecting вҶ’ coverage analysis вҶ’ switch (к°Ҳм•„нғҖкё°/мҠ№нҷҳ) proposal in ONE flow. Analysis is the *start* of the sales action, not a report.
- **Differentiators:** (1) legitimize switching via an auto-generated л№„көҗм•ҲлӮҙм„ң (comparison guide) as a В§97 shield against improper-switching rules; (2) full 100+ coverage (лӢҙліҙ) "frame" + per-carrier coverage-name normalization.
- **BM:** Freemium (all features open, free monthly quota, heavy users subscribe).
- **Org:** CPO = CTO (the user). No external legal counsel вҶ’ compliance handled conservatively (safe defaults + public regulator/association guides), re-reviewed before paid launch.

## 5. Architecture (big picture вҖ” spans multiple files)

### Request flow

## 7. Gotchas (read BEFORE touching the area вҖ” these have bitten)

- **`update_fields` + model `save()` hook**: when a `save(update_fields=[...])` is used but the model's `save()` hook sets ANOTHER field (e.g. `Customer.save()` stamps `fa_reached_at` on stageвҶ’meeting), that field is NOT written unless it's in `update_fields`. Booking auto-promote includes BOTH (`['sales_stage','fa_reached_at']`). Bit once (2026-06-29). Same trap applies to any hook-derived field.
- **Email verification = BE self-contained token** (`accounts`): `signing.dumps(pk)` вҶ’ link and verify use `token` ONLY, no `uid` (UNLIKE reset-password). **If FE requires `uid`, signup is fully blocked вҖ” do NOT regress.** Resend button exists.
- **Consent token is multi-scope** (`customers/tokens.py`): `make_consent_token(customer, scopes=None)`; `read_consent_token` returns `{pk, scopes}` (legacy bare-int вҶ’ `{pk, scopes:['overseas_medical']}`, keep compat). `/c` GET вҶ’ `items[]`; POST `{agreed:[scope]}`; only token-scoped consents recorded (forgery guard); required scope unmet вҶ’ 412.
- **Consent subject enforcement:** planner-recorded consents are `subject=planner_attested` (server-forced) and CANNOT open the `consent_overseas_at` gate; only `customer_self` (via `/c` or self-diagnosis) opens it. **Registration modal no longer has planner self-check consent boxes** вҖ” consent is collected by sending the customer the `/c` link.
- **OCR normalization substring trap** (`core/ocr/ocrparsing.py::COVERAGE_KEYWORDS`): substring + longest-first means a name containing a shorter keyword mis-routes (e.g. 'мғҒн”јлӮҙм•”' contains 'м•”м§„лӢЁ'). Fix in dict DATA, NOT the logic. No migration. Add a regression test calling `_match_coverage` directly.
- **Booking timezone:** `availability.py` must keep ALL comparisons in KST aware. WorkHour/recur-block TimeFields are KST wall-clock (combine with the KST date); meetings/single events are UTC (`timezone.localtime`). Mixing them naively shifts 9h.
- **"This month" = KST, not `timezone.now()`** (`dashboard/`): the `__year`/`__month` ORM lookups bucket by `TIME_ZONE`(Asia/Seoul). So any "current month" window MUST use `timezone.localtime()`/`localdate()` вҖ” `timezone.now()`(UTC) or `date.today()`(OS-local, e.g. CI=UTC) mis-aligns and drops counts on the UTC/KST month-boundary day. Bit on 2026-07-01 (4 dashboard tests went red at midnight KST). Fixed; apply the same rule to any new monthly aggregation (e.g. `billing` UsageMeter).
- **`NEXT_PUBLIC_API_BASE` build-time inline:** changing it in Vercel needs a redeploy. If prod FE calls localhost, it's unset/not-redeployed.
- **`FRONTEND_BASE_URL` is effectively required** (default `localhost:3000`): unset in prod вҶ’ email-verify / consent / booking links generate as localhost = broken.
- **New custom domain = 4 env/console updates, ALL env-driven (no code)** (bit 2026-07-01 when `www.inpa.kr` went live вҶ’ login CORS-blocked, `access-control-allow-origin` absent on preflight while `in-pa.vercel.app` worked): on Render add the new origin to **`CORS_ALLOWED_ORIGINS`** (`prod.py::env.list`, default `[]`) AND **`CSRF_TRUSTED_ORIGINS`**, set **`FRONTEND_BASE_URL`** to the new domain, and add both `https://www.<domain>`+`https://<domain>` (www + apex) since they are distinct origins. Separately, add the domain to the **Google OAuth client's Authorized JavaScript origins** (Google Cloud Console) or the GSI button 403s ("origin not allowed for client ID"). `ALLOWED_HOSTS` = the BE host only (Render auto-injects its own). A 503 right after a merge is usually the Render free-tier redeploy/cold-start, not a real error (check `/healthz/`).
- **402 hidden in beta:** `FREE_TIER_UNLIMITED=True` bypasses all limit checks. BE already returns 402 `{code:'credit_exhausted', kind, limit, used}` for OCR/analysis/compare/promotion. Flipping the flag without an FE notice modal = users see red errors.
- **SQLite vs Postgres seed trap** (memory `sqlite-vs-postgres-seed-trap`): local SQLite ignores varchar length, prod PG rejects. Validate seeds/fixtures against PG too (e.g. `JobRiskCode.name` max 120). Prod `seed_demo` is manual via Render Shell.
