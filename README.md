# 인파 (Inpa) · 보험설계사 AI 영업 파트너

> **인파(Inpa) = 인슈어(Insure) + 파트너(Partner).**
> 위촉직 보험설계사 곁에서 **새 고객 발굴 → 보장 분석 → 갈아타기 제안 → 미팅 예약 → 고객 관리**까지 한 흐름으로 끝내주는 AI 영업 파트너입니다.
> 보장 분석·문구 생성은 **Claude AI** 가 합니다.

---

## 1. 한마디로

보험설계사가 하루에 증권 분석에 30분, 제안서에 40분씩 씁니다. 인파는 그 시간을 **3분·5분**으로 줄여서, 번 시간을 **고객 발굴과 클로징**에 쓰게 합니다.
즉 인파에서 분석은 '리포트'가 아니라 **영업 행동의 시작점**입니다.

## 2. 누구를 위한 건가

개인사업자처럼 일하는 **위촉직 보험설계사**(원수사·GA 소속).

- 1순위: **새내기 설계사** (고객 발굴이 가장 절실)
- 2순위: **중견·관리직** (관리·성과 집계)

> 헷갈리지 않게: 이 문서에서 '고객'은 두 종류입니다. **우리 고객 = 설계사**, **설계사의 고객 = 보험 가입·변경 희망자**.

## 3. 무엇을 하나 (한 흐름)

| 단계 | 하는 일 |
|---|---|
| **① 발굴** | '무료 1분 보장점검' 링크를 설계사가 고객에게 보냅니다. 고객이 증권을 올리고 점검을 받으면, **자동으로 내 고객 목록에 추가**됩니다. |
| **② 분석** | 증권(PDF)을 올리면 AI가 보장 내용을 **자동으로 정리**합니다. 담보 100여 개 '틀'에 맞춰 **신호등 색(넉넉 초록·적정 노랑·부족 빨강)** 으로 한눈에 보여줍니다. |
| **③ 제안·갈아타기** | 기존 보장과 새 제안을 비교하고, 갈아타기(승환) 제안을 만듭니다. 부당승환 규제를 합법적인 **'비교안내서'** 로 바꿔 방패로 씁니다. |
| **④ 고객 관리** | 영업 단계(DB·TA·FA·청약)와 고객 상태(진행중·보류·휴면·종료)로 파이프라인을 관리합니다. 오래 연락 못 한 고객 표시, 메모, 동의, 계약 설명의무 체크리스트까지. |
| **⑤ 일정·예약** | 설계사가 **업무시간만 정해두면**, 그 안의 빈 시간이 고객 링크에 자동으로 보입니다. 고객이 시간을 고르면 설계사가 알림에서 **수락**, 그러면 일정·구글 캘린더에 확정됩니다. |
| **⑥ 대시보드** | 월 목표와 실적, 계약 유지율, 관리직용 팀 성과(ROI)를 한 화면에. |

## 4. 핵심 차별점

- **갈아타기를 합법적으로.** 부당승환 규제를 자동 '비교안내서'로 바꿔 §97 방패로 씁니다.
- **담보 100개+ 전체 '틀'.** 보험사마다 다른 담보명을 정규화해서, 무엇이 부족하고 넉넉한지 한눈에 비교합니다.

## 5. 돈은 어떻게 (비즈니스 모델)

**Freemium.** 기능은 다 열어두되, 무료는 월 사용 한도를 두고, 많이 쓰는 헤비유저가 구독합니다. (구독 가격·결제는 정식 출시 때.)

## 6. 지금 상태

- **Phase 1 진행 중 · 운영 배포 중.** 백엔드(Django) 13개 앱 + 프론트(Next.js) 약 55개 화면이 구현·배포돼 있습니다.
- **배포:** 프론트 → Vercel (`https://in-pa.vercel.app`) · 백엔드 → Render · DB → Neon · 메일 → Resend. (Render 무료 플랜이라 한동안 안 쓰면 첫 응답이 느립니다.)
- **동작 중:** 구글 로그인·캘린더 연동, 셀프진단 발굴 링크, 증권 OCR 보장분석(신호등), 갈아타기 비교, 고객 파이프라인·상태 관리, 캘린더식 미팅 예약(수락/거절), 개인 일정, 동의 흐름, 대시보드(유지율·관리직 ROI), 직업급수 검색, 화법 라이브러리, 공유뷰 콘텐츠 보호, 관리자 사용량 트래킹.
- **정식 출시 전까지 잠가둔 것(법무 검토 대기):** 병력(민감정보) 수집, §97 비교안내서 고객 발송, 국외이전(Claude API), 유료 한도. 안전하게 기본값을 닫아두고, 출시 전 다시 검토합니다.

## 7. 기술 스택 (확정·구현됨)

| 영역 | 스택 |
|---|---|
| 프론트 | Next.js 16 + React 19 + TypeScript + Tailwind |
| 백엔드 | Django 4.2 + DRF + Python 3.11 (기존 Foliio의 OCR·계산·정규화 로직 재활용 = 핵심 자산) |
| DB | PostgreSQL (운영 = Neon / 로컬 = SQLite) |
| AI | Anthropic Claude API (분석·정규화 = Opus, 다건 OCR = Haiku) |
| 배포 | Vercel(FE) · Render(BE) · Neon(DB) · GitHub Actions(CI). 전부 자동배포, $0 |

## 변경 이력 (최근 → 과거, 요약)

> 자세한 결정 이력은 개발 가이드(CLAUDE.md)와 프로젝트 메모리에 누적됩니다.

- **2026-06-29:**
  - **고객 화면**: 영업 단계와 별개로 **고객 상태(진행중·보류·휴면·종료)** 도입(진행중 고객만 방치 표시, 나머지는 흐리게 + 접기). 칸반→**단계별**, 리스트→**목록**, 단계별 보드는 단계마다 상위 10명 + '더보기'. 고객을 누르면 **'정보'가 먼저** 보이고, 영업단계+상태는 한 줄, 요약카드에 최종 연락일·D+경과일.
  - **정보 입력**: 생년월일을 **년·월·일 드롭다운**으로, 보험나이 자동(만 나이+6개월 규칙), **기존 고객 직업도 검색해서 변경**(고르면 직업급수 자동, 메리츠 707개 정본). 동의 끝난 고객은 '동의 완료' 표시.
  - **예약을 캘린더식으로**: 업무시간만 정하면 빈 시간이 고객 링크에 자동 노출 → 고객이 신청(대기) → 설계사가 알림/일정 탭에서 **수락·거절** → 수락 시 확정. 미팅 앞뒤 여유시간(이동시간) 자동 차단, 중복 시 '담당 설계사와 상의' 안내.
  - **분석을 신호등 색으로**: 넉넉=초록·적정=노랑·부족=빨강, '중립' 표기는 없애고 기준선이 없으면 '기준 설정' 안내. 보장 한눈표 2단.
  - **대시보드/문구**: 월 보험료 막대의 금액을 막대 안에, 셀프진단 위젯 문구를 쉬운 말로, 캘린더 범례를 5분류로 통일(알림은 종 알림함으로).
  - **개발 원칙**: '설계사에겐 쉬운 말·긍정 유도, 고객/서비스 용어 분리'를 상시 규칙으로 명문화.
