# 인파 (Inpa) · 보험설계사 AI 영업 파트너

> **인파(Inpa) = 인슈어(Insure) + 파트너(Partner).**
> 위촉직 보험설계사 곁에서 **새 고객 발굴 → 보장 분석 → 갈아타기 제안 → 미팅 예약 → 고객 관리**까지 한 흐름으로 끝내주는 AI 영업 파트너입니다. 보장 분석·문구 생성은 **Claude AI** 가 합니다.

---

## 1. 한마디로

보험설계사는 증권 분석에 30분, 제안서에 40분씩 씁니다. 인파는 그 시간을 **3분·5분**으로 줄여서, 번 시간을 **고객 발굴과 클로징**에 쓰게 합니다. 분석은 '리포트'가 아니라 **영업 행동의 시작점**입니다.

**누구를 위한 건가:** 개인사업자처럼 일하는 위촉직 보험설계사(원수사·GA 소속). 1순위 새내기 설계사(발굴이 절실), 2순위 중견·관리직.

> 용어 정리: 이 문서에서 '고객'은 둘입니다. **우리 고객 = 설계사**, **설계사의 고객 = 보험 가입·변경 희망자**.

---

## 2. 기능 한눈에 (무엇이 들어있나)

### 발굴 (리드 만들기)
- **무료 1분 보장점검 링크(셀프진단):** 설계사가 고객에게 링크를 보냅니다. 고객은 **이름·연락처·생년월일·성별**(필수)을 남기고, 증권(PDF)을 올리면 보장을 1분 만에 정리해 줍니다. 분석 중에는 **인파 로고가 신호등처럼 퍼지는 화면**과 안내 문구가 돌아가며 떠서 기다림이 지루하지 않아요. **증권이 없어도 신청 가능** — 이때는 '담당 설계사가 직접 연락드릴게요' 안내가 뜨고 그대로 내 고객으로 접수됩니다. 점검·신청을 마친 고객은 **자동으로 내 고객 목록에 추가**됩니다. 동의는 상단 '전체 동의하기' 한 번으로 모을 수 있고, 마케팅·플랫폼 활용 동의는 **선택**(거부해도 진행)입니다.
- **내 소개 카드(디지털 명함):** 이름·소속·한줄소개가 담긴 공개 카드 링크를 카톡·문자·명함 QR로 뿌리면, 받은 분이 무료 보장점검을 받거나 '상담 신청'을 누르는 순간 **내 고객(DB)으로 자동 등록**됩니다. (판촉물 화면에서 링크 복사, 한줄소개는 설정에서 수정)
- **유입 경로 측정:** 소개·명함·행사·셀프진단 등 어디서 온 고객인지 기록.

### 보장 분석
- **증권(PDF) 자동 분석:** 증권을 올리면 AI가 보장 내용을 자동으로 읽어 정리(OCR + Claude).
- **담보 100여 개 '틀' + 보험사별 담보명 정규화:** 회사마다 다른 담보명을 하나로 맞춰 한눈에 비교.
- **신호등 보장 한눈표:** 넉넉(초록)·적정(노랑)·부족(빨강). 기준선을 정하지 않았으면 색 판정 없이 보유 내역만 보여주고 '기준 설정'으로 안내(근거 없는 단정 방지).
- **보험별 카드:** 한 고객이 보험을 여러 개 들고 있어도 **보험마다 카드**로(보험제목·종류·계약자·피보험자·월보험료·기간).
- **보험 직접 입력:** 증권 파일이 없을 때 손으로 보험을 추가.
- **요약:** 월 보험료·총 납입·보험 건수.

### 비교 분석 (갈아타기)
- **가입제안서 입력:** 가입제안서·설계서를 **업로드(자동 분석)** 하거나 **직접 입력**.
- **현재 vs 제안 비교:** 보험료와 담보를 나란히, 담보마다 **변동(추가·삭제·변경·유지)** 을 표시.
- **갈아타기 판정:** 유지가 유리/전환 검토 등 판단 근거(설계사 내부용, 고객에겐 안 보임).
- **비교안내서:** 부당승환 규제를 합법적인 '비교안내서'로(법무 검토 완료 후 활성).

### 고객 관리
- **영업 파이프라인:** DB·TA·FA·청약 4단계. **단계별 보드**(단계마다 상위 10명 + 더보기)와 **목록** 보기. 휴대폰에서는 단계 버튼(DB·TA·FA·청약, 인원수 표시)을 눌러 그 단계 고객만 보여요(좌우로 밀 필요 없이 버튼으로 전환).
- **여러 명 한 번에 등록:** 지인 명단을 한 줄에 한 명씩(이름·연락처) **붙여넣으면** 미리보기로 확인하고 한 번에 등록돼요(엑셀·CSV 몰라도 OK, 이미 있는 사람은 자동으로 건너뜀).
- **단계별 '다음 할 일' 버튼:** 고객 상세 위쪽에 지금 단계에서 바로 할 행동이 떠요. DB는 전화·문자·화법, TA는 예약 링크 보내기, FA는 분석 시작, 청약은 청약 체크리스트로 한 번에 이동. (대시보드의 영업 단계 카드를 누르면 그 단계 고객 목록으로 바로 갑니다.)
- **고객 상태:** 진행중·보류·휴면·종료. 진행중 고객만 방치 표시, 나머지는 흐리게 + '진행중만' 접기.
- **방치(무접촉) 표시 + 최종 연락일:** 오래 연락 못 한 고객을 옅은 표시로, 'D+경과일'로.
- **직업·직업급수:** 메리츠 직업표 707개에서 검색하면 직업급수 자동(등록·상세 모두 변경 가능).
- **보험나이 자동:** 생년월일을 넣으면 만 나이+6개월 규칙으로 자동 계산.
- **동의 관리:** 개인정보·마케팅 동의를 고객 본인이 링크로 직접, 완료되면 '동의 완료' 표시.
- **연락 기록:** 전화·문자 결과(부재중·연결·약속·거절·보류)와 메모를 남기면, 무접촉 표시가 초기화되고 '최근 연락'으로 보여요.
- **그 외:** 즐겨찾기·상단고정·'방금 연락함', 메모, 설명의무 체크리스트, 명함 보관, 아바타.

### 일정 · 예약 (캘린더식)
- **빈 시간 자동 노출:** 설계사가 업무시간(요일·시간)만 정하면, 그 안의 빈 시간이 고객 링크에 자동으로 보입니다.
- **신청 → 수락 → 확정:** 고객이 시간을 고르면 '대기'로 잡히고, 설계사가 알림이나 일정 탭에서 수락하면 확정. 수락하면 그 고객은 자동으로 'FA(대면)' 단계로 올라가요(만나기로 했으니까).
- **이동시간 자동 차단·중복 경고:** 미팅 앞뒤 여유시간을 자동으로 비우고, 같은 시간 중복이면 '상의' 안내.
- **구글 캘린더 연동:** 수락하면 구글 캘린더에 자동 등록.
- **개인 일정:** 일정·할일·고정 차단, 5분류 캘린더(고객미팅·생일/기념일·만기/갱신·업무·기타), 생일·기념일 매년 반복.

### 대시보드
- **새 디자인(2026.06.29 배포):** 좌측 메뉴(사이드바) + 한 화면 요약(목표·달성률·통계·영업 단계·차트·캘린더·오늘 일정). 모바일은 하단 탭바.
- **캘린더 날짜 클릭 → 일정 팝업:** 캘린더에서 날짜를 누르면 그 날 일정이 **모달 팝업**으로 떠요(✕ 또는 배경 클릭으로 닫기). '오늘의 일정' 카드는 캘린더에서 어느 달을 보든 **항상 금일 일정만** 보여줍니다.
- **목표·실적:** 이번 달 미팅·보험료·신규 고객, 월별 보험료 추이 차트.
- **단계 전환율:** 영업 단계(DB→TA→FA→청약)에서 지금까지 각 단계를 넘어간 비율을 보여줘요(어디서 막히는지 한눈에). 홈과 관리자 팀 화면 모두 표시.
- **계약 유지율:** 1·2·3년 유지율, 환수 레이더(미납·A/S 위험).
- **관리자 조직관리(팀장·지점장):** 성과 공유에 동의한 소속 설계사들을 한 화면에서. 팀 이번 달 보험료·신규 유치·활동 멤버 수, 팀 영업 단계, 월별 보험료 추이, 취급 보험(생명/손해) 비중, 그리고 **설계사별 순위표**(이번 달 보험료·신규·미팅·단계 분포·유지율·최근 접속). 이번 달 활동이 없는 팀원은 회색으로 표시(코칭 대상). 개별 고객 정보는 보이지 않고 집계 수치만(성과 수치는 추정). 팀원이 설정에서 **공유 단계**(공유 안 함 / 활동만 / 활동+실적)를 골라야 보여요. '활동만'을 고르면 고객수·신규·미팅·단계는 보이고 보험료·유지율은 '비공개'로 가려집니다.

### 알림 · 기타
- **알림:** 만기·생일·상담약속·할일·미열람 공유·셀프진단 리드·예약 요청 등, 우측 상단 종에서.
- **네비 배지(메뉴별 확인 개수):** 알림만이 아니라 **왼쪽 메뉴 옆에도 확인할 개수**가 빨간 숫자로 떠요. 고객(새 리드·생일·미열람) / 일정(예약 요청·만기·약속·할일) / 게시판(댓글·좋아요) / 판촉물(주문 상태·전자자료 준비) / 관리자(전자자료 제작 요청, 어드민) 메뉴 옆에 각각 표시됩니다. 모든 알림이 자기 메뉴로 분류되고, 알림을 읽으면 그 숫자도 같이 줄어듭니다(알림과 동일). 분석·화법·기준 등 들어오는 알림이 없는 메뉴엔 배지가 없어요.
- **화법/문구 라이브러리:** 상황별 메시지 템플릿(고객명·설계사명 자동 치환).
- **공유뷰:** 보장 한눈표를 고객에게 공유(워터마크·복사 차단). 업무시간을 정해 두면 고객이 보는 화면(공유뷰·자가진단 결과)에 **'바로 상담 예약하기'** 버튼이 떠서 고객이 그 자리에서 시간을 고를 수 있어요(업무시간 미설정이면 기존 안내문).
- **판촉물:** 명함·달력·리플렛 등 신청.
- **관리자 백오피스:** 약관·기능 플래그·설계사별 사용량 트래킹, 그리고 **설계사 상세에서 그 설계사가 보유한 고객 목록**(이름·연락처·단계·상태·보유 증권 수 등 비민감 정보만, 읽기 전용).

> **아직 잠가둔 기능(법무 검토 대기):** 병력(민감정보) 수집, 비교안내서 고객 발송, 국외이전(Claude API), 유료 한도. 안전하게 닫아두고 정식 출시 전 다시 검토합니다.

## 3. 핵심 차별점

- **갈아타기를 합법적으로.** 부당승환 규제를 자동 '비교안내서'로 바꿔 방패로.
- **담보 100개+ 전체 '틀'.** 보험사마다 다른 담보명을 정규화해 무엇이 부족·넉넉한지 한눈에.

## 4. 비즈니스 모델

**Freemium.** 기능은 다 열되 무료는 월 사용 한도, 많이 쓰는 헤비유저가 구독. (가격·결제는 정식 출시 때.)

- **요금제·구독 자동화(2026.07.01):** 가입하면 자동으로 '무료' 요금제 구독이 생기도록 정비했습니다. (그동안 프로드에 '무료 요금제'가 안 깔려 있어 구독이 안 만들어지던 문제를 배포 시 자동 시드로 해결. 기존 가입자에게도 무료 구독을 채워 넣음.) → **유료 전환을 켜기만 하면 되는 상태**가 됐어요. 요금제 가격·한도는 관리자 화면에서 바로 조정 가능하고, 정식 출시 때 확정합니다.

## 5. 지금 상태

- **Phase 1 진행 중 · 운영 배포 중.** 백엔드(Django) 13개 앱 + 프론트(Next.js) 약 55개 화면.
- **주소:** 서비스 → `https://www.inpa.kr` (기존 `https://in-pa.vercel.app` 도 유지) · 백엔드 → Render · DB → Neon. (Render 무료 플랜이라 한동안 안 쓰면 첫 응답이 느립니다.)
- **최근 업데이트(2026-07-01):** 로그인 정보 만료 시 자동으로 로그인 화면 안내, 일정 화면 예약 카드 정리, 고객 여러 명 등록 시 성별·생년월일·직업급수·유입경로·메모·아바타까지 한 표에서 입력, 회원가입에 소속·직책·설계사 번호(14자리)와 전체 동의, 담보 한눈표는 기준을 정하기 전엔 넉넉·적정·부족 대신 기준 설정으로 안내, 공지·자주 묻는 질문에 실제 내용 추가, 마이페이지에서 프로필 사진 등록, 이용약관·개인정보처리방침 정리.
- **커스텀 도메인 붙일 때 주의:** 새 도메인(www/non-www)은 백엔드 환경변수 `CORS_ALLOWED_ORIGINS`·`CSRF_TRUSTED_ORIGINS`·`FRONTEND_BASE_URL` 와 구글 로그인 허용 출처에 함께 넣어야 로그인이 됩니다(안 넣으면 CORS 차단).

## 6. 기술 스택

| 영역 | 스택 |
|---|---|
| 프론트 | Next.js 16 + React 19 + TypeScript + Tailwind |
| 백엔드 | Django 4.2 + DRF + Python 3.11 (기존 Foliio의 OCR·계산·정규화 로직 재활용) |
| DB | PostgreSQL (운영 = Neon / 로컬 = SQLite) |
| AI | Anthropic Claude API (분석·정규화 = Opus, 다건 OCR = Haiku) |
| 배포 | Vercel(FE) · Render(BE) · Neon(DB) · GitHub Actions(CI). 전부 자동배포, $0 |
